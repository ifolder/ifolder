Konfigurování nastavení místní brány firewall pro provoz aplikace iFolder

Pokud používáte místní bránu firewall, musíte ji nastavit tak, aby umožňovala aplikaci iFolder vybrat místní dynamický port nebo nakonfigurovat statický port pro klienta iFolder tak, aby komunikoval lokálně (stejný počítač) se serverem Mono XSP Server. iFolder používá webové služby Mono XSP ke komunikaci prostřednictvím protokolu HTTP s podnikovými servery Novell iFolder, kde máte účet iFolder. Správce iFolder konfiguruje služby iFolder, které určují, zda komunikace bude používat ve vašem prostředí protokol HTTP.

Brána firewall neblokuje provoz mezi Mono XSP a podnikovým serverem Novell iFolder, protože Mono XSP používá pro přenos standardní porty HTTP. Pokud však blokujete provoz mezi klientem a Mono XSP, klient nemůže komunikovat se serverem. Nemůžete se přihlásit k účtu iFolder a synchronizovat data iFolder.

V počátku konfigurace iFolder neudává určitý port pro použití ke komunikaci klient-Mono XSP. Když je prvně spuštěna aplikace iFolder, otevře se místní dostupný port pro poslech požadavků od Mono XSP a pro odesílání dat na server prostřednictvím Mono XSP. Zapíše číslo port do konfiguračního souboru a použije poté tento, pokud nebude dostupný vždy, když se aplikace iFolder spustí Pokud je port používán jinou aplikací, iFolder vybere jiný dostupný port a překonfiguruje jej v konfiguračním souboru a poté použije nový port. Pokud je například aplikace iFolder vypnutá a další aplikace port převezme, musí iFolder vybrat a nakonfigurovat jiný port pro použití při restartu aplikace iFolder.

  • Použití lokálního dynamického portu

  • Použití lokálního statického portu

Použití lokálního dynamického portu

Pokud ve svém počítači používáte místní bránu firewall, měla by detekovat místní provoz mezi aplikací iFolder a serverem Mono XSP vždy, když aplikace iFolder kontaktuje server pro synchronizování seznamu uživatelů a dat iFolder. Vyberte možnost Povolit a umožněte tak provoz aplikace iFolder pro tuto relaci. Pokud provoz umožníte trvale, vaše brána firewall by vás již neměla upomínat a měli byste se vyhnout zobrazování kontextových upozornění vždy při spuštění aplikace iFolder.

Pokud zablokujete provoz mezi aplikací iFolder a systémem Mono XSP, brána firewall zabraňuje v připojování k serveru iFolder a nemůžete tak své soubory synchronizovat. Aplikace iFolder nedetekuje, že provoz je zablokován bránou firewall; hlásí pouze, že se nelze připojit k účtu na serveru iFolder.

Použití lokálního statického portu

Volitelně můžete přidat parametr Rozsah portů webové služby do části správce služeb (Service Manager) souboru Simias.config a umožníte tak serveru Mono XSP Server používat statický port nebo rozsah portů pro tento místní provoz. Tento port nebo rozsah portů může být v bráně firewall otevřen a umožní se tak komunikace pro Mono XSP.

Soubor Simias.config naleznete v následujících umístěních:

Operační systém

Umístění souboru Simias.config

Linux

/home/uzivatelske_jmeno/.local/share/simias/Simias.config

Windows

C:\Documents and Settings\uzivatelske_jmeno\Local Settings\Application Data\simias\Simias.config

Chcete-li nakonfigurovat systém tak, aby používal určitý port nebo rozsah portů:

  1. Nakonfigurujte bránu firewall tak, aby umožňovala provoz pro rozsah portů, které chcete pro provoz aplikace iFolder používat.

    Informujte se také v dokumentaci k bráně firewall, kde jsou pokyny pro otevírání portu v bráně firewall.

  2. Zastavte klienta iFolder klepnutím pravým tlačítkem na ikonu iFolder, pak vyberte položku Ukončit.

  3. Změňte soubor Simias.config přidáním parametru Rozsah portů webové služby do části správce služeb (Service Manager).

    <configuration>

      <section name="ServiceManager">

        <setting name="WebServicePortRange" value="monoxsp_port" />   

      </section>

    </configuration>

    1. Nahraďte monoxsp_port číslem portu nebo čísly řady portů:

      • Zadejte hodnotu jednoho čtyřčíselného portu (například 1234) a použijte jeden statický port. Například:

        <setting name="WebServicePortRange" value="1234" />

      • Zadejte rozsah portů (například „1234-1236“) a použijte kterýkoliv dostupný port v zadané řadě. Například:

        <setting name="WebServicePortRange" value="1234-1236" />

  4. Spusťte klienta iFolder.

    Další informace naleznete v části Spuštění klienta iFolder.

Další informace získáte v části Právní upozornění.
