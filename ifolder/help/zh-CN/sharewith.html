共享 iFolder

可以与多个用户共享 iFolder。本节讨论以下任务：

  • 了解用户访问权限

  • 访问“共享”选项卡

  • 将用户添加到 iFolder

  • 修改用户访问权限

  • 从 iFolder 中去除用户

  • 将所有权转让给 iFolder 用户

访问“共享”选项卡

您可以通过“iFolder 属性”对话框中的“共享”选项卡管理 iFolder 的成员资格。

重要说明：仅当 iFolder 管理员已为您的帐户启用共享策略后，您才能共享 iFolder。

  1. 如果 iFolder 没有运行，请启动 iFolder。

  2. 进行下面的操作之一以打开“iFolder 属性”对话框，显示“共享”选项卡。

    • 在文件管理器中，右击要共享的 iFolder，然后选择“共享对象”。

    • 在 iFolder 浏览器中，选择 iFolder，然后选择“共享对象”。

将用户添加到 iFolder

  1. 在“共享”选项卡上，单击“添加”以打开“选择用户”对话框。

  2. 从用户列表中，通过将一个或多个用户或 LDAP 组添加到“要添加的用户”列表来选择一个或多个用户或 LDAP 组，然后单击“确定”。

    用户可能需要花费几秒钟来同步到服务器。同步后不久，页的右底部将出现一个通知。

  3. 在“共享”选项卡上，选择一个或多个要指派相同访问权限的用户。

  4. 单击“权限”，选择“完全控制”、“读/写”或“只读”访问权限，然后单击“确定”。

  5. 重复前面的步骤，直到已邀请所有需要的用户或 LDAP 组共享 iFolder 并为其设置了访问权限。

  6. 单击“关闭”来关闭“iFolder 属性”窗口。

修改用户访问权限

由于 iFolder 成员的工作关系和状态会发生改变，因此您可能需要更改某个用户对 iFolder 的访问权限。例如，您可能希望授予某个可信用户对 iFolder 的“完全控制”权限。

  1. 在“共享”选项卡上，选择一个或多个您希望他们拥有相同访问权限的用户。

  2. 选择“权限”，然后指定“完全控制”、“读/写”或“只读”权限。

    首次邀请一个用户或 LDAP 组参与时，“访问权限”对话框中的“将此用户作为 iFolder 的拥有者”选项是禁用的（灰显）。如果您要将新用户设置为拥有者，则必须等到该用户的状态从“已邀请用户”变为“iFolder 用户”，然后才能转让所有权。将所有权转让给 LDAP 组时，该组的所有成员都将获得该所有权。

  3. 单击“确定”，然后单击“确定”关闭“iFolder 属性”窗口。

从 iFolder 中去除用户

要取消共享一个 iFolder，必须去除用户或 LDAP 组的 iFolder 成员资格。用户或 LDAP 组成员的工作站上的 iFolder 将变为常规文件夹。数据的一份拷贝保留在前成员的工作站上，但文件不再与共享的 iFolder 同步。

  1. 在“共享”选项卡上，从列表中选择一个或多个用户。

  2. 单击“去除”，然后单击“是”。

将所有权转让给 iFolder 用户

一个 iFolder 的拥有者可以将该 iFolder 的所有权转让给状态为“iFolder 用户”的其他单个用户或 LDAP 组。转让所有权后，原拥有者将变为具有读/写权限的成员，新拥有者将对该 iFolder 占用的空间付费。当一个 LDAP 组是 iFolder 的拥有者时，该组的所有成员都将获得拥有权。

  1. 在文件管理器或 iFolder 浏览器中，右击要转让的 iFolder，然后选择“共享对象”打开此 iFolder 的“属性”对话框，显示“共享”选项卡。

  2. 从用户列表中，选择将成为新的拥有者的 iFolder 用户，然后单击“权限”。

  3. 选择“将此用户作为 iFolder 的拥有者”。

    如果用户的状态为“已邀请用户”，则“访问”对话框中的“将此用户作为 iFolder 的拥有者”选项被禁用（灰显）。在此情况下，请退出对话框，等到用户的状态从“已邀请的用户”变为 “iFolder 用户”后，再重复上述步骤转让所有权。

  4. 单击“确定”，然后单击“确定”关闭“iFolder 属性”窗口。

有关更多信息，请参见 法律声明.
