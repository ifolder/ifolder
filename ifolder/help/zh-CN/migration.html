从 iFolder 2.x 迁移到 iFolder 3.7 或更高版本

本节讨论以下内容：

  • 了解迁移过程

  • 迁移过程

了解迁移过程

Novell iFolder 提供了一种简单的迁移方法，可将 iFolder 2.x 客户程序迁移到 iFolder 3.7 或更高版本客户程序。iFolder 客户程序中提供的“迁移助理”功能可以帮助您轻松快捷地迁移现有的 iFolder 2.x 数据。

启动期间，“迁移助理”检测 2.x 安装并提示您迁移数据。数据迁移期间，系统再次提示您转换现有文件夹或者制作 2.x iFolder 的拷贝。有关此内容的更多信息，请参见步骤 6。

iFolder 提示您将文件重新同步到 3.7 或更高版本服务器，并询问您是否需要保留 2.x 客户程序，直到同步完成。在此过程中，只有数据得到迁移，因此用户必须在新的客户程序中设置首选设置。

迁移过程

  1. 安装 iFolder 3.7 或更高版本服务器的 iFolder 客户程序，然后为 iFolder 3.7 或更高版本 Enterprise Server 配置您的帐户。

    有关安装客户程序及设置帐户的信息，请参见管理 iFolder 帐户和首选设置。

  2. 在 iFolder 浏览器中，在本地计算机上创建一个或多个 iFolder。

    可以使用 iFolder 的任意有效名称。不要在与当前 iFolder 2.x 目录相同的目录路径中创建 iFolder。有关信息，请参见创建和上载 iFolder。

  3. iFolder 将显示出一条讯息，指出存在上一版本的数据并请求迁移。

    单击“确定”打开可用的 2 的列表。x iFolder 的列表。

  4. 选择要迁移的 2.x iFolder。

    单击“迁移”显示“迁移助理”。

    注：还可以通过“iFolder”>“迁移”>“从 iFolder 2.x 迁移”访问“迁移助理”。

    迁移助理将帮助您执行迁移过程以完成数据迁移。

  5. 单击“向前”以显示“迁移”选项。

  6. 请选中以下选项之一：

    迁移 iFolder 并断开它与 2.x 域的连接： 选择此选项可以将现有文件夹与 iFolder 2 断开连接。x 域的连接。

    创建 iFolder 的副本并迁移 iFolder： 选择此选项可以创建 iFolder 2 的副本。x 文件的副本并将其转换为 iFolder 3.7 或更高版本。此操作不会从 iFolder 2.x 域去除这些文件，即使将这些文件迁移到 iFolder 3.7 或更高版本后。

    选中此选项后，您还可以选中“复制父文件夹”，前提是您不仅要复制 2.x 文件夹的内容，而且还要复制父文件夹的内容。如果未选中此选项，则只将选定文件夹的内容迁移到 3.7 或更高版本。

    单击“浏览”查找保存迁移的文件夹的位置。

    例如，c:/Documents and Settings/<user name>/ifolder2dir 是一个 2.x iFolder，dir1 和 dir2 是 ifolder2dir 下的目录。选中选项“复制父目录”后，系统将在新位置创建 ifolder2dir 并将目录 dir1 和 dir2 复制到其中。如果未选中此选项，则只将目录“dir1”和“dir2”移动到指定位置。

  7. 单击“转发”可选择域和安全性选项，如下所示：

    1. 从下拉列表中选择域。

    2. 选中所需的安全性选项：

      • 选中“加密”以加密所选的 iFolder。

        如果登录过程中未设置通行口令，则 iFolder 会提示您输入通行口令。可以跳过此选项，以后再设置通行口令。

        如果当前未连接到 iFolder 域，系统会提示您连接到该域。

      • 选中“共享”可与 iFolder 域中的其他成员共享该 iFolder。

        您可将加密的 2.x iFolder 迁移到共享文件夹。当您尝试将加密的 2.x iFolder 迁移到一个共享文件夹时，系统将提示您检验该操作。

        单击“确定”确认操作。

  8. 单击“转发”可迁移选定的 iFolder。

  9. 校验输入信息是否正确，如有必要，单击“后退”返回到前面的页面进行改正。

    如果给定信息正确，请单击“迁移”以开始迁移。

    单击“后退”返回并进行所有必要的更改。

  10. 单击“完成”完成迁移。

有关更多信息，请参见 法律声明.
