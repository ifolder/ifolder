解决文件冲突

同时保存在不同计算机上的文件版本之间会发生冲突，或者文件名只是大小写不同时也会发生冲突。发生冲突时，文件会储存到冲突区域且状态会标记为“名称冲突”。iFolder 上会显示 iFolder 错误图标 。如果在“iFolder 首选设置”中启用了“冲突通知”选项，则还会向通知区域发送一个警报。您可以在“iFolder 首选设置” > “一般”选项卡中设置首选项来控制通知的弹出行为。

要解决文件冲突，请参见以下内容：

  • 解决文件版本冲突

  • 解决文件名冲突

解决文件版本冲突

要解决文件版本冲突，必须在“冲突解决方法”对话框中选择正确版本。

  1. 在通知区域中右击 iFolder 服务图标 ，然后单击“iFolder”以打开 iFolder 浏览器。

  2. 选择存在冲突的文件 ，然后单击“解决”以打开“解决冲突”对话框。

    您可查看同一文件的两种版本：

    • 本地版本： 保存在本地计算机上的文件。

    • 服务器版本： 保存在服务器上的文件。

    您可查看有关本地计算机上文件和服务器上文件的细节来确定要保留哪个文件版本。

  3. 对每个文件单击“打开”以查看该文件。

  4. 根据首选设置为“保存到 iFolder”选择以下任一选项。

    本地版本： 如果要将本地版本保存到 iFolder，请选择此选项。

    服务器版本： 如果要将服务器版本保存到 iFolder，请选择此选项。

  5. 选择“启用”可启用“保存到冲突文件箱”选项。启用此选项后，存在冲突的文件将被自动选中以保存到冲突文件箱。您的选择将决定将哪个版本保存到本地的冲突文件箱供以后使用。

  6. 清除冲突后，单击“保存”。

解决文件名冲突

本地文件版本的名称与服务器文件版本的名称大小写不同时会发生文件名冲突。文件内容可能不同，也可能相同。此类冲突可通过重命名文件来解决，但要注意重命名文件的位置。

  • 如果文件内容相同，则可使用“解决冲突”对话框来重命名其中的一个文件。

  • 如果文件内容不同，请确保创建本地文件的拷贝并为它指定一个不同的名称，然后返回至“解决冲突”对话框，保存文件的服务器版本。这允许文件的两个版本以不同的文件名存在于 iFolder 中。如果在“解决冲突”对话框中重命名一个文件版本，则该文件版本会覆盖另一文件版本，且另一文件的内容也会被破坏。

  1. 在通知区域中右击 iFolder 服务图标 ，然后单击“iFolder”以打开“iFolder 浏览器”。

  2. 选择存在冲突的文件 ，然后单击“解决”以打开“解决冲突”对话框。

  3. 确认它是文件名冲突，并确定文件内容是否相同。

  4. 执行以下操作之一：

    • 如果文件内容相同，则重命名其中一个文件，单击“保存该版本”，然后单击“确定”关闭“解决冲突”对话框。操作完成。

    • 如果内容不同，请继续执行步骤 5。

  5. 关闭“解决冲突”对话框。

  6. 打开文件管理器，并导航到冲突中文件的本地拷贝。

  7. 复制该文件并使用唯一的名称将其保存到 iFolder 中。

    iFolder 将该文件作为新文件同步到服务器和成员计算机中。

  8. 返回至“解决冲突”对话框，选择文件的服务器版本，单击“保存该版本”，然后单击“确定”。

    文件的服务器版本会下载到本地计算机中，并以该名称覆盖文件的本地拷贝。

有关更多信息，请参见 法律声明.
