查看和修改 iFolder 帐户设置

  1. 在“通知”区域中右击 iFolder 图标 ，然后选择“帐户”以打开“iFolder 自选设置”的“帐户”选项卡。

  2. 选择帐户以查看或修改以下参数：

    参数 说明

    服务器

    Enterprise Server 的 DNS 名称。例如：svr21.example.com。

    用户名

    此帐户的用户 ID。既可以使用网络用户 ID（LDAP 常用名）格式，也可以使用电子邮件地址格式，这由您的 iFolder 管理员决定。

    例如，如果您的姓名是 John Smith，用户 ID 是 jsmith，电子邮件地址是 john.smith@example.com，则管理员配置服务器的一个设置，该设置确定“用户名”字段应在鉴定过程中接受哪一种格式。

    只能为每个 Enterprise Server 创建一个帐户。

    口令

    与用户名对应的口令。

    记住口令

    如果选择“记住口令”，每当您登录计算机时就会自动以此 iFolder 帐户登录。有关信息，请参见登录 iFolder 帐户。

    如果网络口令更改，则在下一次 iFolder 尝试登录 iFolder 帐户时，自动鉴定将以合适的方式失败。iFolder 将提示您用新口令登录，并且允许您指定并保存新口令。

    启用帐户

    如果选择“启用帐户”，则允许对此帐户的 iFolder 邀请、用户列表和 iFolder 进行后台同步。这只控制客户端功能。

    从“帐户”列表的“状态”列中，可以看出帐户是处于“已启用”还是“已禁用”状态。如果您设置了多个帐户，则可以一目了然地看到“已启用帐户”设置的状态。

    如果将 iFolder 用于 Novell® iFolder® 3.x，则 iFolder 系统管理员可以在服务器端禁用您的 iFolder 帐户（通常是临时性的）。可能出现以下情况：

    • 如果在您已经登录的情况下从服务器端禁用帐户，您仍然可以管理 iFolder 并且 iFolder 数据继续与服务器保持同步，直到您通过注销或禁用同步结束会话，或者直到服务器断开您的会话（以先发生的为准）。
    • 如果帐户在服务器端被禁用，则在您尝试登录时将收到一个弹出讯息，通知您帐户已被禁用。请联系您的 iFolder 系统管理员获得帮助。

    默认帐户

    如果选择“默认帐户”，创建 iFolder 时帐户下拉列表中将默认选择此帐户。

  3. 在“iFolder 自选设置”的“帐户”选项卡上，选择 iFolder 帐户，然后单击“细节”查看以下有关 iFolder 帐户的信息。

    参数 说明

    服务器

    Enterprise Server 的主机名。例如，如果服务器的 DNS 名称是 svr21.example.com，则报告的信息为 svr1。

    说明

    Enterprise Server、简单服务器或工作组主机的、面向用户的友好说明，由设备管理员输入。如果未输入任何说明，则该值为“无”。

    可用空间

    此 iFolder 帐户上当前可用于您的 iFolder 的空间总量。该值等于空间定额减去已用空间。

    已用空间

    此帐户上您拥有的所有 iFolder 当前使用的空间总量。

    定额

    Enterprise Server 上分配给您的 iFolder 帐户的空间总量。此值可能是为所有用户设置的默认定额，也可能是某个用户特有的定额。如果未设置定额，则此值为空或 N/A（不适用）。
