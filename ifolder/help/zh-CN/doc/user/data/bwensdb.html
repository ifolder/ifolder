不适合同步的文件类型和大小

如果将 iFolder 客户程序与 Novell iFolder 3.x 配合使用，iFolder 管理员可能会在系统、用户或 iFolder 级别指定文件类型限制和最大文件大小限制。就您所拥有的 iFolder 中的数据可以使用的空间而言，您的帐户也可能受到限制。当您参与其它 iFolder 时，服务器上使用的空间将根据该 iFolder 的拥有者进行计算。

某些文件类型不适合进行同步，例如操作系统文件、文件管理器创建的隐藏文件或以关联文件集合的形式实现的数据库。您可以只加入业务中使用的主要文件类型，也可以排除可能与业务无关的文件，如 .mp3 文件。


操作系统文件

不应该将系统目录转换为 iFolder。大多数系统文件很少发生更改，保留基本系统和主要软件的映像文件比尝试将这些文件同步到服务器要好的多。


隐藏文件

如果您的文件系统使用隐藏文件跟踪显示自选设置，则管理员可能会限制这些文件类型，使这种文件类型无法从您的系统上同步。通常，隐藏文件只与创建这些隐藏文件的特定计算机相关，每次访问计算机上的文件或目录时，隐藏文件都将发生更改。您不需要保留这些隐藏文件，同步这些文件将导致重复文件冲突错误。

例如，iFolder 会自动排除两个隐藏文件管理器文件，名称为 thumbs.db 和 .DS_Store。


数据库文件

iFolder 同步各文件或各文件中发生更改的部分；它不会以集合的形式同步文件。如果具有以关联文件集合形式实现的数据库文件，则不要尝试在 iFolder 中同步它们。

不要尝试通过将 GroupWise 存档、超速缓存或远程目录放入 iFolder 中来同步 GroupWise® 数据。如果这样做，在文件同步几次之后，GroupWise 数据文件可能损坏。GroupWise 要求对存档中的文件以文件集的形式进行维护。
