创建 iFolder

通过转换符合以下准则的现有文件夹，您可以创建 iFolder：

  • iFolder 定位准则
  • iFolder 及其文件夹和文件的命名约定

如果这样的文件夹不存在，则可以浏览到某个位置并新建一个文件夹（作为整个过程的一部分）。

在 iFolder 中，您可以设置任何符合个人或公司工作习惯的子目录结构。该子目录结构在所有成员 iFolder 之间是一样的。每个工作站都可以为共享 iFolder 指定不同的父目录。

您可以在文件管理器和客户程序中的 iFolder 浏览器中直接创建 iFolder。创建 iFolder 的方法包括：

  • 在文件管理器中创建 iFolder
  • 在 iFolder 浏览器中创建 iFolder
  • 在 FAT32 装入点 (Linux) 上创建 iFolder


在文件管理器中创建 iFolder

  1. 使用文件管理器定位到要将其转换为 iFolder 的文件夹，然后选择该文件夹。

  2. 使用以下方法之一：

    • 右击文件夹图标，然后单击“转换为 iFolder”。
    • 在工具栏中，单击“文件”>“转换为 iFolder”。

    文件夹图标 变为交叠了 iFolder 图标 的文件夹。

  3. 如果出现“已创建 iFolder”讯息，请执行以下操作之一：

    • 单击“关闭”退出该讯息。
    • 选择“不要再次显示此讯息”（这样日后不会再显示该通知），然后单击“关闭”退出该讯息。

      如果日后希望显示“确认”讯息，则进入“iFolder 自选设置”，选择“创建 iFolder 时显示‘确认’对话框”，然后单击“确定”。


在 iFolder 浏览器中创建 iFolder

  1. 在“通知”区域中，右击 iFolder 图标 ，然后单击“iFolder”以打开 iFolder 浏览器。

  2. 单击“新建”。

  3. 指定要为此 iFolder 使用的 iFolder 帐户。

    如果具有多个 iFolder 帐户，则首先列出默认帐户。使用下拉列表选择其它帐户。

  4. 在浏览器中，定位到要在其中创建 iFolder 的目录。

  5. 使用以下方法之一，选择要令其成为 iFolder 的常规文件夹：

    • 如果常规文件夹已经存在，则选择该文件夹，然后单击“确定”。
    • 如果常规文件夹不存在，则单击“新建文件夹”创建它，选择新建的文件夹，然后单击“确定”。

  6. 单击“确定”。

    iFolder 将显示在 iFolder 列表中。

  7. 如果出现“已创建 iFolder”讯息，则执行以下操作之一：

    • 单击“关闭”退出该讯息。
    • 选择“不要再次显示此讯息”（这样日后不会再显示该通知），然后单击“关闭”退出该讯息。

      如果日后希望显示“确认”讯息，则进入“iFolder 自选设置”，选择“创建 iFolder 时显示‘确认’对话框”，然后单击“确定”。


在 FAT32 装入点 (Linux) 上创建 iFolder

在 Linux 上，iFolder 支持为 FAT32 文件系统的装入点创建和使用 iFolder。在 Linux 计算机的 FAT32 卷上，创建新的 iFolder 或设置共享 iFolder 之前，iFolder 用户必须拥有要在其上储存 iFolder 的 FAT32 装入点，并且 /etc/fstab 文件必须为装入点明确标识 iFolder 用户的本地用户 ID。

重要说明：  因为 FAT32 不支持文件权限，所以这是必需的。任何其它不支持权限的文件系统也有相同的问题。

对于双启动系统，您可以从没有将该文件夹设置为 iFolder 的系统访问 iFolder 中的数据，访问时将该 iFolder 视为常规文件夹。如果同时在两个平台上将文件夹设置为 iFolder，则服务器上将数据集储存在两个不同的 iFolder 中，这样占用的空间是为您分配的服务器磁盘资源的两倍。

因为 FAT32 文件系统不支持 POSIX 文件系统权限，所以请确保 iFolder 用户是装入点的本地拥有者。请修改 /etc/fstab 文件，将该用户的本地登录身份的用户 ID (UID) 参数设置为在该装入点上创建 iFolder 的 iFolder 用户。

  1. 以根用户身份登录计算机。

  2. 创建要在其中装入 FAT32 文件系统的目录。在命令提示符处，输入

    mkdir /home/username/fsmount

    将 username 替换为计划使用 iFolder 的 FAT32 驱动器的用户的用户名。将 fsmount 替换为要用作装入点的目录名。例如，如果用户名是 jsmith，FAT32 驱动器或分区目录名是 fat32mntpt，则输入

    mkdir /home/jsmith/fat32mntpt

  3. 在所需的装入点装入 FAT32 驱动器。在命令提示符处，输入

    mount -t vfat /dev/hda1 /home/username/fsmount

    将 /dev/hda1 替换为非 Linux 设备的设备或分区名称。将 username 替换为 iFolder 用户的用户名。将 fsmount 替换为要用作装入点的目录名。

    让我们来完成这个例子，假定 FAT32 文件系统 (vfat) 在 hda4 设备 (/dev/hda4) 上，用户名是 jsmith，装入点目录是 /home/jsmith/fat32mntpt，则输入

    mount -t vfat /dev/hda4 /home/jsmith/fat32mntpt

  4. 编辑 /etc/fstab 文件，方法是在定义 FAT32 装入点的行上添加 iFolder 用户的 uid=username 参数。

    /dev/hda1 /home/username/fsmount vfat uid=username

    下一步，将 uid=jsmith 添加到装入点，修改 /etc/fstab 文件：

    /dev/hda4  /home/jsmith/fat32mntpt  vfat  uid=jsmith
