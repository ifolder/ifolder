iFolder 及其文件夹和文件的命名约定

iFolder 客户程序规定的命名约定综合考虑了 Linux、Windows 和 Macintosh 文件系统的限制。iFolder、文件夹或文件必须具有符合这些命名约定的有效名称方可进行同步。

请为 iFolder 和其中的文件夹和文件使用以下命名约定：

  • iFolder 支持具有 UTF-8 编码的 Unicode* 字符集。
  • 不要在 iFolder 的名称以及 iFolder 中文件夹和文件的名称中使用以下无效字符： 
    \/:*?"<>|;

    如果在文件或文件夹名称中使用了无效字符，则 iFolder 将出现名称冲突。必须先解决该冲突才能同步文件或文件夹。

  • 单个路径组成部分的最大名称长度是 255 个字节。对于文件名，该最大长度包括句点 (.) 和文件扩展名。
  • iFolder、文件夹和文件的名称是不区分大小写的；但是将保留原有的大小写。如果文件名只是大小写不同，则 iFolder 将出现名称冲突。必须先解决该冲突才能同步文件或文件夹。
  • 如果在 Linux 中的 FAT32 文件系统上创建 iFolder，请避免使用全大写字符命名文件。对于由全大写字符组成的文件名，如果它符合 MS-DOS 8.3 文件格式，Linux 上的 VFAT 或 FAT32 文件处理会自动将其更改为全小写字符。如果 iFolder 是用“只读”访问权限设置的，则对于这些文件，上面的情况会引发同步问题。
