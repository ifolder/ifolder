iFolder 定位准则

通常，每当您创建 iFolder 或设置共享给自己的 iFolder 时，可以将 iFolder 放在本地硬盘驱动器上的任何位置。但是，在实际应用中有一些特殊的限制。当您在计算机上定位 iFolder 时，请注意以下几点：

  • 在 iFolder 所在的目录中，iFolder 的名称必须是唯一的文件夹名。
  • iFolder 的名称不能包含以下无效字符： 
    \/:*?"<>|;

    有关其它命名约定，请参见 iFolder 及其文件夹和文件的命名约定。

  • iFolder 必须驻留在本地硬盘驱动器上；不能在网络驱动器或非物理驱动器上。
  • iFolder 不能在驱动器的根目录下。
  • 只能从现有文件夹中创建 iFolder。
  • 只能在您对文件系统具有读写文件的访问权限的文件夹中创建 iFolder。当您在计算机上设置共享 iFolder 时，您必须对 iFolder 所在的文件夹具有文件系统访问权限。
  • iFolder 不能包含或驻留在系统文件夹中，如 Windows 文件夹、Linux 或 Mac 系统驱动器。
  • iFolder 不能包含或驻留在应用程序文件夹中，如 Program Files 文件夹。
  • iFolder 不能包含或驻留在 iFolder 元数据文件夹中。数据文件夹的默认位置如下所示：

    • Linux：在系统文件夹的 .local\share 子目录中

    • Windows：在 C:\Documents and Settings\Windows 登录用户名\Local Settings\Application Data\simias 目录中

    • Macintosh：在系统文件夹的 .local\share 子目录中

  • iFolder 不能包含或驻留在其它 iFolder 中（包括您的 Novell® iFolder® 2.1x iFolder）。
