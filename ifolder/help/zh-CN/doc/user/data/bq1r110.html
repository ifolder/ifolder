iFolder 的主要功能

在开始使用 iFolder 之前，应该了解以下主要功能：

  • iFolder 客户程序
  • iFolder 帐户
  • 共享 iFolder
  • iFolder 访问权限
  • 文件同步和数据管理
  • 同步日志


iFolder 客户程序

iFolder 客户程序与您的操作系统集成，可以在本机桌面环境中提供 iFolder 服务。iFolder 支持以下操作系统：

  • Novell Linux Desktop 9
  • Windows 2000/XP
  • Macintosh OS X 10.3 版或更高版本

iFolder 客户程序允许您管理自己的 iFolder 并选择在每台计算机上设置哪些共享 iFolder。有关信息，请参见管理 iFolder。


iFolder 帐户

在创建 iFolder 之前，您必须配置至少一个 iFolder 服务帐户。当您登录到某个 iFolder 服务帐户时，iFolder 会话开始；当您注销该帐户或退出 iFolder 客户程序时，iFolder 会话结束。只有当您的会话处于活动状态并且您正在联机工作时，您的 iFolder 才与其它活动 iFolder 同步文件。无论您是否登录到帐户，都可以随时访问本地 iFolder 中的数据。有关信息，请参见配置 iFolder 帐户。


共享 iFolder

iFolder 是一个本地目录，它通过中央计算机，有选择地与用户指定的用户组共享和同步文件。所有 iFolder 成员都可以访问 iFolder 文件，具有适当权限的成员还可以更改 iFolder 文件。您可以在多个工作站上访问您的 iFolder 并与他人共享。

您可以直接在您的文件管理器或从属于客户程序的 iFolder 浏览器中处理 iFolder。在 iFolder 中，您可以设置任何符合个人或公司工作习惯的子目录结构。该子目录结构在所有成员 iFolder 之间是一样的。共享 iFolder 时，具有“写入”权限的成员用户可以修改目录结构，这些更改将应用于该 iFolder 的所有副本。每个用户都可以按照 iFolder 定位准则的内容，将共享 iFolder 放在自己计算机的任意位置上。


iFolder 访问权限

对于 iFolder 成员，iFolder 客户程序支持拥有者和其它三个访问级别：

  • 拥有者：只有一个用户能充当 iFolder 的拥有者。通常是创建该 iFolder 的用户。拥有者用户可以通过 iFolder 客户程序将拥有权转让给该 iFolder 的另一个成员。

    iFolder 的拥有者具有“完全控制”权限。此用户拥有以下权限：读/写 iFolder、管理成员用户的成员资格和访问权限，去除任何成员的“完全控制”权限。

  • 完全控制：共享 iFolder 的成员，具有“完全控制”访问权限。此成员具有对 iFolder 的读/写权限，并有权管理所有用户（拥有者除外）的成员资格和访问权限。

  • 读/写：共享 iFolder 的成员，具有对 iFolder 中目录和文件的“读/写”访问权限。

  • 只读：共享 iFolder 的成员，具有对 iFolder 中目录和文件的“只读”访问权限。此成员可以修改文件，但本地更改不会同步到其他成员的 iFolder 副本。

任何 iFolder 用户都可以在拥有某些 iFolder 的同时成为其它 iFolder 的成员。拥有者用户始终具有“完全控制”权限。成员用户在每个共享 iFolder 中的访问级别可能有所不同，具体取决于任何具有“完全控制”权限的成员授予您的访问权限。


文件同步和数据管理

设置 iFolder 帐户时，可以启用“记住口令”，这样 iFolder 可以在您工作时在后台同步 iFolder 邀请和文件。

可以配置 iFolder 客户程序，使之在您每次登录计算机的桌面环境时自动运行。当您对本地 iFolder 中的文件进行处理时，会话在后台运行，跟踪并记录您所做的任何更改。

您可以按照指定的时间间隔同步文件，也可以按需同步。


同步日志

同步日志显示对 iFolder 后台活动的记录，例如为不同的 iFolder 上载和下载文件。
