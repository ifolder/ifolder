Konfigurowanie ustawień zapory lokalnej w celu umożliwienia komunikacji programu iFolder

W przypadku używania zapory lokalnej należy tak skonfigurować zaporę, aby umożliwić programowi iFolder wybranie lokalnego portu dynamicznego lub skonfigurowanie lokalnego portu statycznego dla klienta programu iFolder do celów komunikacji lokalnej (na tym samym komputerze) z serwerem Mono® XSP Server. Program iFolder korzysta z usług webserwisu Mono XSP do komunikacji poprzez protokół HTTP z firmowymi serwerami Novell iFolder, na których użytkownik ma konta iFolder. Administrator programu iFolder konfiguruje usługi iFolder określające, czy w środowisku użytkownika do komunikacji wykorzystywany jest protokół HTTP.

Zapora nie blokuje ruchu między serwerem Mono XSP i serwerem firmowym Novell iFolder, ponieważ serwer Mono XSP wykorzystuje do komunikacji standardowe porty. Jeśli jednak ruch między klientem i serwerem Mono XSP zostanie zablokowany, klient nie będzie mógł komunikować się z serwerem. Nie można zalogować się do konta programu iFolder i zsynchronizować danych z iFolderu.

Początkowo konfiguracja programu iFolder nie podaje określonego portu służącego do komunikacji z klienta z serwerem Mono XSP. Podczas pierwszego uruchomienia program iFolder otwiera dostępny port lokalny do nasłuchiwania żądań z serwera Mono XSP i wysyłania danych do serwera przez serwer Mono XSP. Numer portu jest zapisywany w pliku konfiguracyjnym i później wykorzystywany, chyba że nie będzie dostępny podczas uruchamiania programu iFolder. Jeżeli port jest wykorzystywany przez inną aplikację, program iFolder wybierze inny dostępny port, zmieni port w pliku konfiguracyjnym i będzie korzystać z tego portu. Jeśli na przykład program iFolder jest wyłączony i inna aplikacja przejmuje port, program iFolder musi przy ponownym uruchomieniu wybrać i skonfigurować do wykorzystywania inny port.

  • Korzystanie z lokalnego portu dynamicznego

  • Korzystanie z lokalnego portu statycznego

Korzystanie z lokalnego portu dynamicznego

Jeśli na komputerze wykorzystywana jest lokalna zapora, powinna ona wykrywać lokalny ruch między programem iFolder i serwerem Mono XSP za każdym razem, gdy program iFolder łączy się z serwerem w celu zsynchronizowania listy użytkowników i danych programu iFolder. Zaznacz opcję Zezwalaj, aby umożliwić komunikację z programem iFolder dla tej sesji. Jeśli zgoda na komunikację zostanie wydana na stałe, zapora nie powinna ponownie przypominać o tym, dzięki czemu można będzie uniknąć wyskakujących alertów przy każdym uruchomieniu programu iFolder.

Jeśli komunikacja między programem iFolder i serwerem Mono XSP zostanie zablokowana, zapora uniemożliwi połączenie się z serwerem iFolder i nie będzie można zsynchronizować plików użytkownika. Program iFolder nie wykryje zablokowania komunikacji przez zaporę; zgłosi jedynie niemożność połączenia do konta użytkownika na serwerze iFolder.

Korzystanie z lokalnego portu statycznego

Opcjonalnie można dodać parametr Web Service Port Range w sekcji Service Manager pliku Simias.config, aby zezwolić na wykorzystanie przez serwer Mono XSP statycznego portu lub zakresu portów dla tego ruchu lokalnego. Ten port lub zakres portów może być następnie otwierany w zaporze, aby umożliwić komunikację z serwerem Mono XSP.

Plik Simias.config można znaleźć w następujących lokalizacjach:

System operacyjny

Lokalizacja pliku Simias.config

Linux

/home/nazwa_użytkownika/.local/share/simias/Simias.config

Windows

C:\Documents and Settings\nazwa_użytkownika\Local Settings\Application Data\simias\Simias.config

Aby skonfigurować system do wykorzystywania określonego portu lub zakresu portów:

  1. Skonfiguruj zaporę tak, aby zezwalała na ruch dla portu lub zakresu portów, który ma być wykorzystywany przez program iFolder.

    Informacje dotyczące otwierania portów w zaporze można znaleźć w dokumentacji zapory.

  2. Zatrzymaj klienta programu iFolder, klikając prawym przyciskiem myszy ikonę iFolderu, a następnie wybierając opcję Zakończ.

  3. Zmodyfikuj plik Simias.config, dodając parametr Web Service Port Range w sekcji Service Manager.

    <configuration>

      <section name="ServiceManager">

        <setting name="WebServicePortRange" value="monoxsp_port" />   

      </section>

    </configuration>

    1. Parametr monoxsp_port należy zastąpić numerem portu lub zakresem numerów portów:

      • Aby używać pojedynczego portu statycznego, należy podać wartość, będącą czterocyfrowym numerem portu (taką jak 1234). Na przykład:

        <setting name="WebServicePortRange" value="1234" />

      • Aby używać dowolnego dostępnego portu z podanego zakresu, należy podać zakres portów (taki jak "1234-1236"). Na przykład:

        <setting name="WebServicePortRange" value="1234-1236" />

  4. Uruchom klienta programu iFolder.

    Więcej informacji można znaleźć w dokumencie Uruchamianie klienta programu iFolder.

Aby uzyskać więcej informacji, patrz sekcja Informacje prawne.
