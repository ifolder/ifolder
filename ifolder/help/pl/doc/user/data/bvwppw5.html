Konfigurowanie ustawień zapory lokalnej w celu umożliwienia komunikacji programu iFolder

W przypadku korzystania z zapory lokalnej trzeba ją tak skonfigurować, aby pozwalała programowi iFolder na wybór lokalnego portu dynamicznego, lub skonfigurować lokalny port statyczny przypisany do klienta programu iFolder w celu nawiązywania komunikacji lokalnej (na tym samym komputerze) z serwerem Mono XSP. Program iFolder korzysta z usług sieciowych Mono XSP do komunikacji przy użyciu protokołu HTTP lub HTTPS (SSL, opcja domyślna) z serwerami systemu Novell iFolder, na których znajdują się konta iFolder. Administrator systemu iFolder konfiguruje usługi iFolder, określając, czy komunikacja korzysta w danym środowisku z protokołu HTTP czy HTTPS.

Zapora nie blokuje ruchu między serwerem Mono XSP a serwerem systemu Novell iFolder, ponieważ serwer Mono XSP do celów ruchu sieciowego korzysta ze standardowych portów HTTP lub HTTPS. Jeśli jednak ruch między klientem a serwerem Mono XSP zostanie zablokowany, klient nie będzie miał możliwości komunikowania się z serwerem. Użytkownik nie będzie mógł zalogować się do konta iFolder ani zsynchronizować danych w iFolderach.

Początkowa konfiguracja programu iFolder nie określa konkretnego portu używanego do celów komunikacji między klientem a serwerem Mono XSP. Podczas pierwszego uruchomienia program iFolder otwiera dostępny port lokalny, przez który prowadzi nasłuch żądań z serwera Mono XSP i wysyła dane do serwera za pośrednictwem usługi Mono XSP. Program zapisuje ten numer portu w pliku konfiguracyjnym i korzysta z niego w kolejnych sesjach, chyba że ten port nie jest dostępny w momencie uruchomienia programu iFolder. Jeśli dany port jest używany przez inną aplikację, program iFolder wybiera inny dostępny port, zmienia konfigurację portu w pliku konfiguracyjnym i w kolejnych sesjach korzysta z tego nowego portu. Jeśli na przykład program iFolder jest wyłączony i inna aplikacja przejmuje dany port, podczas ponownego uruchamiania programu iFolder musi on wybrać i skonfigurować do użycia inny port.


Korzystanie z lokalnego portu dynamicznego

Jeśli komputer korzysta z zapory lokalnej, zapora powinna wykrywać ruch lokalny między programem iFolder i serwerem Mono XSP podczas każdego kontaktu programu iFolder z serwerem nawiązywanego w celu zsynchronizowania listy użytkowników i danych w iFolderach. Aby uniknąć wyświetlania alertów w okienkach podręcznych przy każdym uruchomieniu klienta, należy zezwolić na ten ruch sieciowy. Jeśli zezwolenie na ten ruch sieciowy będzie stałe, zapora nie powinna wysyłać więcej powiadomień.


Korzystanie z lokalnego portu statycznego

W sekcji Service Manager pliku Simias.config można opcjonalnie dodać parametr Web Service Port Range, aby umożliwić serwerowi Mono XSP wykorzystanie portu statycznego lub zakresu portów do obsługi ruchu lokalnego. Ten port lub zakres portów może być otwierany w zaporze w celu umożliwienia komunikacji z serwerem Mono XSP.

Plik Simias.config można znaleźć w następujących lokalizacjach:

System operacyjny Lokalizacja pliku Simias.config

Linux

/home/nazwa_użytkownika/.local/share/simias/Simias.config

Windows

C:\Documents and Settings\nazwa_użytkownika\Local Settings\Application Data\simias\Simias.config

Macintosh

/Users/nazwa_użytkownika/.local/share/simias/Simias.config

Dodawanie parametru Web Service Port Range do sekcji Service Manager w pliku Simias.config. Na przykład:

<configuration>
  <section name="ServiceManager">
    <setting name="WebServicePortRange" value="port_monoxsp" />   
  </section>
</configuration>

W miejsce port_monoxsp wpisz numer portu lub zakres numerów portów:

  • Aby korzystać z pojedynczego portu statycznego, określ jeden czterocyfrowy numer portu (na przykład 1234). Na przykład: 
    <setting name="WebServicePortRange" value="1234" />
  • Określ zakres portów (na przykład "1234-1236"), aby korzystać z dowolnego dostępnego portu z podanego zakresu. Na przykład: 
    <setting name="WebServicePortRange" value="1234-1236" />

Aby skonfigurować system do korzystania z określonego portu lub zakresu portów:

  1. Skonfiguruj zaporę tak, aby umożliwiała ruch przez dany port lub zakres portów, które mają być używane przez program iFolder do przesyłania danych.

    Informacje na temat otwierania portu w zaporze można znaleźć w dokumentacji danej zapory.

  2. Zatrzymaj klienta programu iFolder, klikając prawym przyciskiem myszy ikonę programu iFolder , a następnie wybierając polecenie Zakończ.

  3. Zmodyfikuj plik Simias.config, dodając parametr Web Service Port Range do sekcji Service Manager.

  4. Uruchom klienta programu iFolder.

    Więcej informacji można znaleźć w sekcji Uruchamianie klienta programu iFolder.
