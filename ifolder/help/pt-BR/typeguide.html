Diretrizes para tipos e tamanhos de arquivos não sincronizados

Quando o cliente do iFolder é usado, o administrador do iFolder pode especificar restrições para tipo e tamanho máximo de arquivo no nível de sistema, usuário, Grupo LDAP ou iFolder. Sua conta também pode ficar restrita na quantidade de espaço que pode ser usado para dados nas pastas iFolder de sua propriedade. Quando você participa de outras pastas iFolder, o espaço consumido no servidor é contado contra o proprietário daquele iFolder.

Alguns tipos de arquivos não são bons candidatos para sincronização, como arquivos de sistemas operacionais, arquivos ocultos criados por um gerenciador de arquivos ou bancos de dados que são implementados como uma coleção de arquivos vinculados. Você pode incluir apenas os principais tipos de arquivo usados para seu negócio, ou excluir arquivos que provavelmente não se relacionam ao negócio, como arquivos .mp3.

Arquivos de sistema operacional

Você não deve converter arquivos de diretórios de sistema para pastas iFolder. A maioria dos arquivos de sistema muda raras vezes e é melhor manter um arquivo de imagem de seu sistema básico e software chave do que tentar sincronizar aqueles arquivos para o servidor.

Arquivos ocultos

Se o seu sistema de arquivos utiliza arquivos ocultos para rastrear preferências, seu administrador pode restringir esses tipos de arquivos para excluí-los da sincronização em seu sistema. Normalmente, os arquivos ocultos são relevantes somente para os computadores particulares onde eles foram criados e mudam todas as vezes que o arquivo ou diretório é acessado. Você não precisa manter esses arquivos e a sincronização dos mesmos resulta em erros repetidos de conflito de arquivos.

Por exemplo, o iFolder exclui automaticamente dois arquivos ocultos do gerenciador de arquivos chamados thumbs.db e .DS_Store.

Arquivos de banco de dados

O iFolder sincroniza arquivos individuais ou as partes mudadas de arquivos individuais; ele não sincroniza arquivos como um conjunto. Se você tem um arquivo de banco de dados que seja implementado como uma coleção de arquivos vinculados, não tente sincronizá-los em um iFolder.

Não tente sincronizar seus dados do GroupWise transformando os diretórios de arquivo e cache ou os diretórios remotos em pastas iFolder. Se você fizer isso, os arquivos de dados do GroupWise serão corrompidos depois que o arquivo for sincronizado algumas vezes. O GroupWise precisa que os arquivos no arquivo sejam mantidos como um conjunto de arquivos.

Para obter mais informações, consulte Informações legais.
