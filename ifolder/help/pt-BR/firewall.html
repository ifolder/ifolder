Configurando as definições de firewall local para o tráfego do iFolder

Se você usar um firewall local, deverá configurá-lo para que o iFolder possa escolher uma porta dinâmica local ou configurar uma porta estática local para que o cliente do iFolder se comunique localmente (o mesmo computador) com o Mono XSP Server. O iFolder usa serviços Web do Mono XSP para se comunicar via HTTP com os servidores empresariais Novell iFolder em que residem suas contas do iFolder. O administrador do iFolder configura serviços do iFolder que determinam se a comunicação usa HTTP no seu ambiente.

Seu firewall não bloqueia o tráfego entre o Mono XSP e o servidor empresarial Novell iFolder, pois o Mono XSP usa as portas HTTP padrão para o tráfego. No entanto, se você bloquear o tráfego entre o cliente e o Mono XSP, o cliente não poderá se comunicar com o servidor. Você não poderá efetuar login na sua conta do iFolder e sincronizar os dados dele.

Inicialmente, a configuração do iFolder não especifica uma porta particular a ser usada para as comunicações cliente-a-Mono XSP. Quando o iFolder é executado pela primeira vez, ele abre uma porta local disponível para receber as solicitações do Mono XSP e enviar dados ao servidor por esse programa. Ele grava o número da porta em um arquivo de configuração e utiliza essa porta posteriormente, a não ser que ela não esteja disponível quando o iFolder for iniciado. Se a porta estiver sendo usada por outro aplicativo, o iFolder seleciona uma porta diferente disponível, reconfigura a porta no arquivo de configuração e utiliza a nova porta posteriormente. Por exemplo, se o iFolder estiver inativo e outro aplicativo utilizar a porta, o iFolder deverá selecionar e configurar outra porta a ser usada quando você reiniciá-lo.

  • Utilizando uma porta dinâmica local

  • Utilizando uma porta estática local

Utilizando uma porta dinâmica local

Se você usar um firewall local no computador, o firewall deverá detectar o tráfego local entre o iFolder e o servidor Mono XSP sempre que o iFolder entrar em contato com o servidor para sincronizar suas listas de usuários e dados do iFolder. Selecione Permitir para permitir o tráfego do iFolder nesta sessão. Se você permitir o tráfego permanentemente, seu firewall não deverá lembrá-lo novamente por meio de alertas pop-up sempre que o iFolder for iniciado.

Se você bloquear o tráfego do iFolder para o Mono XSP, seu firewall o impedirá de se conectar ao servidor iFolder e não será possível sincronizar seus arquivos. O iFolder não detecta que o tráfego está bloqueado pelo firewall; ele relata apenas que ele não pode se conectar à sua conta no servidor iFolder.

Utilizando uma porta estática local

Se preferir, você pode adicionar o parâmetro de Faixa de Portas do Serviço Web na seção de Gerenciamento de Serviços do arquivo Simias.config para que o Servidor Mono XSP use uma porta estática ou uma faixa de portas para esse tráfego local. Essa porta ou faixa de portas pode ser aberta no firewall para permitir a comunicação do Mono XSP.

O arquivo Simias.config pode estar nas seguintes localizações:

Sistema Operacional

Localização do arquivo Simias.config

Linux

/home/username/.local/share/simias/Simias.config

Windows

C:\Documents and Settings\username\Local Settings\Application Data\simias\Simias.config

Para configurar o seu sistema para usar uma porta específica ou uma faixa de portas:

  1. Configure seu firewall para permitir tráfego para a porta ou faixa de portas que deseja usar para o tráfego do iFolder.

    Consulte a documentação do seu firewall para obter informações sobre como abrir uma porta no firewall.

  2. Pare o cliente do iFolder clicando o botão direito do mouse no ícone do iFolder e selecionando Sair.

  3. Modifique o arquivo Simias.config adicionando o parâmetroFaixa de Portas do Serviço Web (Faixa de Portas do Serviço Web) à seção Service Manager (Gerenciador de Serviços).

    <configuration>

      <section name="ServiceManager">

        <setting name="WebServicePortRange" value="monoxsp_port" />   

      </section>

    </configuration>

    1. Substitua monoxsp_port por um número de porta ou faixa de números de porta:

      • Especifique um valor de um único número de quatro dígitos (como 1234) para utilizar uma única porta estática. Por exemplo:

        <setting name="WebServicePortRange" value="1234" />

      • Especifique uma faixa de portas (como “1234-1236”) para usar qualquer porta disponível na faixa especificada de portas. Por exemplo:

        <setting name="WebServicePortRange" value="1234-1236" />

  4. Inicie o cliente iFolder.

    Para obter mais informações, consulte a Iniciando o cliente iFolder.

Para obter mais informações, consulte Informações legais.
