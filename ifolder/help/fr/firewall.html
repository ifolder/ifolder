Configuration des paramètres du pare-feu local pour le trafic iFolder

Si vous utilisez un pare-feu local, vous devez le configurer de manière à permettre à iFolder de choisir un port dynamique local ou bien configurer un port statique local pour que le client iFolder puisse communiquer localement (même ordinateur) avec le serveur Mono XSP. iFolder utilise les services Web Mono XSP pour communiquer via HTTP avec les serveurs d'entreprise Novell iFolder où vous possédez des comptes iFolder. C'est l'administrateur iFolder qui configure les services iFolder qui déterminent si les communications utilisent le protocole HTTP dans votre environnement.

Votre pare-feu ne bloque pas le trafic entre Mono XSP et le serveur d'entreprise Novell iFolder car Mono XSP utilise les ports HTTP standard pour le trafic. Toutefois, si vous bloquez le trafic entre le client et Mono XSP, le client n'a plus aucun moyen de communication avec le serveur. Il ne vous est alors plus possible de vous connecter à votre compte iFolder et synchroniser les données iFolder.

A l'origine, la configuration iFolder ne spécifie pas de port particulier à utiliser pour les communications client-Mono XSP. À la première exécution de iFolder, celui-ci ouvre un port local disponible pour guetter les requêtes de Mono XSP et envoyer des données au serveur via Mono XSP. Il écrit le numéro de port dans un fichier de configuration et utilise ce port par la suite sauf s'il n'est pas disponible lorsque iFolder est lancé. Si le port est utilisé par une autre application, iFolder sélectionne un autre port disponible, reconfigure le port dans le fichier de configuration et utilise le nouveau port par la suite. Par exemple, si iFolder n'est pas exécuté et si une autre application utilise le port, iFolder doit sélectionner et configurer un autre port à utiliser lors de son redémarrage.

Utilisation d'un port dynamique local

Si vous utilisez un pare-feu local sur votre ordinateur, il doit détecter le trafic local entre iFolder et le serveur Mono XSP lorsque iFolder contacte le serveur pour synchroniser sa liste d'utilisateurs et ses données iFolder. Sélectionnez Autoriser pour autoriser le trafic iFolder pour cette session. Si vous autorisez le trafic de manière permanente, votre pare-feu ne doit pas se souvenir de vous, et vous pouvez éviter de recevoir les alertes contextuelles au démarrage de iFolder.

Si vous bloquez le trafic iFolder vers Mono XSP, votre pare-feu empêche votre connexion au serveur iFolder et vous ne pouvez pas synchroniser vos fichiers. iFolder ne détecte pas que le trafic est bloqué par le pare-feu ; il signale simplement qu'il ne peut pas se connecter à votre compte sur le serveur iFolder.

Utilisation d'un port statique local

Vous pouvez éventuellement ajouter le paramètre Web Service Port Range dans la section Service Manager du fichier Simias.config pour autoriser le serveur Mono XSP à utiliser un port statique ou une plage de ports pour ce trafic local. Il est ensuite possible d'ouvrir ce port ou cette plage de ports dans le pare-feu pour autoriser les communications pour le serveur Mono XSP.

Le fichier Simias.config se trouve aux endroits suivants :

Système d'exploitation

Emplacement du fichier Simias.config

Linux

/home/nom_utilisateur/.local/share/simias/Simias.config

Windows

C:\Documents and Settings\nom_utilisateur\Local Settings\Application Data\simias\Simias.config

Pour configurer le système de sorte qu'il utilise un port ou une plage de ports spécifiques :

  1. Configurez le pare-feu de sorte à autoriser le trafic sur le port ou la plage de ports que vous voulez utiliser pour le trafic iFolder.

    Consultez la documentation de votre pare-feu pour plus d'informations sur la manière d'ouvrir un port dans le pare-feu.

  2. Arrêtez le client iFolder en cliquant avec le bouton droit sur l'icône iFolder, puis en sélectionnant Quitter.

  3. Modifiez le fichier Simias.config en ajoutant le paramètre Web Service Port Range dans la section Service Manager.

    <configuration>

      <section name="ServiceManager">

        <setting name="WebServicePortRange" value="monoxsp_port" />   

      </section>

    </configuration>

    1. Remplacez port_monoxsp par le numéro de port ou la plage de numéros de port correspondants :

      • Spécifiez un numéro de port unique à quatre chiffres (par exemple, 1234) pour utiliser un seul port statique. Par exemple :

        <setting name="WebServicePortRange" value="1234" />

      • Spécifiez une plage de ports (par exemple, « 1234-1236 ») pour utiliser n'importe quel port disponible dans la plage spécifiée. Par exemple :

        <setting name="WebServicePortRange" value="1234-1236" />

  4. Démarrez le client iFolder.

    Pour plus d'informations, reportez-vous au Démarrage du client iFolder.

Pour plus d'informations, reportez-vous à la rubrique Mentions légales.
