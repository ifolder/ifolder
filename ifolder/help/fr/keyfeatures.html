Fonctionnalités clés de iFolder

Avant de commencer à utiliser iFolder, il est important de comprendre les fonctions clés suivantes :

  • Partage iFolder

  • Droits d'accès iFolder

  • Synchronisation de fichiers

  • Migration

  • le codage

  • Mise à niveau

  • Fusionner

  • Dossier iFolder par défaut

  • Amélioration de Web Access

  • Fichier journal de synchronisation

Partage iFolder

En principe, lorsque vous travaillez à divers emplacements ou avec d'autres personnes, vous devez gérer les différentes versions des fichiers avec beaucoup de précaution. Avec iFolder, la toute dernière version de vos fichiers peut vous suivre sur tous les ordinateurs sur lesquels vous avez installé le client iFolder et créé un dossier iFolder. iFolder vous permet également de partager plusieurs dossiers iFolder et leur contenu avec d'autres utilisateurs autorisés ou groupes LDAP. Pour plus d'informations sur les dossiers iFolder partagés, reportez-vous à la rubrique Dossiers iFolder partagés.

Le client iFolder prend en charge le partage en synchronisant des fichiers sur plusieurs ordinateurs via un serveur central. Il vous permet d'effectuer les opérations suivantes :

  • partager des fichiers entre plusieurs ordinateurs ;

  • partager des fichiers avec d'autres utilisateurs ;

  • posséder plusieurs dossiers iFolder ;

  • participer à plusieurs dossiers iFolder que d'autres utilisateurs partagent avec vous ;

  • participer à plusieurs comptes en tant que propriétaire ou membre de dossiers iFolder.

Vous manipulez les dossiers iFolder directement dans le gestionnaire de fichiers ou dans le navigateur iFolder qui fait partie intégrante du client. Au sein de iFolder, vous pouvez configurer une structure de sous-répertoires adaptée à vos habitudes professionnelles ou personnelles. La structure de sous-répertoires est constante pour tous les dossiers iFolders membres.

Dossiers iFolder partagés

Un dossier iFolder est un répertoire local qui partage des fichiers et les synchronise de manière sélective via un serveur central avec des utilisateurs spécifiés par l'utilisateur. Les fichiers iFolder sont accessibles par tous les membres iFolder et peuvent être modifiés par ceux qui disposent des droits appropriés. Vous pouvez accéder à vos dossiers iFolder sur plusieurs postes de travail et les partager avec des utilisateurs individuels et des groupes LDAP.

Vous choisissez qui participe à chaque iFolder et vous déterminez le niveau d'accès de chaque membre. De même, vous pouvez participer aux dossiers iFolder partagés qui appartiennent à d'autres utilisateurs de votre environnement de collaboration. Lorsque vous partagez un dossier iFolder, les utilisateurs membres disposant du droit Écrire peuvent modifier la structure du répertoire, modifications qui s'appliquent à toutes les copies de iFolder. Chaque utilisateur peut placer le dossier iFolder partagé à n'importe quel emplacement de son ordinateur s'il satisfait aux exigences de la section Directives sur l'emplacement et l'utilisation des dossiers iFolder.

Droits d'accès iFolder

Le client iFolder prend en charge le propriétaire et trois niveaux d'accès pour les membres d'un dossier iFolder :

IMPORTANT :Les membres d'un groupe LDAP héritent des droits d'accès définis pour ce groupe.

  • Propriétaire: Un seul utilisateur peut être propriétaire d'un dossier iFolder. Il s'agit, en principe, de celui qui crée le dossier iFolder. Le propriétaire peut utiliser le client iFolder pour transférer la propriété à un autre membre du dossier iFolder.

    Le propriétaire d'un iFolder dispose d'un droit de contrôle total sur le dossier. Il possède des droits de lecture et d'écriture dans le dossier, gère les membres et leurs droits d'accès et peut supprimer le droit de contrôle total d'un membre.

    Si un groupe LDAP possède des dossiers iFolder, tous les utilisateurs membres de ce groupe se voient accorder le contrôle total.

  • Contrôle total: Membre du dossier iFolder partagé avec le droit de contrôle total. Ce membre possède des droits de lecture et d'écriture dans le dossier iFolder et gère les membres et les droits d'accès de tous les utilisateurs sauf le propriétaire.

  • Lecture-écriture: Membre du dossier iFolder partagé disposant du droit Lecture/écriture sur les répertoires et fichiers du dossier iFolder. Ce membre peut modifier un fichier, mais les changements locaux sont synchronisés avec les copies du dossier iFolder des autres membres.

  • Lecture seule: Membre du dossier iFolder partagé disposant du droit Lecture seule sur les répertoires et fichiers du dossier iFolder. Ce membre peut modifier un fichier, mais les changements locaux ne sont pas synchronisés avec les copies du dossier iFolder des autres membres.

Tout utilisateur iFolder ou groupe LDAP peut posséder des dossiers iFolder et être membre d'autres dossiers. En tant que propriétaire, vous disposez systématiquement du droit de contrôle total. En tant que membre, votre niveau d'accès à chaque dossier iFolder partagé peut varier selon les droits qui vous ont été octroyés par un membre disposant du contrôle total.

Synchronisation de fichiers

Lorsque vous êtes connecté au serveur, iFolder peut synchroniser des membres et des fichiers iFolder en arrière-plan pendant que vous travaillez. Lorsque vous configurez un compte iFolder, vous pouvez activer l'option Mémoriser mon mot de passe pour que iFolder mémorise le mot de passe sur cette machine et, au prochain démarrage, se connecte automatiquement au serveur et synchronise les données sans demander le mot de passe.

Vous pouvez configurer le client iFolder pour qu'il s'exécute automatiquement chaque fois que vous vous loguez à l'environnement de bureau de votre ordinateur. La session est exécutée en arrière-plan pendant que vous travaillez dans les fichiers de vos dossiers iFolder locaux et tous vos changements sont relevés et consignés.

Vous pouvez synchroniser les fichiers manuellement ou à des intervalles spécifiques.

iFolder dispose également de fonctionnalités de synchronisation delta. Lorsque vous modifiez un fichier iFolder, seuls les deltas ou les changements sont synchronisés avec le serveur. En d'autres termes, la synchronisation delta récupère uniquement les mises à jour depuis la dernière synchronisation entre le client iFolder et le serveur. L'efficacité est accrue puisque la durée de synchronisation est réduite.

Migration

Novell iFolder permet une migration simple et transparente de l'interface utilisateur côté client de iFolder2.x vers iFolder 3.6 et version ultérieure. L'assistant de migration fourni dans le client iFolder vous aide à migrer rapidement et aisément des données iFolder 2.x existantes vers iFolder 3.6 et versions ultérieures. Pour plus d'informations, reportez-vous au Migration et mise à niveau de Novell iFolder.

le codage

iFolder offre un niveau de sécurité accru pour vos fichiers iFolder confidentiels afin de les protéger d'un accès intentionnel ou accidentel de personnes non autorisées. Auparavant, la sécurité des données n'était pas garantie côté serveur. Dès lors, les données étaient accessibles par l'administrateur ou toute personne non autorisée qui parvenait à accéder au serveur. Par ailleurs, lorsque vous accédez à des fichiers iFolder via un fournisseur de services Internet, il se peut qu'une personne non autorisée trouve vos fichiers confidentiels par inadvertance. Désormais, vous pouvez coder et enregistrer vos fichiers sur le serveur, puis les récupérer grâce à une phrase secrète que vous seul connaissez. En cas d'oubli de votre phrase secrète, l'agent de récupération que vous avez sélectionné lors de la création du dossier iFolder codé vous aide à récupérer vos données.

IMPORTANT :Les dossiers iFolder codés ne peuvent pas être partagés.

Pour plus d'informations sur le codage, reportez-vous à la section Gestion des phrases secrètes pour les dossiers iFolder codés.

Mise à niveau

Novell iFolder permet une mise à niveau simple à partir de iFolder 3.x vers le client iFolder pour Novell iFolder 3.6 et versions ultérieures. Pour plus d'informations, reportez-vous au Mise à jour des clients iFolder 3.x.

Fusionner

La fusion permet de consolider les données iFolder et de minimiser le transfert de données entre le client iFolder et le serveur. La fusion offre également la possibilité de déplacer un dossier existant sur la machine cliente. Pour plus d'informations, reportez-vous au Fusion de dossiers iFolder.

Dossier iFolder par défaut

iFolder vous permet de paramétrer un dossier iFolder par défaut lorsque vous configurez un nouveau compte. Chaque fois que vous configurez un nouveau compte iFolder, iFolder vérifie automatiquement la présence du dossier iFolder par défaut. En l'absence d'un dossier iFolder par défaut, iFolder vous invite à en créer un. S'il existe déjà, le client iFolder vous invite à télécharger. Vous êtes également autorisé à ignorer ces deux options. Pour plus d'informations, reportez-vous à l'Étape 7de la section Configuration d'un compte iFolder.

Amélioration de Web Access

iFolder propose une interface Web améliorée qui vous permet d'accéder à vos fichiers iFolder sur des serveurs distants. Grâce à la console Web Access, vous pouvez utiliser un navigateur Web pour accéder à vos fichiers sur le serveur iFolder à partir de n'importe quel poste de travail non équipé du client iFolder. L'interface Web Access vous permet d'effectuer toutes les opérations du client iFolder.

Par rapport aux versions précédentes, iFolder 3.6 et versions suivantes vous permet de créer, parcourir, partager, télécharger et supprimer des dossiers iFolder via la console Web Access améliorée. Des utilisateurs mobiles peuvent également utiliser cette fonctionnalité via le Web.

Fichier journal de synchronisation

Le journal de synchronisation affiche un journal de vos opérations d'arrière-plan iFolder, telles que le téléchargement de fichiers pour différents dossiers iFolder, de violations de stratégie, de fichiers exclus et de fichiers non synchronisés.

Pour plus d'informations, reportez-vous à la rubrique Mentions légales.
