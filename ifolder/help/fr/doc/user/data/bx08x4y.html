Directives pour la localisation des dossiers iFolder

En principe, vous pouvez localiser les dossiers iFolder sur votre disque dur lorsque vous créez un dossier iFolder ou que vous en configurez un partagé par vous. Toutefois, certaines pratiques et limitations sont imposées. Suivez les directives ci-après lorsque vous localisez des dossiers iFolder sur l'ordinateur.

  • Les dossiers iFolder doivent avoir un nom unique dans le répertoire dans lequel ils se trouvent.
  • Un nom de dossier iFolder ne peut pas contenir les caractères suivants : 
    \/:*?"<>|;

    Pour d'autres conventions de dénomination, reportez-vous à Conventions de dénomination des fichiers et des dossiers d'un compte iFolder.

  • Le dossier iFolder doit résider sur le disque dur local ; il ne peut pas figurer sur un lecteur de réseau ou un lecteur non physique.
  • Un dossier iFolder ne peut pas être créé à la racine du lecteur.
  • Un dossier iFolder ne peut être créé qu'à partir d'un dossier existant.
  • Vous ne pouvez créer un dossier iFolder qu'à partir d'un dossier dans lequel vous disposez de droits d'accès en lecture et en écriture dans le système de fichiers. Lorsque vous configurez un dossier iFolder partagé sur un ordinateur, vous devez disposer de droits d'accès au dossier dans lequel vous placez ce dossier iFolder.
  • Un dossier iFolder ne peut pas contenir un dossier système comme le dossier Windows ou un lecteur système Linux ou Mac, ni résider dans un tel système ou lecteur.
  • Un dossier iFolder ne peut pas contenir un dossier d'applications comme le dossier Program Files, ni résider dans un tel dossier.
  • Un dossier iFolder ne peut pas contenir un dossier de métadonnées iFolder, ni résider dans un tel dossier. L'emplacement par défaut du dossier de données est :

    • Linux : Dans votre dossier privé du sous-répertoire .local\share.

    • Windows : Dans le répertoire C:\Documents and Settings\nom_utilisateur_session_Windows\Local Settings\Application Data\simias.

    • Macintosh : Dans votre dossier privé du sous-répertoire .local\share.

  • Un dossier iFolder ne peut pas contenir un autre dossier iFolder, y compris votre dossier Novell® iFolder® 2.1x, ni résider dans un tel dossier.
