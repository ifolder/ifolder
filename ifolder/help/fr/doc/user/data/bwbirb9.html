Configuration des paramètres du scanner de virus local pour le trafic iFolder

Si vous avez installé un scanner de virus sur votre ordinateur, excluez le répertoire ..\simias\WorkArea\ de la détection de virus. Par exemple, le répertoire ..\simias\WorkArea\ se situe dans c:\documents and settings\nom-utilisateur\local settings\application data\simias\WorkArea\.simias.wf.guid sous Windows.

Le répertoire ..\simias\WorkArea\ correspond à l'emplacement où iFolder stocke les fichiers téléchargés du serveur. Si un fichier iFolder contient un virus et que le scanner de virus le détecte lorsque le fichier se trouve dans le répertoire ..\simias\WorkArea, le scanner affiche un message indiquant qu'un virus a été trouvé, met ce fichier en quarantaine, puis iFolder essaie à nouveau de synchroniser le fichier.

Si le répertoire WorkArea n'est pas analysé par le scanner de virus, la synchronisation est effectuée et le fichier contaminé est téléchargé et placé dans le dossier iFolder local. Le scanner de virus détecte alors le fichier contaminé dans le dossier iFolder et le met en quarantaine pour empêcher la contamination de l'ordinateur. iFolder détecte la suppression du fichier et lorsque la prochaine synchronisation commence, il supprime celui-ci du serveur et des ordinateurs membres.

Toutefois, si le dossier iFolder dans lequel le fichier est téléchargé est en Lecture seule, la suppression locale n'est pas implémentée sur le serveur ni sur d'autres ordinateurs membres. Le virus doit être détecté à un autre emplacement où les autorisations permettent la synchronisation des changements. Le scanner de virus de l'utilisateur avertit sans doute l'utilisateur de la présence du virus pour que celui-ci le signale aux autres membres iFolder.
