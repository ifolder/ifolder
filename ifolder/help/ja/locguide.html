iFolderの場所と利用法のガイドライン

一般に、iFolderは、iFolderを作成するか、共有するiFolderを設定したときに、ローカルハードドライブの任意の場所に配置できます。 ただし、実際にはいくつかの制約があります。 コンピュータ上にiFolderを配置して使用する際には、次のガイドラインを考慮してください。

  • iFolderの名前は、iFolderを配置するディレクトリ内で一意である必要があります。

  • iFolderの名前に次の無効な文字を含めることはできません。

    \/:*?"<>|;

    その他の命名規則については、「iFolderとそのフォルダおよびファイルの命名規則」を参照してください。

  • iFolderは、ローカルハードドライブに存在する必要があります。ネットワークドライブまたは非物理ドライブには配置できません。

  • iFolderは、ハードドライブのルートには配置できません。

  • iFolderは、ファイルの読み込みと書き込みを行うファイルシステムアクセス権があるフォルダからのみ作成できます。 コンピュータに共有iFolderを設定する場合、iFolderを配置するフォルダに対するファイルシステムアクセス権が必要です。

  • iFolderは、Program Filesフォルダのようなアプリケーションフォルダを含んだり、そこに配置したりすることはできません。

  • iFolderは、メタデータフォルダを含んだり、そこに配置したりすることはできません。 メタデータフォルダのデフォルトの場所は次のとおりです。

    • Linux: <ホームディレクトリ>\.local\share\simiasサブディレクトリのホームフォルダ内

    • Windows: C:\Documents and Settings\<Windowsログオンユーザ名>\Local Settings\Application Data\simiasディレクトリ内

    • Macintosh: $HOME/.local/share/simias内

  • iFolderは、お使いのNovell iFolder 2.xなどの他のiFolderを含めたり、そこに配置したりすることはできません。

    iFolderでは、iFolderをネットワークフォルダとして使用することはできません。たとえば、iFolderを、ネットワークベースの/homeディレクトリとしてユーザのワークステーションにマップすることはできません。iFolderは、デスクトップからiFolderサーバのユーザデータボリュームにあるiFolderコレクションへのWebDAVによるアクセスもサポートしていません。

  • 一度に複数のiFolderアカウントでログインできますが、1つのiFolderには1つのiFolderアカウントしか関連付けることができません。複数のアカウントで1つのiFolderを同期させることはできません。iFolderを作成すると、そのiFolderは同じiFolderサーバ/ドメイン内でプロビジョニングされている他のユーザおよびLDAPGroupとのみ共有できます。

  • iFolderがLDAPGroupと共有されている場合、そのLDAPGroupに属しているすべてのメンバーがiFolderを使用できます。

  • 複数のユーザが1台のコンピュータを使用する際に、ファイルシステムがプライバシー機能をサポートしており、これを使用するようにシステムを設定してある場合、iFolderのローカルコピーはディスク上の自分専用のストレージ領域に維持されます。

    Windowsの場合、NTFSにファイルやフォルダのアクセス権や、NTFSボリューム上でファイルを暗号化して保存する技術であるEncrypting File System (EFS)などのセキュリティ機能が組み込まれています。このようなセキュリティ機能はFAT32ファイルシステムではサポートされません。管理者のIDを除き、ユーザのローカルログインIDに、管理者権限や同等の権限を付与しないようにしてください。管理者ユーザには、コンピュータ上のすべてのファイルやディレクトリへのアクセス権があります。共有のコンピュータで、ユーザごとにプライバシーとセキュリティを維持する必要がある場合は、iFolderのローカルコピーをEFSまたはサードパーティのファイルシステム暗号化機能を使用して暗号化します。

    LinuxおよびMacintoshのファイルシステムは、複数のユーザ環境を考慮して設計されています。ファイルのアクセス権により、ユーザ、グループおよびその他のアクセスを制御できます。ユーザの個人的なファイルには、通常はそのユーザに対してのみ、すべてのアクセス権が設定され、他のグループやその他には設定されません。

    iFolderをLinux上のFAT32ボリュームにローカル保存する場合、FAT32ファイルシステムはPOSIXとの互換性がないため、特別な対処が必要です。詳細については、FAT32マウントポイント(Linux)上にiFolderを作成するを参照してください。

  • 同じコンピュータのユーザがiFolderを共有する場合、デバイス上に複数のiFolderのコピーを保存できます。その場合、各ユーザがサーバからダウンロードしたコピーが維持されます。別のローカルログインIDを使用して、共有iFolderを1つの場所で共有することはできません。

詳細については、「 保証と著作権.
