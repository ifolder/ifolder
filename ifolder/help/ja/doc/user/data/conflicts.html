ファイルの競合の解決

ファイルを同時に保存したときにファイルバージョンに競合が発生するか、またはファイル名が大文字と小文字だけで異なっている場合、「名前の競合」というフラグが付けられ、競合エリアに保存されます。また、ファイルにフラグを付けるためのアラートがiFolderクライアントを介して送信されます。


バージョンの競合

バージョンの競合は、他のメンバーと調整してから競合を解決する必要があります。その後で、［Conflict Resolution (競合の解決)］ダイアログで正しいバージョンを選択します。

  1. 競合が発生すると、通知スペースにポップアップメッセージが表示されます。メッセージを閉じるには［X］をクリックします。

    このポップアップ表示は、［iFolderの初期設定］の［一般］タブで設定することで無効にできます。

  2. ［通知］エリアで、iFolderアイコンを右クリックしてから、［iFolder］をクリックしてiFolderブラウザを開きます。

  3. 競合が発生しているファイルを選択し、［解決］をクリックして［競合の解決］ダイアログボックスを開きます。

  4. ローカルコンピュータ上とサーバ上のファイル情報を表示し、どのファイルバージョンを維持するかを決定し、［Save This Version (このバージョンを保存)］をクリックして正しいファイルのバージョンを維持します。

  5. 競合を解決したら、［OK］をクリックします。


ファイル名の競合

ファイル名の競合は、ローカルファイルのバージョンの名前とサーバファイルのバージョンの名前の大文字小文字が異なる場合に発生します。コンテンツが異なるか、または同じである可能性があります。このタイプの競合は、ファイルをリネームすることによって解決できますが、リネームするファイルの保存場所に注意してください。

  • ファイルのコンテンツが同じ場合、［競合の解決］ダイアログボックスで2つのファイルのうちの1つをリネームします。
  • ファイルのコンテンツが異なる場合、ローカルのファイルをコピーして、別の名前で保存するようにしてください。その後、［競合の解決］ダイアログボックスに戻り、サーババージョンのファイルを保存します。これにより、ファイルの両方のバージョンを別の名前でiFolder内に保存することができます。［競合の解決］ダイアログボックスでファイルのバージョンをリネームすると、そのファイルのバージョンによって他方のファイルが上書きされてしまうため、上書きされた方のファイルのコンテンツは失われます。

  1. 競合が発生すると、［通知］エリアにポップアップメッセージが表示されます。メッセージを閉じるには［X］をクリックします。

  2. ［通知］エリアで、iFolderアイコンを右クリックしてから、［iFolder］をクリックしてiFolderブラウザを開きます。

  3. 競合が発生しているファイルを選択し、［解決］をクリックして［競合の解決］ダイアログボックスを開きます。

  4. それがファイル名の競合であることを確認し、ファイルのコンテンツが同じであるかどうかを判断します。

  5. 次のいずれかの操作を行います。

    • ファイルのコンテンツが同じである場合、一方のファイルをリネームし、［Save This Version (このバージョンを保存)］をクリックします。その後、［OK］をクリックして［競合の解決］ダイアログを閉じます。これで終了です。
    • ファイルのコンテンツが異なる場合は、手順6に進みます。

  6. ［競合の解決］ダイアログボックスを閉じます。

  7. ファイルマネージャを開いて、競合が発生しているファイルのローカルコピーに移動します。

  8. そのファイルをコピーし、一意の名前でiFolder内に保存します。

    iFolderにより、この新しいファイルがサーバとメンバーのコンピュータに同期されます。

  9. ［競合の解決］ダイアログボックスに戻り、サーババージョンのファイルを選択し、［Save This Version (このバージョンを保存)］をクリックしてから［OK］をクリックします。

    サーババージョンのファイルがローカルコンピュータ上にダウンロードされ、そのサーババージョンの名前でローカルコピーが上書きされます。
