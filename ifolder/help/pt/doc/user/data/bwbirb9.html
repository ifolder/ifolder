Configurando as definições do programa de varredura de vírus local para o tráfego do iFolder

Se você usa um software de varredura de vírus no computador, exclua o diretório ..\simias\WorkArea\ da varredura de vírus. Por exemplo, a localização do diretório ..\simias\WorkArea\ no Windows é c:\documents and settings\username\local settings\application data\simias\WorkArea\.simias.wf.guid.

O diretório ..\simias\WorkArea\ está onde o iFolder arranja arquivos para download do servidor. Se um arquivo iFolder tiver um vírus anexado e o software de varredura de vírus varrer tal vírus enquanto o arquivo estiver no diretório ..\simias\WorkArea, o programa de varredura de vírus emitirá uma mensagem pop-up informando que um vírus foi encontrado, moverá o arquivo para quarentena e, então, o iFolder tentará sincronizar o arquivo novamente.

A não varredura da WorkArea (Área de Trabalho) permite que a sincronização seja concluída; o arquivo infectado tem um download bem-sucedido e é movido para o iFolder local. O programa de varredura de vírus detecta o arquivo infectado no iFolder e o move desta pasta para a quarentena, protegendo o computador da infecção. O iFolder detecta que o arquivo foi apagado e quando a próxima sincronização começar, o iFolder removerá o arquivo do servidor e dos computadores dos membros.

Porém, se o iFolder para onde é realizado o download de um arquivo for Apenas leitura, a exclusão local não será forçada para o servidor e para os computadores dos outros membros. O vírus deve ser detectado nas outras partes onde as permissões possibilitam que as mudanças sejam sincronizadas. O programa de varredura de vírus do usuário provavelmente o alertará sobre o vírus, então o usuário deve alertar os outros membros da pasta iFolder sobre o vírus.
