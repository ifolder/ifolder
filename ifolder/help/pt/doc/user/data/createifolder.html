Criando uma pasta iFolder

Você pode criar uma pasta iFolder convertendo uma pasta existente que satisfaça as seguintes diretrizes:

  • Diretrizes para localização de pastas iFolder
  • Convenções de nomeação para um iFolder e suas pastas e arquivos

Se não existir uma pasta, você poderá procurar uma localização e criar a nova pasta como parte do processo.

Dentro da pasta iFolder, é possível configurar qualquer estrutura de subdiretório que se adapte aos seus hábitos de trabalho pessoais ou profissionais. A estrutura de subdiretórios é constante em todos os membros das pastas iFolder. Cada estação de trabalho pode especificar um diretório pai diferente para a pasta iFolder compartilhada.

Você pode criar pastas iFolder diretamente em seu gerenciador de arquivos e no browser de pastas iFolder, que está disponível no aplicativo cliente. Use os seguintes métodos para criar uma pasta iFolder:

  • Criando pastas iFolder em um gerenciador de arquivos
  • Criando pastas iFolder no browser de pastas iFolder
  • Criando pastas iFolder em um Ponto de Montagem FAT32 (Linux)


Criando pastas iFolder em um gerenciador de arquivos

  1. Use o gerenciador de arquivos para navegar até a pasta que deseja converter em uma pasta iFolder e, em seguida, selecione a pasta.

  2. Use um dos seguintes métodos:

    • Clique o botão direito do mouse no ícone da pasta e, em seguida, clique em Converter em um iFolder.
    • Na barra de ferramentas, clique em Arquivo > Converter em um iFolder.

    O ícone da pasta muda para uma pasta com uma sobreposição do ícone do iFolder.

  3. Se a mensagem iFolder Criado for mostrada, execute um dos seguintes procedimentos:

    • Clique em Fechar para ignorar a mensagem.
    • Selecione Não Mostrar Esta Mensagem Novamente para desabilitar notificações futuras e, em seguida, clique em Fechar para ignorar a mensagem.

      Se posteriormente decidir que deseja receber mensagens de confirmação, vá para Preferências do iFolder, selecione Mostrar caixa de diálogo de confirmação ao criar pastas iFolder e, em seguida, clique em OK.


Criando pastas iFolder no browser de pastas iFolder

  1. Na área de Notificação, clique o botão direito do mouse no ícone do iFolder e, em seguida, clique em Pastas iFolder para abrir o browser de pastas iFolder.

  2. Clique em Novo.

  3. Especifique a conta do iFolder que deseja usar para esta pasta iFolder.

    Se você tiver várias contas do iFolder, a conta padrão será listada primeiro. Use a lista suspensa para selecionar uma conta diferente.

  4. No browser, navegue até o diretório onde deseja criar uma pasta iFolder.

  5. Selecione a pasta normal que deseja transformar em uma pasta iFolder, utilizando um dos seguintes métodos:

    • Se a pasta normal existir, selecione a pasta e clique em OK
    • Se a pasta normal não existir, clique em Criar Nova Pasta para criá-la, selecione a pasta recém-criada e clique em OK.

  6. Clique em OK.

    A pasta iFolder aparece na lista de pastas iFolder.

  7. Se a mensagem iFolder Criado for mostrada, execute um dos seguintes procedimentos:

    • Clique em Fechar para ignorar a mensagem.
    • Selecione Não Mostrar Esta Mensagem Novamente para desabilitar notificações futuras e, em seguida, clique em Fechar para ignorar a mensagem.

      Se posteriormente decidir que deseja receber mensagens de confirmação, vá para Preferências do iFolder, selecione Mostrar caixa de diálogo de confirmação ao criar pastas iFolder e, em seguida, clique em OK.


Criando pastas iFolder em um Ponto de Montagem FAT32 (Linux)

No Linux, o iFolder suporta a criação e a utilização de pastas iFolder em um ponto de montagem para um sistema de arquivos FAT32. Para que você possa criar novas pastas iFolder ou configurar pastas iFolder compartilhadas em um volume FAT32 no seu computador Linux, o usuário do iFolder deverá ter um ponto de montagem FAT32 onde você deseja armazenar as pastas iFolder, e o arquivo /etc/fstab deverá identificar explicitamente o ID de usuário local do usuário do iFolder para o ponto de montagem.

IMPORTANTE:  Isso é necessário porque o FAT32 não suporta permissões de arquivo. Qualquer outro sistema de arquivos que não suporte permissões pode apresentar o mesmo problema.

Para sistemas com inicialização dupla, você pode acessar os dados em pastas iFolder como pastas normais do sistema onde elas não estiverem configuradas como pastas iFolder. Se você configurar a pasta como uma pasta iFolder em ambas as plataformas, o conjunto de dados será armazenado em duas pastas iFolder diferentes no servidor, o que consumirá o dobro dos recursos de disco do servidor alocado.

Como o sistema de arquivos FAT32 não suporta permissões do sistema de arquivos POSIX, verifique se o usuário do iFolder é o proprietário local do ponto de montagem. Modifique o arquivo /etc/fstab para definir o parâmetro do ID de usuário (UID) da identidade de login local do usuário para o usuário do iFolder que criar pastas iFolder naquele ponto de montagem.

  1. Efetue o login no computador como usuário Root.

  2. Crie o diretório onde deseja montar o sistema de arquivos FAT32. No prompt de comando, digite

    mkdir /home/nomedeusuario/fsmount

    Substitua nomedeusuario pelo nome do usuário que pretende usar a unidade FAT32 para as pastas iFolder. Substitua fsmount pelo nome do diretório que você deseja usar como o ponto de montagem. Por exemplo, se o nome de usuário for jsmith e o nome do diretório for fat32mntpt para uma unidade ou partição FAT32, digite

    mkdir /home/jsmith/fat32mntpt

  3. Monte a unidade FAT32 no ponto de montagem desejado. No prompt de comando, digite

    mount -t vfat /dev/hda1 /home/nomedeusuario/fsmount

    Substitua /dev/hda1 pelo nome do dispositivo ou da partição do dispositivo não Linux. Substitua nomedeusuario pelo nome de usuário do usuário do iFolder. Substitua fsmount pelo nome do diretório que você deseja usar como o ponto de montagem.

    Continuando o exemplo, onde o sistema de arquivos FAT32 (vfat) estiver no dispositivo hda4 (/dev/hda4), o nome de usuário for jsmith e o diretório do ponto de montagem for /home/jsmith/fat32mntpt, digite

    mount -t vfat /dev/hda4 /home/jsmith/fat32mntpt

  4. Edite o arquivo /etc/fstab adicionando o parâmetro uid=nomedeusuario do usuário do iFolder na linha que define o ponto de montagem FAT32.

    /dev/hda1 /home/nomedeusuario/fsmount vfat uid=nomedeusuario

    Continuando com o exemplo, modifique o arquivo /etc/fstab adicionando o uid=jsmith ao ponto de montagem:

    /dev/hda4  /home/jsmith/fat32mntpt  vfat  uid=jsmith
