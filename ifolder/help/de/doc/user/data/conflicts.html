Lösen von Dateikonflikten

Wenn Konflikte zwischen gleichzeitig gespeicherten Dateiversionen auftreten oder wenn Dateinamen nur durch Groß-/Kleinschreibung voneinander abweichen, wird die Datei als Namenskonflikt gekennzeichnet und im Konfliktbereich gespeichert. Über den iFolder-Client wird eine Warnmeldung zur Kennzeichnung der Datei gesendet.


Versionskonflikte

Versionskonflikte müssen mit anderen Mitgliedern koordiniert werden, bevor der Konflikt gelöst wird. Wählen Sie dann die richtige Version im Dialogfeld "Konfliktlösung" aus.

  1. Wenn ein Konflikt auftritt, wird eine Popup-Meldung im Benachrichtigungsbereich angezeigt. Klicken Sie auf das "X", um die Meldung auszublenden.

    Sie können das Popup-Verhalten deaktivieren, indem Sie die Einstellung unter "iFolder-Einstellungen" > Registerkarte "Allgemein" festlegen.

  2. Klicken Sie im Benachrichtigungsbereich mit der rechten Maustaste auf das iFolder-Symbol () und klicken Sie dann auf "iFolder-Ordner", um den Browser für iFolder-Ordner zu öffnen.

  3. Wählen Sie die Datei aus, bei der der Konflikt besteht, und klicken Sie dann auf "Auflösen" (), um das Dialogfeld "Konflikte lösen" anzuzeigen.

  4. Lassen Sie die Informationen für die Datei auf Ihrem lokalen Computer und die Datei auf dem Server anzeigen und bestimmen Sie, welche Version der Datei Sie beibehalten möchten. Klicken Sie anschließend auf "Diese Version speichern", um die richtige Dateiversion weiterhin zu verwenden.

  5. Wenn Sie mit dem Lösen des Konflikts fertig sind, klicken Sie auf "OK".


Dateinamenskonflikte

Dateinamenskonflikte treten auf, wenn die Namen der lokalen Dateiversion und der Dateiversion auf dem Server sich durch die Groß- und Kleinschreibung unterscheiden. Der Inhalt kann unterschiedlich oder identisch sein. Diese Arten von Konflikten können durch Umbenennen der Dateien gelöst werden, Sie sollten beim Umbenennen von Dateien allerdings Vorsicht walten lassen.

  • Wenn der Inhalt identisch ist, können Sie eine der Dateien im Dialogfeld "Konflikte lösen" umbenennen.
  • Wenn der Inhalt unterschiedlich ist, müssen Sie eine Kopie der lokalen Datei erstellen und ihr einen anderen Namen geben, bevor Sie zum Dialogfeld "Konflikte lösen" zurückkehren und die Serverversion der Datei speichern. Dadurch können beide Versionen der Datei mit verschiedenen Dateinamen im iFolder-Ordner gespeichert werden. Wenn Sie eine Datei im Dialogfeld "Konflikte lösen" umbenennen, überschreibt diese Dateiversion die andere und der Inhalt der anderen Datei wird gelöscht.

  1. Wenn ein Konflikt auftritt, wird eine Popup-Meldung im Benachrichtigungsbereich angezeigt. Klicken Sie auf das "X", um die Meldung auszublenden.

  2. Klicken Sie im Benachrichtigungsbereich mit der rechten Maustaste auf das iFolder-Symbol () und klicken Sie dann auf "iFolder-Ordner", um den Browser für iFolder-Ordner zu öffnen.

  3. Wählen Sie die Datei aus, bei der der Konflikt besteht, und klicken Sie dann auf "Auflösen" (), um das Dialogfeld "Konflikte lösen" anzuzeigen.

  4. Bestätigen Sie, dass es sich um einen Dateinamenskonflikt handelt, und ermitteln Sie, ob der Inhalt identisch ist oder nicht.

  5. Führen Sie einen der folgenden Schritte aus:

    • Wenn der Inhalt identisch ist, benennen Sie eine der Dateien um, klicken Sie dann auf "Diese Version speichern" und klicken Sie schließlich auf "OK", um das Dialogfeld "Konflikte lösen" zu schließen. Der Vorgang ist abgeschlossen.
    • Wenn der Inhalt unterschiedlich ist, fahren Sie fort mit Schritt 6.

  6. Schließen Sie das Dialogfeld "Konflikte lösen".

  7. Öffnen Sie einen Datei-Manager und wechseln Sie zur lokalen Kopie der Konfliktdatei.

  8. Kopieren Sie die Datei und speichern Sie sie mit einem eindeutigen Namen im iFolder-Ordner.

    iFolder synchronisiert die Datei als neue Datei auf dem Server und den Mitgliedercomputern.

  9. Kehren Sie zum Dialogfeld "Konflikte lösen" zurück, wählen Sie die Serverversion der Datei aus, klicken Sie auf "Diese Version speichern" und klicken Sie dann auf "OK".

    Die Serverversion der Datei wird auf Ihren lokalen Computer heruntergeladen und überschreibt die lokale Kopie der Datei mit diesem Namen.
