Wichtigste Funktionen von iFolder

Vor der Arbeit mit iFolder sollten Sie sich unbedingt mit folgenden wichtigen Funktionen vertraut machen:

iFolder-Freigabe

Wenn Sie an unterschiedlichen Orten tätig sind oder mit anderen zusammenarbeiten, ist die gewissenhafte Verwaltung von Dateiversionen unerlässlich. Mit iFolder können Sie die aktuellen Versionen Ihrer Dateien auf alle Computer mitnehmen, auf denen Sie den iFolder-Client installiert und einen iFolder-Ordner für diese Dateien erstellt haben. Mit iFolder können Sie außerdem mehrere iFolder-Ordner und die jeweiligen Inhalte für andere autorisierte Benutzer und LDAP-Gruppen freigeben. Weitere Informationen zu freigegebenen iFolder-Ordnern finden Sie unter Freigegebene iFolder-Ordner.

Der iFolder-Client unterstützt die Freigabe, indem von einem zentralen Server aus Dateien auf mehreren Computern synchronisiert werden. Außerdem können Sie:

  • Dateien für mehrere Computer freigeben

  • Dateien für andere Benutzer freigeben

  • Eigentümer mehrerer iFolder-Ordner sein

  • Mehrere iFolder-Ordner nutzen, die andere Benutzer für Sie freigeben

  • Als Eigentümer oder Mitglied iFolder-Ordner mehrerer Konten nutzen

Sie arbeiten mit iFolder-Ordnern direkt in Ihrem Dateimanager oder im iFolder-Browser, der Teil des Client ist. Innerhalb des iFolder-Ordners können Sie eine beliebige Struktur von Unterverzeichnissen erstellen, die Ihren persönlichen Gewohnheiten und den Bedürfnissen Ihres Unternehmens entspricht. Die Unterverzeichnisstruktur ist in allen iFolder-Mitgliedsordnern gleich.

Freigegebene iFolder-Ordner

Bei einem iFolder-Ordner handelt es sich um ein lokales Verzeichnis, das über einen zentralen Server die Freigabe und Synchronisierung von Dateien für eine benutzerdefinierte Gruppe von Benutzern vornimmt. Sämtliche iFolder-Mitglieder können auf die iFolder-Dateien zugreifen; Änderungen können von denjenigen Mitgliedern vorgenommen werden, die hierzu berechtigt sind. Sie können auf Ihre iFolder-Ordner über mehrere Arbeitsstationen zugreifen und diese sowohl für einzelne Benutzer als auch für LDAP-Gruppen freigeben.

Sie steuern, welche Benutzer den jeweiligen iFolder-Ordner nutzen dürfen und welche Zugriffsstufe den einzelnen Mitgliedern zugewiesen wird. Ebenso können Sie freigegebene iFolder-Ordner nutzen, die sich im Besitz anderer Benutzer in Ihrer für die Zusammenarbeit konfigurierten Umgebung befinden. Wenn Sie einen iFolder-Ordner freigeben, können Mitglieder mit Schreibberechtigung die Verzeichnisstruktur bearbeiten und die vorgenommenen Änderungen haben für alle Kopien des iFolder-Ordners Gültigkeit. Jeder Benutzer kann den freigegebenen iFolder-Ordner an einem von ihm ausgewählten Ort speichern, sofern dieser die Anforderungen in Richtlinien für den Speicherort und die Verwendung von iFolder-Ordnern erfüllt.

iFolder-Zugriffsrechte

Der iFolder-Client unterstützt das Eigentümer-Attribut sowie drei Zugriffsstufen für Mitglieder eines iFolder-Ordners:

WICHTIG:Mitglieder einer LDAP-Gruppe erben die für diese Gruppe festgelegten Zugriffsrechte.

  • Eigentümer: Nur ein einziger Benutzer fungiert als Eigentümer eines iFolder-Ordners. Hierbei handelt es sich normalerweise um den Benutzer, der den iFolder-Ordner erstellt hat. Der Benutzer mit Eigentümer-Attribut kann über den iFolder-Client das Eigentum auf ein anderes Mitglied des iFolder-Ordners übertragen.

    Der Eigentümer eines iFolder-Ordners verfügt über das Recht "Vollständige Kontrolle". Dieser Benutzer verfügt über den Lese-/Schreibzugriff für den jeweiligen iFolder-Ordner, verwaltet die Mitgliedschaft sowie die Zugriffsrechte der Mitglieder und kann das Recht "Vollständige Kontrolle" für jedes Mitglied aufheben.

    Wenn eine LDAP-Gruppe einen iFolder-Ordner besitzt, besitzen alle Mitglieder der Gruppe das Recht "Vollständige Kontrolle".

  • Vollständige Kontrolle: Ein Mitglied des freigegebenen iFolder-Ordners mit dem Zugriffsrecht "Vollständige Kontrolle". Dieses Mitglied verfügt über den Lese-/Schreibzugriff für den jeweiligen iFolder-Ordner, verwaltet die Mitgliedschaft sowie die Zugriffsrechte für alle Benutzer mit Ausnahme des Eigentümers.

  • Lesen/Schreiben: Ein Mitglied des freigegebenen iFolder-Ordners mit dem Zugriffsrecht "Lesen/Schreiben" für die Verzeichnisse und Dateien im iFolder-Ordner. Dieses Mitglied kann eine Datei bearbeiten, die lokalen Änderungen werden jedoch mit den Kopien des iFolder-Ordners anderer Mitglieder synchronisiert.

  • Nur Lesen: Ein Mitglied des freigegebenen iFolder-Ordners, mit dem Zugriffsrecht "Nur Lesen" für Verzeichnisse und Dateien im iFolder-Ordner. Dieses Mitglied kann eine Datei bearbeiten, die lokalen Änderungen werden jedoch nicht mit den Kopien des iFolder-Ordners anderer Mitglieder synchronisiert.

Jeder iFolder-Benutzer und jede LDAP-Gruppe kann Eigentümer einiger iFolder-Ordner und Mitglied anderer iFolder-Ordner sein. Als Benutzer mit Eigentümer-Attribut verfügen Sie stets über das Recht "Vollständige Kontrolle". Als Benutzer mit Mitglied-Attribut verfügen Sie in jedem freigegebenen iFolder-Ordner möglicherweise über eine andere Zugriffsstufe, abhängig davon, welche Zugriffsrechte Ihnen ein Mitglied erteilt hat, das über das Recht "Volle Kontrolle" verfügt.

Dateisynchronisierung

Wenn Sie mit dem Server verbunden sind, kann iFolder die iFolder-Mitgliedschaft und -Dateien, während Sie arbeiten, im Hintergrund synchronisieren. Wenn Sie ein iFolder-Konto einrichten, können Sie Passwort nicht vergessen aktivieren, sodass iFolder das Passwort auf dem Computer speichern kann. Wenn Sie iFolder das nächste Mal starten, wird die Verbindung zum Server automatisch hergestellt und die Daten werden ohne erneute Angabe des Passworts synchronisiert.

Sie können den iFolder-Client so konfigurieren, dass er jedes Mal automatisch ausgeführt wird, wenn Sie sich bei der Desktop-Umgebung Ihres Computers anmelden. Die Sitzung wird im Hintergrund ausgeführt, während Sie mit Dateien in Ihren lokalen iFolder-Ordnern arbeiten; dabei werden sämtliche von Ihnen vorgenommenen Änderungen verfolgt und protokolliert.

Sie können die Dateien manuell oder in angegebenen Intervallen synchronisieren.

iFolder verfügt über Delta-Synchronisierungsfähigkeiten. Wenn Sie Änderungen an einer iFolder-Datei vornehmen, werden nur die Deltas und Änderungen mit dem Server synchronisiert. In anderen Worten: Die Deltasynchronisierung ruft nur Aktualisierungen nach der letzten Synchronisierung zwischen dem iFolder-Client und dem Server ab. Auf diese Weise wird die Effizienz gesteigert, da die Synchronisierungszeit reduziert wird.

Migration

Novell iFolder bietet eine einfache, benutzerfreundliche Schnittstelle für die clientseitige Migration von iFolder2.x auf iFolder 3.6 und höher. Der auf dem iFolder-Client bereitgestellte Migrationsassistent hilft bei der schnellen und einfachen Migration der bestehenden iFolder 2.x-Daten auf iFolder 3.6 und höher. Weitere Informationen finden Sie unter Novell iFolder-Migration und -Upgrade.

Verschlüsselung

iFolder bietet höhere Sicherheit für Ihre vertraulichen iFolder-Dateien, um sie vor absichtlichem oder versehentlichem Zugriff durch unautorisierte Personen zu schützen. In der Vergangenheit wurde die Datensicherheit nicht auf Serverseite sichergestellt, sodass der Administrator oder Personen mit unautorisiertem Zugriff auf den Server auf die Daten zugreifen konnten. Außerdem besteht beim Zugriff auf iFolder-Dateien über einen ISP-Anbieter die Möglichkeit, dass eine unautorisierte Person unabsichtlich auf vertrauliche Dateien stoßen kann. Jetzt können Sie Ihre Dateien auf dem Server verschlüsseln und speichern sowie durch einen Passwortsatz abrufen, der nur Ihnen bekannt ist. Wenn Sie den Passwortsatz vergessen, hilft Ihnen der Wiederherstellungsagent, den Sie bei der Erstellung des verschlüsselten iFolder-Ordners ausgewählt haben, beim Wiederherstellen Ihrer Daten.

WICHTIG:Verschlüsselte iFolder-Ordner können nicht freigegeben werden.

Weitere Informationen zur Verschlüsselung finden Sie unter Verwalten von Passwortsätzen für verschlüsselte iFolder-Ordner.

Upgrade

Novell iFolder ermöglicht ein einfaches Upgrade von iFolder 3.x auf den iFolder-Client für Novell iFolder 3.6 und höhere Versionen. Weitere Informationen finden Sie unter Upgraden vom iFolder 3.x-Clients.

Zusammenführen

Mithilfe der Funktion "Zusammenführen" können Sie die Daten in den iFolder-Ordnern konsolidieren und den Datentransfer zwischen dem iFolder-Client und dem Server minimieren. Mithilfe von "Zusammenführen" können Sie außerdem einen vorhandenen Ordner an eine andere Position auf dem Client-Computer verschieben. Weitere Informationen finden Sie unter iFolder-Ordner zusammenführen.

Standard-iFolder-Ordner

iFolder ermöglicht Ihnen das Einrichten eines Standard-iFolder-Ordners beim Konfigurieren eines neuen Kontos. Immer wenn Sie ein neues iFolder-Konto konfigurieren, überprüft iFolder automatisch das Vorhandensein eines Standard-iFolder-Ordners. Falls Sie einen solchen Ordner noch nicht erstellt haben, fordert Sie iFolder zum Erstellen auf. Wenn er bereits vorhanden ist, fordert Sie der iFolder-Client zum Herunterladen auf. Sie können diese beiden Optionen auch ignorieren. Weitere Informationen hierzu finden Sie in Schritt 7 in Konfigurieren von iFolder-Konten.

Verbesserter Webzugriff

iFolder bietet eine verbesserte Webschnittstelle, mit der Sie auf Ihre iFolder-Dateien auf Remoteservern zugreifen können. Mit der Webzugriffskonsole können Sie einen Webbrowser für den Zugriff auf Ihre Dateien auf dem iFolder-Server von allen Arbeitsstationen ohne iFolder-Client aus verwenden. Die Benutzeroberfläche für den Webzugriff ermöglicht Ihnen das Ausführen aller Vorgänge des iFolder-Clients.

Im Vergleich zu den vorherigen iFolder-Versionen können Sie mit iFolder 3.6 und späteren Versionen iFolder-Ordner über die verbesserte Webzugriffskonsole erstellen, durchsuchen, freigeben, hochladen und löschen. Mobile Benutzer können diese Fähigkeit auch über das Web nutzen.

Synchronisierungsprotokoll

Aus dem Synchronisierungsprotokoll gehen die im Hintergrund ausgeführten iFolder-Aktivitäten hervor, beispielsweise das Herauf- und Herunterladen von Dateien für unterschiedliche iFolder-Ordner, Richtlinienverletzungen, ausgeschlossene Dateien und nicht synchronisierte Dateien.

Weitere Informationen hierzu finden Sie unter Rechtliche Hinweise.
