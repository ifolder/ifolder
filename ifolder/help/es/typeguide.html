Directrices para tipos y tamaños de archivos que no se van a sincronizar

Si se utiliza el Cliente iFolder, es posible que el administrador de iFolder aplique restricciones de tipo de archivo y de tamaño máximo de archivo al sistema, usuario, grupo LDAP o carpeta iFolder. Es posible que también se apliquen restricciones a la cuenta en cuanto a la cantidad de espacio que se puede utilizar para los datos de las carpetas iFolder que posee. Si participa en otras carpetas iFolder, el espacio consumido en el servidor se cuenta en el espacio del propietario de dicha carpeta iFolder.

Algunos tipos de archivo no son buenos candidatos para la sincronización como, por ejemplo, los archivos de sistema operativo, los archivos ocultos creados por un administrador de archivos o las bases de datos implementadas como una recopilación de archivos enlazados. Es posible incluir únicamente los tipos de archivo importantes utilizados en la empresa, o bien excluir los archivos que probablemente no están relacionados con ésta como, por ejemplo, los archivos .mp3.

Archivos de sistema operativo

Los directorios del sistema no deben convertirse en carpetas iFolder. La mayoría de archivos de sistema cambian raramente, y es mejor mantener un archivo de imagen del sistema básico y del software importante que intentar sincronizar dichos archivos en el servidor.

Archivos ocultos

Si el sistema de archivos utiliza archivos ocultos para realizar un seguimiento de las preferencias de visualización, es posible que el administrador aplique restricciones a estos tipos de archivo para evitar que se sincronicen en el sistema. Por lo general, los archivos ocultos sólo son importantes para el equipo concreto en el que se han creado y cambian cada vez que se accede al archivo o directorio. No es necesario conservar estos archivos, además, su sincronización provoca errores de conflicto de archivos repetidos.

Por ejemplo, iFolder excluye automáticamente dos archivos ocultos del gestor de archivos denominados thumbs.db y .DS_Store.

Archivos de base de datos

iFolder sincroniza archivos individuales o las partes modificadas de dichos archivos; no sincroniza archivos como un conjunto. Si tiene un archivo de base de datos que se ha implementado como una recopilación de archivos enlazados, no intente sincronizarlos en una carpeta iFolder.

No trate de sincronizar los datos de GroupWise convirtiendo los directorios del archivo de reserva, caché o los directorios remotos de GroupWise en carpetas iFolder. De lo contrario, los archivos de datos de GroupWise resultarán dañados tras sincronizarlos varias veces. GroupWise necesita que los archivos del archivo de reserva se mantengan como un conjunto de archivos.

Para obtener más información, consulte Información legal.
