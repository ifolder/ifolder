Actualización del Cliente iFolder 3.x

En esta sección se analizan los elementos siguientes:

  • Actualización automática a iFolder 3.7 o versiones posteriores

  • Actualización manual a versiones de iFolder 3.7 o posteriores

IMPORTANTE:Antes de actualizar el cliente 3.4.1, debe realizar una copia de seguridad del sistema de almacenamiento Simias local. Si no se lleva a cabo la copia de seguridad, se perderá toda la información de configuración del equipo, ya que éste suprime el sistema de almacenamiento Simias cuando se produce un error en la actualización.

Actualización automática a iFolder 3.7 o versiones posteriores

La actualización automática sólo es posible si cuenta con derechos de acceso raíz, ya que el proceso debe instalar los paquetes RPM. La contraseña raíz se solicita durante la actualización. Si no dispone de derechos de acceso raíz, el paquete no se instalará automáticamente. Sin embargo, los paquetes de cliente se descargarán en una ubicación temporal. El Cliente Linux se descargará en /tmp/<32chardir>/ifolder3-linux.tar.gz. De este modo podrá actualizarlo más adelante ya que la ubicación contiene dos guiones de instalación que puede ejecutar manualmente. Hasta que el cliente no se actualice, no podrá conectarse a iFolder 3.7 o posterior con un Cliente iFolder 3.x de Novell.

  1. Desde una carpeta iFolder 3.En un cliente iFolder x anterior a iFolder 3.7, intente conectarse al servidor iFolder. Se le preguntará si desea actualizar.

  2. Haga clic en Sí para actualizar automáticamente el cliente iFolder 3.x de Novell.

Para iFolder 3.4.1, si hace clic en No e intenta conectar el Cliente iFolder 3.4.1 al servidor iFolder 3.7 o posterior, aparecerá el siguiente mensaje de error. Haga clic en Aceptar para cerrar el mensaje.

Actualización manual a versiones de iFolder 3.7 o posteriores

  1. Haga clic en Actualizar en el menú Ayuda. Se le preguntará si desea actualizar. Haga clic en Aceptar para actualizar su cliente iFolder 3.x.

    NOTA:Al hacer clic en la opción de actualización, iFolder comprobará si existen actualizaciones en todos los servidores conectados.

    O bien

  1. Abra un terminal. En el indicador de comandos, ejecute install-ifolder.sh.

  2. Introduzca la contraseña raíz cuando se le solicite.

    Si está en ejecución, el Cliente iFolder se cerrará. A continuación, el servidor Web XSP y el Cliente iFolder se actualizarán junto con el módulo auxiliar (plug-in) Nautilus-ifolder3.

  3. Inicie iFolder desde el menú Aplicación.

Para obtener más información, consulte Información legal.
