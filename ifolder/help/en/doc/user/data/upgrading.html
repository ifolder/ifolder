4.2.1 Upgrading iFolder 3.x client to iFolder 3.6

If you are currently using the iFolder 3.x client and when you try to connect it to the iFolder 3.6 server, you are prompted whether you want to upgrade.

  1. Click Yes to automatically upgrade your iFolder 3.x client.

The automatic upgrade is possible only if you have the root access rights, as this process needs to install the rpm packages. If you don’t have the root access rights, the package does not get installed automatically. But, the client packages is downloaded to a temporary location that enable you to upgrade later as it contains an install script that can be run manually. Until you upgrade the client, you will not be able to connect to the iFolder 3.6 domain.

4.2.2 Coexistence of Novell iFolder 2.x and iFolder Clients

The iFolder client for Novell iFolder 3.6 and the Novell iFolder 2.x client can coexist on the same computer, but they are not integrated and their software should not be installed in the same folder. If you plan to use Novell iFolder 2.x on your computer, you must comply with the following restrictions:

  • Any local directory where you store your Novell iFolder 2.x data should not be converted to an iFolder.
  • Any directory that contains a local directory where you store your Novell iFolder 2.x data should not be converted to an iFolder.
