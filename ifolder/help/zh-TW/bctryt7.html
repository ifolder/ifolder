升級 iFolder 3.x 用戶端

本節討論以下各項：

  • 自動升級至 iFolder 3.7或更新版本

  • 手動升級到 iFolder 3.7 或更新版本

重要：您必須先備份本地的 Simias 儲存區，再升級至 3.4.1 用戶端。如果您不備份，會遺失電腦上的所有組態資訊，因為升級失敗時會刪除 Simias 儲存區。

自動升級至 iFolder 3.7或更新版本

只有在您有根部存取權限時才能自動升級，因為此程序需要安裝 RPM 套件。升級期間必須有根部密碼。如果您沒有根部存取權限，則無法自動安裝套件。但是，用戶端套件會下載至暫時的位置。Linux 用戶端會下載至 /tmp/<32chardir>/ifolder3-linux.tar.gz。這讓您可以稍後升級，因為位置包含兩個可以手動執行的安裝指令碼。除非您升級用戶端，否則無法使用 iFolder 3.x 用戶端連接到 iFolder 3.7 或更新版本的伺服器。

  1. 從 iFolder 3.x 之前的 iFolder 3.7 用戶端，嘗試連接到 iFolder 伺服器。系統提示您進行升級。

  2. 按一下「是」以自動升級您的 iFolder 3.x 用戶端。

若是 iFolder 3.4.1，如果您按一下「否」並嘗試將 iFolder 3.4.1 用戶端連接到 iFolder 3.7 或更新版本的伺服器，會顯示錯誤訊息。按一下「確定」關閉訊息。

手動升級到 iFolder 3.7 或更新版本

  1. 從「說明」功能表中按一下「升級」。系統提示您進行升級。按一下「接受」升級 iFolder 3.x 用戶端。

    附註：當您按下升級選項時，iFolder 將會檢查所有連接伺服器的升級內容。

    或

  1. 開啟一部終端機。在指令提示下，執行 install-ifolder.sh.

  2. 要求根部密碼時請輸入密碼。

    如果 iFolder 用戶端已經在執行中，這會關閉 iFolder 用戶端，然後使用 Nautilus-ifolder3 外掛程式升級 XSP Web 伺服器和 iFolder 用戶端。

  3. 從「應用程式」功能表啟動 iFolder。

有關更詳細的資訊，請參閱 法律聲明.
