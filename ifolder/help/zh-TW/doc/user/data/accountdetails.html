檢視和修改 iFolder 帳戶設定

  1. 以滑鼠右鍵按一下「通知」區域中的 iFolder 圖示 ，然後選取「帳戶」以開啟「iFolder 優先設定」至「帳戶」索引標籤。

  2. 選取帳戶以檢視或修改下列參數：

    參數 描述

    伺服器

    企業伺服器的 DNS 名稱。例如：svr21.example.com。

    使用者名稱

    此帳戶的使用者識別。您的 iFolder 管理員會決定您應該使用的格式是您的網路使用者 ID (LDAP 公用名稱) 或電子郵件地址。

    例如，如果您的姓名是 John Smith 並且使用者 ID 是 smith 而電子郵件地址是 john.smith@example.com，您的管理員就會為伺服器進行設定，該設定會決定要在「使用者名稱」欄位中使用這兩種格式中的哪一種，來進行驗證。

    您只能為每個企業伺服器建立一個帳戶。

    密碼

    使用者名稱的密碼。

    記住密碼

    選取「記住密碼」可以在每次登入電腦時都自動登入到此 iFolder 帳戶。若需要更多資訊，請參閱登入 iFolder 帳戶。

    如果您的網路密碼發生了變更，則下一次 iFolder 嘗試登入到 iFolder 帳戶時自動驗證理所當然就會失敗。iFolder 會提示您使用新密碼登入並且允許您指定並儲存這個新密碼。

    開啟帳戶

    選取「開啟帳戶」以允許對此帳戶的 iFolder 邀請、使用者清單及 iFolder 進行背景同步化處理。此操作僅僅管理用戶端功能。

    帳戶是「開啟」還是「關閉」的狀態則顯示在「帳戶」清單的「狀態」欄中。如果您設定了多個帳戶，就可以迅速檢視「開啟帳戶」設定的狀態。

    當您使用 Novell® iFolder® 3.x 的 iFolder 時，iFolder 系統管理員就可以在伺服器端關閉您的 iFolder 帳戶 (通常是暫時地)。您可以預期下列行為：

    • 如果您在該帳戶從伺服器端上關閉時登入，就可以管理 iFolder，而您的 iFolder 資料會繼續與伺服器同步，直到您通過登出或者關閉同步來結束會期，或者是伺服器切斷與會期的連接為止 (視哪一個先發生而定)。
    • 如果帳戶是在伺服器端關閉的，您就會接收到一個立現式訊息，告知您每次嘗試登入時帳戶都是關閉的。請聯絡您的 iFolder 系統管理員尋求協助。

    預設帳戶

    選取「預設帳戶」，使此帳戶成為在建立 iFolder 時帳戶下拉式清單中預設選取的帳戶。

  3. 在「iFolder 優先設定」的「帳戶」索引標籤上，選取 iFolder 帳戶，然後按一下「細節」以檢視有關「iFolder 帳戶」的下列資訊。

    參數 描述

    伺服器

    企業伺服器的主機名稱。例如，如果伺服器的 DNS 名稱是 svr21.example.com，則 svr1 就是所報告的資訊。

    描述

    由管理員輸入之企業伺服器、簡單伺服器或工作群組主機電腦的使用者友善描述。如果沒有輸入描述，該值就為「無」。

    可用的

    目前可用於此 iFolder 帳戶之 iFolder 的總空間量。該值等於容量減去已用空間。

    已使用

    此帳戶上所擁有之所有 iFolder 目前已消耗的總空間量。

    容量

    企業伺服器上配置給您的 iFolder 帳戶的總空間量。此值可能是為所有使用者設定的預設容量，或是使用者指定的容量。如果沒有設定容量，該值就為空或 N/A (不適用)。
