建立 iFolder

您可以藉由轉換符合下列指示的現有資料夾來建立 iFolder：

  • 放置 iFolder 的指示
  • iFolder 及其資料夾和檔案的命名慣例

如果該資料夾不存在，您可以瀏覽至某個位置並建立新資料夾作為程序的一部分。

在 iFolder 中，您可以設定任何次目錄結構，配合您個人或公司的工作習慣。該次目錄結構在所有成員 iFolder 之間均保持不變。每個工作站都可以為共用的 iFolder 指定不同的父目錄。

您可以直接在檔案管理員及 iFolder 瀏覽器 (位於用戶端應用程式中) 中建立 iFolder。使用下列方式建立 iFolder：

  • 在檔案管理員中建立 iFolder
  • 在 iFolder 瀏覽器中建立 iFolder
  • 在 FAT32 裝載點上建立 iFolder (Linux)


在檔案管理員中建立 iFolder

  1. 使用檔案管理員導覽至要轉換為 iFolder 的資料夾，然後選取該資料夾。

  2. 請使用下列其中一個方法：

    • 以滑鼠右鍵按一下資料夾圖示，然後按一下「轉換為 iFolder」。
    • 在工具列中，按一下「檔案」 > 「轉換為 iFolder」。

    該資料夾圖示 會變更為具有 iFolder 圖示 之外觀的資料夾。

  3. 如果出現「iFolder 已建立」訊息，請執行下列其中一項：

    • 按一下「關閉」，關閉該訊息。
    • 選取「不要再顯示這個訊息」以停用未來的通知，然後按一下「關閉」，關閉該訊息。

      如果稍後您希望獲得該「確認」訊息，請移至「iFolder 優先設定」，選取「建立 iFolder 時顯示確認對話方塊」，然後按一下「確定」。


在 iFolder 瀏覽器中建立 iFolder

  1. 在「通知」區域中，以滑鼠右鍵按一下 iFolder 圖示 ，然後按一下「iFolder」，以開啟 iFolder 瀏覽器。

  2. 按一下「新增」。

  3. 指定您要用於此 iFolder 的 iFolder 帳戶。

    如果您有多個 iFolder 帳戶，則預設為列出的第一個帳戶。使用下拉式清單以選取其他帳戶。

  4. 在瀏覽器中，導覽至要建立 iFolder 的目錄。

  5. 選取您要轉換為 iFolder 的一般資料夾，然後使用下列方式之一：

    • 如果該一般資料夾存在，則選取該資料夾，然後按一下「確定」。
    • 如果該一般資料夾不存在，則按一下「轉換新資料夾」來建立，然後選取新建立的資料夾並按一下「確定」。

  6. 按一下「確定」。

    該 iFolder 即會出現在 iFolder 清單中。

  7. 如果出現「iFolder 已建立」訊息，請執行下列其中一項：

    • 按一下「關閉」，關閉該訊息。
    • 選取「不要再顯示這個訊息」以停用未來的通知，然後按一下「關閉」，關閉該訊息。

      如果稍後您希望獲得該「確認」訊息，請移至「iFolder 優先設定」，選取「建立 iFolder 時顯示確認對話方塊」，然後按一下「確定」。


在 FAT32 裝載點上建立 iFolder (Linux)

在 Linux 上，iFolder 支援在 FAT32 檔案系統的裝載點上建立及使用 iFolder。iFolder 使用者必須擁有您要儲存 iFolder 的 FAT32 裝載點，且 /etc/fstab 檔案必須明確識別該裝載點之 iFolder 使用者的本地使用者 ID，您才可以在 Linux 電腦的 FAT32 卷冊上建立新的 iFolder 或設定共用 iFolder。

重要： 此為必要條件，因為 FAT32 不支援檔案許可權。任何其他不支援許可權的檔案系統都會有相同的問題。

若為雙開機系統，您可以如同在未將資料夾設為 iFolder 的系統中存取一般資料夾一樣，存取 iFolder 中的資料。如果您在兩個平台中同時將資料夾設為 iFolder，則資料集會儲存在伺服器上兩個不同的 iFolder 中，這將耗費兩倍的已配置伺服器磁碟資源。

因為 FAT32 檔案系統不支援 POSIX 檔案系統許可權，請確定 iFolder 使用者是裝載點的本地擁有者。請修改 /etc/fstab 檔案，將使用者本地登入識別的使用者 ID (UID) 參數設為在該裝載點上建立 iFolder 的 iFolder 使用者。

  1. 以根部使用者身份登入電腦。

  2. 建立要裝載 FAT32 檔案系統的目錄。在指令提示下，輸入

    mkdir /home/username/fsmount

    將 username 取代為要將 FAT32 磁碟機用於 iFolder 之使用者的使用者名稱。將 fsmount 取代為要用做裝載點的目錄名稱。例如，如果 FAT32 磁碟機或分割區的使用者名稱為 jsmith，目錄名稱為 fat32mntpt，則輸入

    mkdir /home/jsmith/fat32mntpt

  3. 將 FAT32 磁碟機裝載到想要的裝載點。在指令提示下，輸入

    mount -t vfat /dev/hda1 /home/username/fsmount

    將 /dev/hda1 取代為非 Linux 裝置的裝置或分割區名稱。將 username 取代為 iFolder 使用者的使用者名稱。將 fsmount 取代為要用做裝載點的目錄名稱。

    繼續範例，其中 FAT32 檔案系統 (vfat) 位於 hda4 裝置 (/dev/hda4)，使用者名稱為 jsmith，裝載點目錄為 /home/jsmith/fat32mntpt，請輸入

    mount -t vfat /dev/hda4 /home/jsmith/fat32mntpt

  4. 編輯 /etc/fstab 檔案，在定義 FAT32 裝載點的行上新增 iFolder 使用者的 uid=username 參數。

    /dev/hda1 /home/username/fsmount vfat uid=username

    繼續範例，修改 /etc/fstab 檔案，將 uid=jsmith 新增至裝載點：

    /dev/hda4  /home/jsmith/fat32mntpt  vfat  uid=jsmith
