設定 iFolder 交通的本地防火牆設定

如果您使用本地防火牆，則您必須將防火牆設為允許 iFolder 選擇本地動態連接埠或設定本地靜態連接埠的組態，以便 iFolder 用戶端可以在本地 (同一電腦) 與 Mono XSP Server 進行通訊。iFolder 會使用 Mono XSP Web 服務透過 HTTP 或 HTTPS (SSL，預設) 與您具有 iFolder 帳戶的 Novell iFolder 企業伺服器進行通訊。iFolder 管理員會設定 iFolder 服務的組態，決定您環境中的通訊是使用 HTTP 還是 HTTPS。

因為 Mono XSP 將標準 HTTP 或 HTTPS 連接埠用於交通，所以您的防火牆不會封鎖 Mono XSP 與 Novell iFolder 企業伺服器之間的交通。不過，如果您封鎖用戶端與 Mono XSP 之間的交通，則用戶端將無法與伺服器通訊。您將無法登入 iFolder 帳戶及同步化 iFolder 資料。

開始時，iFolder 組態並未指定要用於用戶端至 Mono XSP 通訊的特定連接埠。當 iFolder 初次執行時，它會開啟一個可用的本地連接埠，以傾聽 Mono XSP 的要求並透過 Mono XSP 將資料傳送給伺服器。它會將連接埠編號寫入組態檔案，並在此後一直使用此連接埠，直到每次啟動 iFolder 時無法使用該連接埠為止。如果該連接埠正由另一應用程式使用，則 iFolder 會選取其他可用的連接埠，並在組態檔案中重新設定該連接埠的組態，此後會一直使用這個新的連接埠。例如，如果 iFolder 已關閉，所以另一應用程式採用了該連接埠，則當您重新啟動 iFolder 時，iFolder 必須選取其他連接埠並設定其組態才可以使用。


使用本地動態連接埠

如果您在電腦上使用本地防火牆，則每當 iFolder 聯絡 Mono XSP 伺服器以同步化其使用者清單及 iFolder 資料時，該防火牆都應偵測 iFolder 與該伺服器之間的本地交通。讓交通在每次啟動用戶端時避免接收立現式警告。如果您永久性的允許該交通，則防火牆將不會再提醒您。


使用本地靜態連接埠

您可以選擇性地在 Simias.config 檔案的 Service Manager 區段新增「Web 服務埠範圍」參數，以允許 Mono XSP Server 使用靜態連接埠或某範圍的連接埠來進行此本地交通。然後可以在防火牆中開啟此埠或此範圍的埠，以允許 Mono XSP 的通訊。

您可以在下列位置找到 Simias.config：

作業系統 Simias.config 檔案的位置

Linux

/home/username/.local/share/simias/Simias.config

Windows

C:\Documents and Settings\username\Local Settings\Application Data\simias\Simias.config

Macintosh

/Users/username/.local/share/simias/Simias.config

將「Web 服務連接埠範圍設定」新增到 Simias.config 檔案的「服務管理員」區段。例如：

<configuration>
  <section name="ServiceManager">
    <setting name="WebServicePortRange" value="monoxsp_port" />   
  </section>
</configuration>

將 monoxsp_port 取代為某一連接埠編號或某一範圍的連接埠編號：

  • 指定一個單一的四位數連接埠編號值 (如 1234)，以使用單一的靜態連接埠。例如：
    <setting name="WebServicePortRange" value="1234" />
  • 指定某一範圍的連接埠 (如 "1234-1236")，以使用指定連接埠範圍內的任意可用連接埠。例如：
    <setting name="WebServicePortRange" value="1234-1236" />

設定系統的組態，以使用特定連接埠或某一範圍的連接埠：

  1. 設定防火牆的組態，以允許您要用於 iFolder 交通之連接埠或連接埠範圍的交通。

    請參閱您的防火牆文件，以取得如何在防火牆中開啟連接埠的相關資訊。

  2. 藉由以滑鼠右鍵按一下 iFolder 圖示 ，然後選取「離開」，來停止 iFolder 用戶端。

  3. 藉由將「Web 服務連接埠範圍」參數新增到「服務管理員」區段來修改 Simias.config 檔案。

  4. 啟動 iFolder 用戶端。

    如需更多資訊，請參閱啟動 iFolder 用戶端。
