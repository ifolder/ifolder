Общие сведения о процессе миграции

Приложение Novell iFolder предоставляет возможность простой и быстрой миграции с клиента iFolder 2.x на клиент iFolder 3.7 или более поздней версии. Помощник по миграции в клиенте iFolder позволяет быстро и легко выполнить миграцию имеющихся данных iFolder 2.x.

При запуске мастер миграции обнаруживает установку версии 2.x и выводит запрос на миграцию данных. Во время миграции появляется запрос на преобразование существующих папок или на сохранение копий папок приложения iFolder 2.x. Для получения дополнительной информации об этом см. Шаг 6.

Приложение iFolder предлагает выполнить повторную синхронизацию файлов с сервером 3.7 или более поздней версии и спрашивает, необходимо ли сохранить клиент 2.x до завершения синхронизации. В ходе этого процесса выполняется только миграция данных, поэтому пользователь должен выполнить настройку нового клиента.

Процедура миграции

  1. Установите клиент iFolder для сервера iFolder 3.7 или более поздней версии, а затем настройте учетную запись для сервера iFolder Enterprise Server 3.7 или более поздней версии.

    Для получения информации об установке клиента и настройке учетной записи см. раздел Управление учетными записями и настройками iFolder.

  2. В навигаторе iFolder создайте одну или несколько папок iFolder на локальном компьютере.

    Для папки iFolder можно использовать любое допустимое имя. Не создавайте папку iFolder в папке iFolder 2.x. См. дополнительные сведения: Создание и отправка папки iFolder.

  3. В приложении iFolder отобразится сообщение о существовании предыдущей версии данных и запрос на миграцию.

    Щелкните ОК, чтобы открыть список доступных папок iFolder 2.x.

  4. Выберите папки iFolder 2.x, которые необходимо мигрировать.

    Щелкните Миграция для запуска помощника по миграции.

    ПРИМЕЧАНИЕ.Помощника по миграции можно также запустить, выбрав iFolder > Миграция > Миграция с версии iFolder 2.x.

    Помощник по миграции помогает последовательно осуществить процесс миграции для полного переноса данных.

  5. Щелкните Далее для отображения опций миграции.

  6. Выберите одно из следующих действий:

    Выполните миграцию папки iFolder и отключите ее от сервера 2.х: Выберите этот параметр для отключения существующей папки от iFolder 2.x сервер после миграции на версию 3.7 или более позднюю.

    Создайте копию папки iFolder и перенесите ее: Выберите этот параметр для создания копии iFolder 2.x и преобразования их в формат iFolder 3.7 или более поздней версии. При этом файлы, находящиеся в домене iFolder 2.x сервер, даже после миграции файлов на сервер iFolder 3.7 или более поздней версии.

    При выборе этой опции необходимо также выбрать функцию Копировать родительскую папку, если необходимо скопировать не только содержимое папки 2.x, но и родительскую папку. Если эта опция не выбрана, в версию 3.7 или более позднюю будет перенесено только содержимое выбранной папки.

    Щелкните Просмотр, чтобы определить местоположение для хранения мигрированной папки.

    Например каталог c:/Documents and Settings/<имя_пользователя>/ifolder2dir является папкой iFolder 2.x, а dir1 и dir2 - каталоги в папке ifolder2dir. При выборе опции Копировать родительскую папку в новом местоположении создается каталог ifolder2dir и каталоги dir1 и dir2 копируются в него. Если эта опция не выбрана, в указанное местоположение перемещаются только каталоги dir1 и dir2.

  7. Щелкните кнопку Далее, чтобы выбрать домен и параметры безопасности, как указано ниже:

    1. В раскрывающемся списке выберите сервер.

    2. Выберите необходимые опции защиты:

      • Выберите Шифрование для шифрования выбранной папки iFolder.

        Если контрольная фраза не была указана при входе, iFolder выдаст запрос на ввод контрольной фразы. Можно пропустить это действие и настроить контрольную фразу позднее.

        Если соединение с сервером iFolder еще не установлено, появится запрос на подключение к нему.

      • Выберите параметр Совместное использование для совместного использования папки iFolder с другими участниками на сервере iFolder.

        Можно выполнить миграцию зашифрованной папки iFolder 2.x в совместно используемую папку. При попытке выполнить миграцию зашифрованной папки iFolder 2.x в совместно используемую папку появляется запрос на подтверждение выполнения действия.

        Для подтверждения действия щелкните ОК.

  8. Щелкните Далее для миграции выбранной папки iFolder.

  9. Проверьте правильность указанной информации, при необходимости щелкните Назад , чтобы вернуться на предыдущую страницу и внести изменения.

    Если информация указана правильно, щелкните Миграция , чтобы начать миграцию.

    Щелкните Назад для возврата и внесения необходимых изменений.

  10. Щелкните Готово, чтобы выполнить миграцию.

Дополнительную информацию см. в Юридическая информация.
