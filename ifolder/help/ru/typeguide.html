Рекомендации по типам и размерам не синхронизируемых файлов

Когда используется клиент iFolder, администратор iFolder может ввести ограничения на типы и максимальный размер файлов на уровне системы, пользователя, группы LDAPGroup или iFolder. Для Вашей учетной записи могут быть также установлены ограничения на объем пространства, которое может использоваться для данных в папках iFolder, владельцем которых Вы являетесь. Если Вы становитесь членом других папок iFolder, пространство, потребляемое на сервере, вычитается из количества пространства владельца данной папки iFolder.

Некоторые типы файлов не предназначены для синхронизации, например, файлы операционной системы, скрытые файлы, созданные менеджером файлов, или базы данных, которые реализованы как набор связанных фалов. Вы можете включить в синхронизацию только основные типы файлов, относящиеся к Вашей работе, или исключить те файлы, которые не связаны к нею, например файлы .mp3.

Файлы операционной системы

Вам не следует преобразовывать системные каталоги в папки iFolder. Большинство системных файлов изменяются редко, поэтому рекомендуется хранить файл образа основной системы и ключевого программного обеспечения, но не синхронизировать данные файлы с сервером.

Скрытые файлы

Если файловая система использует скрытые для отображения на дисплее файлы, администратор может установить ограничение на файлы такого типа, чтобы исключить их синхронизацию в системе. Как правило, скрытые файлы относятся только к тому компьютеру, на котором они были созданы, и изменяются при каждом доступе к файлу или каталогу. Хранить данные файлы не требуется, а их синхронизация приведет к повторяющимся ошибкам конфликтов файлов.

Например iFolder автоматически исключает два скрытых файла менеджера файлов с именами thumbs.db и .DS_Store.

Файлы базы данных

iFolder синхронизирует отдельные файлы или измененные части отдельных файлов и не выполняет синхронизацию файлов как набора. Если у Вас есть база данных, которая реализована как набор связанных файлов, не пытайтесь синхронизировать их в папке iFolder.

Не пытайтесь синхронизировать данные GroupWise путем преобразования архива GroupWise, кэша или удаленных каталогов в папки iFolder. В противном случае после выполнения нескольких синхронизаций файлы базы данных GroupWise станут поврежденными. Для GroupWise требуется, чтобы файлы архива обслуживались в виде набора файлов.

Дополнительную информацию см. в Юридическая информация.
