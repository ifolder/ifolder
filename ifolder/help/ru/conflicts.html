Разрешение конфликтов файлов

Конфликты могут возникнуть при одновременном хранении версий файла на разных компьютерах или, если символы в имени файла отличаются только значением регистра. При возникновении конфликтов файл сохраняется в области конфликта и получает отметку конфликта имен. Значок ошибки iFolder отображается на папке iFolder. При включении опции Оповещать о конфликтах в настройках iFolder Preferences в область оповещений отправляется предупреждение. Можно управлять всплывающими оповещениями путем настройки параметров в диалоговом окне Настройки iFolder на вкладке > Общие.

Способы разрешения конфликтов файлов см. в следующих разделах:

  • Разрешение конфликтов версий файлов

  • Разрешение конфликтов имен файлов

Разрешение конфликтов версий файлов

Для разрешения конфликта версий файлов необходимо выбрать правильную версию в диалоговом окне "Разрешение конфликтов".

  1. В области оповещений щелкните правой кнопкой значок Сервисы iFolder , затем щелкните Папки iFolder, чтобы открыть навигатор iFolder.

  2. Выберите файл в конфликте , затем щелкните Разрешить, чтобы открыть диалоговое окно Разрешение конфликтов.

    Для просмотра доступны две версии одного файла:

    • Локальная версия. Файл, сохраненный на локальном компьютере.

    • Версия сервера. Файл, сохраненный на сервере.

    Можно просмотреть подробную информацию о файле на локальной рабочей станции и о файле на сервере, чтобы определить, какую версию файла нужно сохранить.

  3. Щелкните Открыть для просмотра каждого файла.

  4. Выберите одно из следующих значений для опции Сохранить в iFolder в соответствии со своими предпочтениями.

    Локальная версия. Выберите эту опцию для сохранения локальной версии в iFolder.

    Версия сервера. Выберите эту опцию для сохранения версии сервера в iFolder.

  5. Выберите Включить для включения параметра Сохранить в корзину. При включении этой опции конфликтующий файл автоматически выбирается для сохранения в корзине. Выбранный вариант определяет, какая версия должна локально сохранять в корзине для дальнейшего использования.

  6. После устранения конфликта щелкните Сохранить.

Разрешение конфликтов имен файлов

Конфликты имен файлов возникают, когда имя локального файла и имя файла на сервере указаны различным регистром. Причем, содержимое версий файла может быть идентичным или отличаться. Данный тип конфликта можно устранить путем переименования файлов, но внимательно изучите, какую именно версию файла нужно переименовать.

  • Если содержимое файлов не отличается, можно переименовать один из файлов в диалоговом окне Разрешение конфликтов.

  • Если содержимое отличается, убедитесь, что у Вас есть копия локального файла и назначьте ей другое имя, затем вернитесь в диалоговое окно "Разрешение конфликтов" и сохраните серверную версию файла. Это действие позволяет сохранить в папке iFolder обе версии файла с различными именами. При переименовании версии файла в диалоговом окне "Разрешение конфликтов", эта версия файла перезаписывает другую, содержимое которой удаляется.

  1. В области оповещений щелкните правой кнопкой значок сервисов iFolder , затем щелкните Папки iFolder, чтобы открыть навигатор iFolder.

  2. Выберите файл в конфликте , затем щелкните Разрешить, чтобы открыть диалоговое окно "Разрешение конфликтов".

  3. Убедитесь в конфликте имен файла и определите, отличается ли содержимое этих версий файла.

  4. Выполните одно из следующих действий.

    • Если содержимое не отличается, переименуйте один из файлов, щелкните Сохранить эту версию, затем щелкните ОК, чтобы закрыть диалоговое окно "Разрешение конфликтов". Действие выполнено.

    • Если содержимое отличается, перейдите к Шаг 5.

  5. Закройте диалоговое окно "Разрешение конфликтов".

  6. Откройте менеджер файлов и найдите локальную копию конфликтного файла.

  7. Скопируйте файл и сохраните его с уникальным именем в папке iFolder.

    iFolder синхронизирует новый файл с сервером и компьютерами пользователей.

  8. Вернитесь в диалоговое окно Разрешение конфликтов, выберите серверную версию файла, щелкните Сохранить эту версию, затем щелкните ОК.

    Серверная версия файла будет загружена на локальную рабочую станцию. Она перезапишет локальную копию файла с этим именем.

Дополнительную информацию см. в Юридическая информация.
