Создание и отправка папки iFolder

Можно создать папку iFolder на основе существующей папки, которая удовлетворяет следующим принципам:

  • Рекомендации по размещению и использованию папок iFolder

  • Соглашения об именовании папки iFolder и содержащихся в ней папок и файлов

Если папка не существует, можно найти требуемое местоположение и создать новую папку.

ВАЖНО.Можно совместно использовать только обычные папки iFolder. Совместное использование зашифрованных папок iFolder не поддерживается.

Внутри папки iFolder можно установить любую структуру подкаталогов, которая соответствует личным или корпоративным потребностям. Структура подкаталогов одинакова для всех членов папок iFolder. На каждой рабочей станции можно указать различный родительский каталог для совместно используемой папки iFolder.

При превышении ограничения на число папок iFolder, которые может создать отдельный пользователь, iFolder отображает сообщение об ошибке. Например, если для политики Число папок на пользователя администратор установил значение 10, а пользователь пытается создать одиннадцатую папку, iFolder уведомит о нарушении политики.

Linux

Папки iFolder можно создавать с помощью менеджера файлов или в навигаторе iFolder, который доступен в приложении клиента. Для создания папки iFolder воспользуйтесь следующими методами:

  • Создание папок iFolder в средстве просмотра iFolder

  • Создание папок iFolder в менеджере файлов

Создание папок iFolder в средстве просмотра iFolder

  1. В области оповещений откройте навигатор iFolder одним из следующих способов:

    • Щелкните значок Сервисы iFolder .

    • Щелкните правой кнопкой значок Сервисы iFolder , затем щелкните Папки iFolder.

  2. Щелкните Создать iFolder.

  3. В поле Дополнительно укажите Учетную запись iFolder, которую необходимо использовать для данной папки iFolder.

    При наличии нескольких учетных записей iFolder первой будет выведена учетная запись по умолчанию. При необходимости выберите другую учетную запись в раскрывающемся списке.

  4. В средстве просмотра найдите каталог, в катаром нужно создать папку iFolder.

  5. Выберите обычную папку, в которой нужно создать папку iFolder. Для этого воспользуйтесь одним из следующих способов:

    • Если обычная папка существует, выберите ее, затем нажмите Добавить.

    • Если стандартная папка не существует, щелкните Создать папку для ее создания, выберите созданную папку, затем щелкните Добавить.

  6. Выберите одно из предлагаемых значений для опции Тип папки iFolder. По своему усмотрению можно создать зашифрованную или обычную папку iFolder.

    ВАЖНО.Создание зашифрованных папок iFolder возможно только в том случае, если администратор iFolder включил политику шифрования. В противном случае возможно создание только обычных папок iFolder (эта опция выбрана по умолчанию). В этом случае оба значения для опции Тип отключены.

    • Выберите Шифрование с контрольной фразой, чтобы включить шифрование для выбранной папки iFolder.

      Если во время сеанса входа контрольная фраза не была задана, при выборе значения Шифрование с контрольной фразой появляется запрос на ввод контрольной фразы. Укажите контрольную фразу в диалоговом окне "Контрольная фраза".

    • Выберите Обычный, чтобы разрешить совместное использование папки iFolder.

      В области оповещений появляется всплывающее сообщение iFolder с информацией о доступности совместно используемых папок iFolder. При этом пользователи, совместно использующие папку, могут щелкнуть сообщение, чтобы настроить папку iFolder, или закрыть сообщение, чтобы загрузить папку iFolder позднее.

  7. Выберите Защищенная синхронизация, чтобы включить защищенную синхронизацию файлов в создаваемой папке iFolder.

    Можно включить использование защищенного канала как для зашифрованных, так и для обычных папок iFolder. Если эта опция выбрана, iFolder использует защищенный канал для синхронизации папки iFolder между сервером и локальным компьютером.

    ПРИМЕЧАНИЕ.Можно включить защищенную синхронизацию для выбранной папки iFolder на локальном компьютере, щелкнув правой кнопкой iFolder и выбрав опцию "Свойства" в основном окне iFolder. Для этого щелкните правой кнопкой iFolder, выберите Свойства и на вкладке "Общие" установите флажок Защищенная синхронизация.

  8. Нажмите кнопку ОК.

    Папка iFolder появляется в списке Папки iFolder на этом компьютере.

  9. При появлении сообщения Папка iFolder создана выполните одно из следующих действий:

    • Щелкните Закрыть, чтобы закрыть сообщение.

    • Выберите Не выводить больше это сообщение, чтобы отключить дальнейшие оповещения, затем щелкните Закрыть, чтобы закрыть сообщение.

      Если в дальнейшем потребуется получать сообщения о подтверждении, перейдите в меню Настройки iFolder, выберите Отображение диалогового окна "Подтверждение" при создании iFolder, затем щелкните ОК.

Создание папок iFolder в менеджере файлов

  1. С помощью менеджера файлов найдите папку, из которой нужно сделать папку iFolder, затем выберите ее.

  2. Воспользуйтесь одним из следующих способов:

    • Щелкните правой кнопкой мыши значок папки, затем выберите Преобразовать в папку iFolder.

    • На панели инструментов щелкните Файл > Преобразовать в папку iFolder.

    Значок папки сменится значком папки со значком iFolder (зеленая буква "i").

  3. При появлении сообщения Папка iFolder создана выполните одно из следующих действий:

    • Щелкните Закрыть, чтобы закрыть сообщение.

    • Выберите Не выводить больше это сообщение, чтобы отключить дальнейшие оповещения, затем щелкните Закрыть, чтобы закрыть сообщение.

      Если в дальнейшем потребуется получать сообщения о подтверждении, перейдите в меню "Настройки iFolder", выберите Отображение диалогового окна "Подтверждение" при создании iFolder, затем нажмите ОК.

Создание папок iFolder в точке монтирования FAT32 (Linux)

В системе Linux можно создавать и использовать папки iFolder в точке монтирования для файловой системы FAT32. Перед созданием новых папок iFolder или настройкой совместного использования папок iFolder на томе FAT32 компьютера Linux пользователь iFolder должен быть владельцем точки монтирования FAT32, в которой будут храниться папки iFolder, а в файле /etc/fstab должно находиться явное указание идентификатора локального пользователя iFolder для точки монтирования.

ВАЖНО.Это необходимо, так как файловая система FAT32 не поддерживает разрешения файлов. Данная проблема характерна для всех файловых систем, которые не поддерживают разрешения файлов.

При наличии двойной загрузочной системы можно получать доступ к данным в папках iFolder как к данным обычных папок из системы, в которой эта папка не настроена как папка iFolder. Если Вы установили папку iFolder на обеих платформах, набор данных хранится в двух различных папках iFolder на сервере, которые потребляют двойную часть выделенного дискового пространства на сервере.

Так как файловая система FAT32 не поддерживает разрешения файловой системы POSIX, убедитесь, что пользователь iFolder является локальным владельцем точки монтирования. Измените файл /etc/fstab, чтобы установить идентификатор пользователя (UID) для пользователя iFolder, который создает папки iFolder в этой точке монтирования.

  1. Войдите на компьютер в качестве пользователя root.

  2. Создайте каталог, в котором нужно смонтировать файловую систему FAT32. В командной строке введите команду

    mkdir /home/username/fsmount

    Вместо параметра имя_пользователя укажите имя пользователя, который будет использовать диск FAT32 для папок iFolder. Вместо fsmount укажите имя каталога, который должен использоваться в качестве точки монтирования. Например, если именем пользователя является jsmith, а именем каталога является fat32mntpt для диска FAT32 или раздела, введите команду

    mkdir /home/jsmith/fat32mntpt

  3. Смонтируйте диск FAT32 в качестве требуемой точки монтирования. В командной строке введите команду

    mount -t vfat /dev/hda1 /home/username/fsmount

    Вместо /dev/hda1 укажите имя устройства или раздела для устройства, которое не принадлежит системе Linux. Вместо параметра имя_пользователя укажите имя пользователя iFolder. Вместо fsmount укажите имя каталога, который должен использоваться в качестве точки монтирования.

    Таким образом, если имеются файловая система FAT32 (vfat) , которая находится на устройствеhda4 (/dev/hda4), имя_пользователя jsmith и каталог точки монтирования/home/jsmith/fat32mntpt, введите команду

    mount -t vfat /dev/hda4 /home/jsmith/fat32mntpt

  4. Измените файл /etc/fstab, добавив параметр uid=имя_пользователяпользователя iFolder в строку, которая определяет точку монтирования FAT32.

    /dev/hda1  /home/username/fsmount  vfat  uid=username

    В данном примере измените файл /etc/fstab, добавив параметрuid=jsmithв точку монтирования:

    /dev/hda4  /home/jsmith/fat32mntpt  vfat  uid=jsmith

Дополнительную информацию см. в Юридическая информация.
