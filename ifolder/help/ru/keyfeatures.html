Совместное использование папки iFolder

Обычно, если Вы работаете в нескольких местоположениях или совместно работаете с другими пользователями, необходимо постоянное управление версиями файлов. Чтобы получить самую последнюю версию файлов на всех компьютерах, на которых Вы работаете, достаточно установить на них клиент iFolder и создать папку iFolder. iFolder также позволяет совместно использовать несколько папок iFolder и их содержимое с другими авторизованными пользователями или группами LDAPGroup. Для получения дополнительной информации о совместном использовании папки iFolder см. раздел Совместное использование папок iFolder.

Клиент iFolder поддерживает совместное использование информации путем синхронизации файлов на различных компьютерах через центральный сервер. С его помощью можно выполнять следующие задачи:

  • совместно использовать файлы на различных компьютерах;

  • совместно использовать файлы с другими пользователями;

  • владеть несколькими папками iFolder;

  • использовать различные папки iFolder, доступ к которым был предоставлен другими пользователями;

  • быть владельцем или членом папок iFolder с помощью различных учетных записей.

Работа с папками iFolder осуществляется непосредственно из менеджера файлов или с помощью навигатора iFolder, который входит в состав клиента. Внутри папки iFolder можно установить любую структуру подкаталогов, которая соответствует личным или корпоративным потребностям. Структура подкаталогов одинакова для всех членов папок iFolder.

Совместное использование папок iFolder

Папка iFolder представляет собой локальный каталог, файлы которого могут совместно использоваться определенными пользователями и синхронизироваться через центральный сервер. Файлы iFolder доступны всем членам папки iFolder и могут изменяться теми пользователями, кто обладает соответствующими правами. Доступ к папкам iFolder можно осуществлять с нескольких рабочий станций; кроме того, папки iFolder можно использовать совместно с отдельными пользователями и членами групп LDAPGroup.

Вы решаете, какие пользователи смогут получать доступ к папкам iFolder, а также устанавливаете права доступа каждого члена. Подобным образом Вы можете стать пользователем совместно используемых папок iFolder, владельцами которых являются другие сотрудники Вашей организации. При совместном использовании папки iFolder пользователи с правом "Запись" могут изменять структуру каталогов, после чего изменения осуществляются во всех копиях папки iFolder. Каждый пользователь может указать совместно используемую папку iFolder в любом расположении на своем компьютере в соответствии с требованиями раздела Рекомендации по размещению и использованию папок iFolder.

Права доступа к iFolder

Клиент iFolder поддерживает владельца и три уровня доступа для членов папки iFolder:

ВАЖНО.Члены группы LDAPGroup наследуют права доступа, установленные для данной группы.

  • Владелец: Только один пользователь может быть владельцем папки iFolder. Как правило, это пользователь, который создал папку iFolder. Владелец может использовать клиент iFolder для передачи права на владение другому члену папки iFolder.

    Владелец папки iFolder имеет право на полный контроль. Этот пользователь имеет право доступа для чтения/записи к папке iFolder, управляет членством и правами доступа пользователей, а также может удалять право "Полный контроль" для любого члена.

    Если группа LDAPGroup является владельцем папки iFolder, всем пользователям, являющимся членами этой группы, предоставляется право "Полный контроль".

  • Полный контроль: Уровень доступа члена совместно используемой папки iFolder с правом на полный контроль. Этот член имеет право доступа для чтения/записи к папке iFolder, управляет членством и правами доступа всех пользователей за исключением владельца.

  • Для чтения/записи: Это право получает член совместно используемой папки iFolder для доступа к каталогам и файлам в папке iFolder с правами "Чтение/Запись". Данный член может изменять свой файл, и эти локальные изменения синхронизируются с копиями папки iFolder других членов.

  • Только чтение: Уровень доступа члена совместно используемой папки iFolder с правом только для чтения в каталогах и файлах папки iFolder. Данный член может изменять свой файл, но эти локальные изменения не синхронизируются с копиями папки iFolder других членов.

Любой пользователь iFolder может владеть несколькими папками iFolder и быть членом других папок iFolder. Владелец всегда обладает правом "Полный контроль". Пользователи могут иметь различный уровень доступа к каждой совместно используемой папке iFolder в зависимости от права доступа, устанавливаемого членом с правом "Полный контроль".

Синхронизация файлов

При подключении к серверу приложение iFolder может выполнить синхронизацию членства и файлов iFolder в фоновом режиме во время работы пользователя. При настройке учетной записи iFolder пользователь может включить функцию Запомнить пароль. Таким образом, пароль запоминается в приложении iFolder на компьютере. При последующем запуске iFolder происходит автоматическое подключение к серверу и выполняется синхронизация данных без отображения запроса на ввод пароля.

Можно настроить программное обеспечение клиента iFolder для автоматического запуска при каждом входе пользователя на рабочую станцию. Сеанс выполняется в фоновом режиме, пока пользователь работает со своими файлами в локальной папке iFolder. При этом осуществляется отслеживание и протоколирование всех выполняемых изменений.

Можно указать интервалы синхронизации файлов.

В приложении iFolder также предусмотрена возможность дельта-синхронизации. После изменения файла iFolder выполняется только дельта-синхронизация или синхронизация изменений с сервером. Другими словами, при дельта-синхронизации осуществляются только обновления, выполненные с момента последней синхронизации между клиентом iFolder и сервером. Благодаря сокращению времени синхронизации эффективность процесса повышается.

Migration (Переход)

Novell iFolder предоставляет простой интуитивно понятный пользовательский интерфейс для перехода с iFolder 2.x на клиент iFolder 3.6 или более поздней версии. Мастер миграции в клиенте iFolder помогает быстро и легко выполнить переход с установленной версии iFolder 2.x на приложение iFolder 3.6 или более поздней версии. См. дополнительные сведения: Миграция и обновление Novell iFolder.

Шифрование

В приложении iFolder предусмотрены дополнительные меры безопасности, необходимые для защиты конфиденциальных файлов iFolder от намеренного или непреднамеренного несанкционированного доступа. Ранее безопасность данных на сервере не гарантировалась, поэтому данные были доступны администратору или любому пользователю, получившему несанкционированный доступ к серверу. Кроме того, если доступ к файлам iFolder осуществляется через поставщика Интернет-услуг, существует вероятность несанкционированного непреднамеренного доступа к конфиденциальным файлам. Теперь можно шифровать и хранить файлы на сервере, а затем дешифровывать их, указав контрольную фразу, известную только Вам. Если вы забыли контрольную фразу, восстановить данные можно с помощью агента восстановления, выбранного при создании зашифрованной папки iFolder.

ВАЖНО.Совместное использование зашифрованных папок iFolder не поддерживается.

Для получения дополнительной информации о шифровании см. раздел Управление контрольными фразами для зашифрованных папок iFolder.

Upgrade

Novell iFolder предоставляет возможность простого обновления версии iFolder 3.x на клиент iFolder для Novell iFolder 3.6 и более поздних версий. См. дополнительные сведения: Обновление клиента iFolder 3.x.

Объединить

Функция слияния позволяет консолидировать данные iFolder и уменьшить объем данных, передаваемых между клиентом iFolder и сервером. Функция слияния также позволяет перемещать существующие папки в другое местоположение на клиентской рабочей станции. См. дополнительные сведения: Слияние папок.

Папка iFolder по умолчанию

В приложении iFolder при конфигурировании новой учетной записи можно настроить папку iFolder по умолчанию. При конфигурировании новой учетной записи iFolder в приложении iFolder выполняется автоматическая проверка наличия папки iFolder по умолчанию. Если папка iFolder по умолчанию не была создана, появляется запрос на создание. При наличии такой папки клиент iFolder отображает запрос на загрузку. Можно также отказаться от обоих указанных вариантов. Для получения дополнительной информации см. подраздел Шаг 7 в разделе Конфигурирование учетной записи iFolder.

Расширенный веб-доступ

В приложении iFolder предоставляется усовершенствованный веб-интерфейс, позволяющий получать доступ к файлам iFolder на удаленных серверах. Благодаря усовершенствованной консоли веб-доступа можно использовать веб-навигатор для доступа к файлам на сервере iFolder с любой рабочей станции без установки клиента iFolder. Клиент веб-доступа позволяет выполнять все операции клиента iFolder.

По сравнению с предыдущими версиями iFolder 3.6 и более поздние версии позволяют создавать, просматривать, совместно использовать, отправлять и удалять папки iFolder через усовершенствованную консоль веб-доступа. Мобильные пользователи могут также получать доступ к этим функциям через Интернет.

Журнал синхронизации

В журнале синхронизации отображаются все действия iFolder в фоновом режиме, например отправка или получение файлов для различных папок iFolder, нарушения политик, исключенные файлы и несинхронизированные файлы.
