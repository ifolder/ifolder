Рекомендации по типам и размерам не синхронизируемых файлов

Если клиент iFolder используется с Novell iFolder 3.x, администратор iFolder может ввести ограничения на типы и максимальный размер файлов на уровне системы, пользователя или iFolder. Для Вашей учетной записи могут быть также установлены ограничения на объем пространства, которое может использоваться для данных в папках iFolder, владельцем которых Вы являетесь. Если Вы являетесь не владельцем, а членом других папок iFolder, используемое на сервере пространство папок учитывается только в ограничениях пространства владельца данной папки iFolder.

Некоторые типы файлов не предназначены для синхронизации, например, файлы операционной системы, скрытые файлы, созданные менеджером файлов, или файлы баз данных, которые реализованы как набор взаимосвязанных фалов. Вы можете включить в синхронизацию только основные типы фалов, относящиеся к Вашей работе, или исключить те файлы, которые не связаны к нею, например, файлы .mp3.


Файлы операционной системы

Вам не следует преобразовывать системные каталоги в папки iFolder. Большинство системных файлов изменяются редко, поэтому лучше скопировать файлы системы и основного программного обеспечения, чем пытаться синхронизировать эти файлы с сервером.


Скрытые файлы

Если файловая система использует скрытые для отображения на дисплее файлы, администратор может установить ограничение на файлы такого типа, чтобы исключить их синхронизацию в системе. Как правило, скрытые файлы относятся только к тому компьютеру, на котором они были созданы, и изменяются при каждом доступе к файлу или каталогу. Хранить данные файлы не требуется, а их синхронизация приведет к повторяющимся ошибкам конфликтов файлов.

Например, iFolder автоматически исключает два скрытых файла менеджера файлов с именами thumbs.db и .DS_Store.


Файлы базы данных

iFolder синхронизирует отдельные файлы или измененные части отдельных файлов и не выполняет синхронизацию файлов как набора. Если у Вас есть файл базы данных, который входит в набор связанных файлов, не пытайтесь синхронизировать их в папке iFolder.

Не пытайтесь синхронизировать данные GroupWise®, путем преобразования архива GroupWise, кэша или удаленных каталогов в папки iFolder. В противном случае после выполнения нескольких синхронизаций файлы базы данных GroupWise станут поврежденными. GroupWise требует, чтобы файлы архива обслуживались как набор файлов.
