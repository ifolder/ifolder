Рекомендации по расположению папок iFolder

Как правило, папки iFolder могут располагаться в любом месте локального жесткого диска как при создании папки iFolder, так и при установке такой папки для совместного использования. Однако существует несколько особых ограничений. При размещении папок iFolder на компьютере примите во внимание следующие рекомендации:

  • Папка iFolder должна иметь уникальное имя в каталоге, где она будет расположена.
  • В имени папки iFolder запрещено использовать следующие недопустимые символы: 
    \/:*?"<>|;

    Информацию о других соглашениях об именовании см. в разделе Соглашения об именовании папки iFolder, содержащихся в ней папок и файлов.

  • Папка iFolder должна находиться на локальном жестком диске, она не может располагаться на сетевом диске или на логическом (не физическом) диске.
  • Нельзя указывать в качестве папки iFolder корень диска.
  • Папку iFolder можно создать только на основе существующей папки.
  • Папку iFolder можно создать только на основе папки, в которой у Вас есть права доступа файловой системы на чтение и запись файлов. При установке совместно используемой папки на компьютере у Вас должны быть права доступа файловой системы на папку, в которой будет расположена папка iFolder.
  • Папка iFolder не может содержать системную папку или находиться в ней. Такой папкой, в частности, считается папка Windows или системный диск Linux либо Mac.
  • Папка iFolder не может содержать папку приложений, например, папку "Program Files", или находиться в ней.
  • Папка iFolder не может содержать папку метаданных iFolder или находиться в ней. Местоположением по умолчанию папки данных является:

    • Linux: Личная папка в подкаталоге .local\share

    • Windows: Каталог C:\Documents and Settings\WindowsLogonUsername\Local Settings\Application Data\simias

    • Macintosh: Личная папка в подкаталоге .local\share

  • Папка iFolder не может содержать другие папки iFolder или находиться в них, включая Novell® iFolder® 2.1x iFolder.
