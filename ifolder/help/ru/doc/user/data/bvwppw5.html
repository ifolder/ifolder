Конфигурирование параметров локального брандмауэра для трафика iFolder

При использовании локального брандмауэра Вы должны настроить брандмауэр так, чтобы iFolder мог выбрать локальный динамический порт, или сконфигурируйте локальный статический порт клиента iFolder для осуществления локального взаимодействия (на одном и том же компьютере) с сервером Mono XSP. Клиент iFolder использует веб-сервисы Mono XSP для взаимодействия по протоколу HTTP или HTTPS (по умолчанию SSL) с серверами Novell iFolder, на которых у пользователя есть учетные записи. Администратор iFolder конфигурирует сервисы iFolder, определяя, какое взаимодействие (HTTP или HTTPS) используется в Вашей среде.

Брандмауэр не блокирует трафик между сервером Mono XSP и сервером Novell iFolder, поскольку Mono XSP использует для трафика стандартные порты HTTP или HTTPS. Однако, если сам пользователь заблокирует трафик между клиентом и сервером Mono-XSP, у клиента не будет возможности взаимодействия с сервером. В этом случае пользователь не сможет зарегистрироваться с помощью учетной записи iFolder и синхронизировать данные iFolder.

Изначально в конфигурации iFolder не указан конкретный порт, который следует использовать для взаимодействия клиента с сервером Mono XSP. При первом запуске iFolder он открывает доступный локальный порт для прослушивания запросов с сервера Mono XSP и отправки данных на сервер с помощью Mono XSP. Номер порта записывается в конфигурационный файл, а затем этот порт всегда используется при запуске iFolder, за исключением тех случаев, когда он недоступен. Если порт используется другим приложением, iFolder выбирает следующий доступный порт, перенастраивает его в конфигурационном файле и затем использует новый порт. Например, если другое приложение захватит порт iFolder пока он неактивен, при перезагрузке клиента iFolder он должен выбрать и сконфигурировать другой порт.


Использование локального динамического порта

Если на Вашем компьютере используется локальный брандмауэр, он должен обнаружить локальный трафик между iFolder и сервером Mono XSP, возникающий при синхронизации списка пользователей и данных iFolder между клиентом iFolder и сервером. Разрешите этот трафик, чтобы при запуске клиента не отображалось всплывающее предупреждение. Если разрешить трафик на постоянной основе, Ваш брандмауэр не будет выдавать повторные напоминания.


Использование локального статического порта

При необходимости можно добавить параметр "Web Service Port Range" (Диапазон портов веб-сервиса) в раздел "Service Manager" (Менеджер сервиса) конфигурационного файла Simias, чтобы разрешить серверу Mono-XSP использовать статический порт или диапазон портов для данного локального трафика. Этот порт или диапазон портов может затем быть открыт в брандмауэре для разрешения взаимодействия с сервером Mono-XSP.

Конфигурационный файл Simias может располагаться в следующих местах:

Операционная система Расположение конфигурационного файла Simias

Linux

/home/username/.local/share/simias/Simias.config

Windows

C:\Documents and Settings\имя_пользователя\Local Settings\Application Data\simias\Simias.config

Macintosh

/Users/имя_пользователя/.local/share/simias/Simias.config

Добавьте параметр "Web Service Port Range" (Диапазон портов веб-сервиса) в раздел "Service Manager" (Менеджер сервиса) конфигурационного файла Simias. Например:

<configuration>
  <section name="ServiceManager">
    <setting name="WebServicePortRange" value="monoxsp_port" />   
  </section>
</configuration>

Замените monoxsp_port номером порта или номерами диапазона портов:

  • Укажите значение номера отдельного порта (например, 1234). Этот порт будет использоваться в качестве отдельного статического порта. Например: 
    <setting name="WebServicePortRange" value="1234" />
  • Укажите диапазон портов (например "1234-1236") для использования любого доступного порта в указанном диапазоне. Например: 
    <setting name="WebServicePortRange" value="1234-1236" />

Для конфигурирования системы с использованием определенного порта или диапазона портов выполните следующие действия:

  1. Сконфигурируйте брандмауэр таким образом, чтобы разрешить трафик порта или диапазона портов, которые нужно использовать для работы iFolder.

    Для получения информации об открытии порта в брандмауэре см. документацию по брандмауэру.

  2. Остановите клиент iFolder, щелкнув правой кнопкой значок iFolder , затем выбрав "Выход".

  3. Измените конфигурационный файл Simias путем добавления параметра "Web Service Port Range" (Диапазон портов веб-сервиса) в раздел "Service Manager" (Менеджер сервиса).

  4. Запустите клиент iFolder.

    Дополнительную информацию см. в разделе Запуск клиента iFolder.
