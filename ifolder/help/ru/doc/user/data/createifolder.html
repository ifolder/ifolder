Создание папки iFolder

Можно создать папку iFolder, преобразуя в нее существующую папку, если она удовлетворяет следующим принципам:

  • Рекомендации по расположению папок iFolder
  • Соглашения об именовании папки iFolder, содержащихся в ней папок и файлов

Если папка не существует, можно найти требуемое расположение и создать новую папку.

В каталоге iFolder можно создать любую структуру подкаталогов в соответствии с личными или корпоративными потребностями. Структура подкаталогов одинакова для всех членов папок iFolder. На каждой рабочей станции можно указать различный родительский каталог для совместно используемой папки iFolder.

Можно создавать папки iFolder непосредственно в менеджере файлов и в средстве просмотра iFolder, которое входит в состав приложения клиента. Для создания папки iFolder воспользуйтесь следующими способами:

  • Создание папок iFolder в менеджере файлов
  • Создание папок iFolder в средстве просмотра iFolder
  • Создание папок iFolder в точке монтирования FAT32 (Linux)


Создание папок iFolder в менеджере файлов

  1. С помощью менеджера файлов найдите папку, которую нужно преобразовать в папку iFolder, затем выберите ее.

  2. Воспользуйтесь одним из следующих способов:

    • Щелкните правой кнопкой мыши значок папки, затем выберите "Преобразовать в папку iFolder".
    • На панели инструментов щелкните "Файл" > "Преобразовать в папку iFolder".

    Значок папки сменится значком папки со значком iFolder .

  3. При появлении сообщения "Папка iFolder создана" выполните одно из следующих действий:

    • Щелкните "Закрыть", чтобы закрыть сообщение.
    • Выберите "Не выводить больше это сообщение", чтобы запретить дальнейшие оповещения, затем щелкните "Закрыть", чтобы закрыть сообщение.

      Если в дальнейшем потребуется получать сообщения о подтверждении, перейдите в меню "Настройки iFolder", выберите "Отображение диалогового окна "Подтверждение" при создании iFolder", затем нажмите "OK".


Создание папок iFolder в средстве просмотра iFolder

  1. В области оповещения щелкните правой кнопкой значок iFolder , затем щелкните "Папки iFolder" для открытия средства просмотра iFolder.

  2. Щелкните "Создать".

  3. Укажите учетную запись iFolder, которая будет использоваться для этой папки iFolder.

    При наличии нескольких учетных записей iFolder первой будет отображена учетная запись по умолчанию. При необходимости выберите другую учетную запись в раскрывающемся списке.

  4. В средстве просмотра найдите каталог, в котором нужно создать папку iFolder.

  5. Выберите обычную папку, в которой нужно создать папку iFolder. Для этого воспользуйтесь одним из следующих способов:

    • Если обычная папка существует, выберите ее, затем нажмите "OK".
    • Если обычная папка не существует, щелкните кнопку создания новой папки, чтобы создать папку, выберите ее, затем нажмите "OK".

  6. Щелкните кнопку "OK".

    Папка iFolder появится в списке папок iFolder.

  7. При появлении сообщения "Папка iFolder создана" выполните одно из следующих действий:

    • Щелкните "Закрыть", чтобы закрыть сообщение.
    • Выберите "Не выводить больше это сообщение", чтобы запретить дальнейшие оповещения, затем щелкните "Закрыть", чтобы закрыть сообщение.

      Если в дальнейшем потребуется получать сообщения о подтверждении, перейдите в меню "Настройки iFolder", выберите "Отображение диалогового окна "Подтверждение" при создании iFolder", затем нажмите "OK".


Создание папок iFolder в точке монтирования FAT32 (Linux)

В системе Linux можно создавать и использовать папки iFolder в точке монтирования для файловой системы FAT32. Перед созданием новых папок iFolder или настройкой совместного использования папок iFolder на томе FAT32 компьютера Linux пользователь iFolder должен быть владельцем точки монтирования FAT32, в которой будут храниться папки iFolder, а в файле /etc/fstab должно находиться явное указание идентификатора локального пользователя iFolder для точки монтирования.

ВАЖНО.  Это необходимо, так как файловая система FAT32 не поддерживает разрешения файлов. Данная проблема характерна для всех файловых систем, которые не поддерживают разрешения файлов.

При наличии двойной загрузочной системы можно получать доступ к данным в папках iFolder как к данным обычных папок из системы, в которой эта папка не настроена как папка iFolder. Если Вы установили папку iFolder на обеих платформах, набор данных хранится в двух различных папках iFolder на сервере, которые тем самым дважды расходуют дисковое пространство сервера.

Так как файловая система FAT32 не поддерживает разрешения файловой системы POSIX, убедитесь, что пользователь iFolder является локальным владельцем точки монтирования. Измените файл /etc/fstab, чтобы установить параметр идентификатора пользователя (UID) для локальной учетной записи пользователя iFolder, который создает папки iFolder для этой точки монтирования.

  1. Зарегистрируйтесь на компьютере в качестве пользователя "root".

  2. Создайте каталог, в котором нужно смонтировать файловую систему FAT32. В командной строке введите команду

    mkdir /home/имя_пользователя/fsmount

    Вместо параметра имя_пользователя укажите имя пользователя, который будет использовать диск FAT32 для папок iFolder. Вместо параметра fsmount укажите имя каталога, который должен использоваться в качестве точки монтирования. Например, если именем пользователя является jsmith, а именем каталога является fat32mntpt для диска FAT32 или раздела, введите команду

    mkdir /home/jsmith/fat32mntpt

  3. Смонтируйте диск FAT32 в качестве требуемой точки монтирования. В командной строке введите команду

    mount -t vfat /dev/hda1 /home/имя_пользователя/fsmount

    Вместо /dev/hda1 укажите имя устройства или раздела для не принадлежащего Linux устройства. Вместо параметра имя_пользователя укажите имя пользователя iFolder. Вместо параметра fsmount укажите имя каталога, который должен использоваться в качестве точки монтирования.

    Таким образом, если имеются файловая система FAT32 (vfat), которая находится на устройстве hda4 (/dev/hda4), имя пользователя jsmith и каталог точки монтирования /home/jsmith/fat32mntpt, введите команду

    mount -t vfat /dev/hda4 /home/jsmith/fat32mntpt

  4. Измените файл /etc/fstab, указав в нем параметр uid=имя_пользователя_iFolder в строке, которая определяет точку монтирования FAT32.

    /dev/hda1 /home/имя_пользователя/fsmount vfat uid=имя_пользователя

    В данном примере измените файл /etc/fstab, добавив параметр uid=jsmith в точку монтирования:

    /dev/hda4  /home/jsmith/fat32mntpt  vfat  uid=jsmith
