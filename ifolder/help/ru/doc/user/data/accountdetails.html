Просмотр и изменение настроек учетной записи iFolder

  1. Щелкните правой кнопкой значок iFolder в области оповещения, затем выберите "Учетные записи" для открытия "Настройки iFolder" на вкладке "Учетные записи".

  2. Выберите учетную запись для просмотра и изменения следующих параметров:

    Параметр Описание

    Сервер

    DNS-имя сервера. Например: svr21.example.com.

    Имя пользователя

    Имя пользователя для этой учетной записи. Администратор iFolder определяет, какой формат необходимо использовать: ИД пользователя сети (общее имя LDAP) или адрес электронной почты.

    Например, для пользователя John Smith с сетевым именем jsmith и адресом электронной почты john.smith@example.com, администратор настраивает параметр для сервера, который определяет, какой из двух форматов будет использоваться в поле "Имя пользователя" для аутентификации.

    Вы можете создать только одну учетную запись для сервера.

    Пароль

    Пароль, соответствующий имени пользователя.

    Запомнить пароль

    Выберите "Запомнить пароль", чтобы при каждой Вашей регистрации в компьютере автоматически использовалась данная учетная запись iFolder. Для получения дополнительной информации см. раздел Регистрация с помощью учетной записи iFolder.

    После смены сетевого пароля iFolder при следующей попытке регистрации с помощью учетной записи iFolder произойдет сбой автоматической аутентификации. Программа iFolder запросит Вас зарегистрироваться с новым паролем и позволит определить и сохранить его.

    Разрешение учетной записи

    Выберите "Разрешение учетной записи" для выполнения фоновой синхронизации приглашений iFolder, списка пользователей и папок iFolder для данной учетной записи. Этот параметр применяется только к функциям клиента.

    В столбце "Состояние" списка "Учетные записи" указывается состояние учетной записи: "Разрешена" или "Запрещена". Вы можете сразу просмотреть состояние параметра "Разрешение учетной записи", если у Вас создано несколько учетных записей.

    При использовании iFolder с Novell® iFolder® 3.x администратор iFolder может запретить (как правило, временно) учетную запись iFolder на сервере. В этом случае можно выполнить следующие действия:

    • Если учетная запись запрещена на сервере после того, как Вы зарегистрировались, можно по-прежнему управлять папками iFolder, а данные iFolder будут по-прежнему синхронизироваться с сервером до тех пор, пока не будет разрегистрирован Ваш сеанс, отключена синхронизация или пока сервер не отсоединит Ваш сеанс в зависимости от того, что произойдет раньше.
    • Если учетная запись запрещена на сервере, при попытке зарегистрироваться Вы получите всплывающее сообщение, что Ваша учетная запись запрещена. Обратитесь за помощью к администратору iFolder.

    Учетная запись по умолчанию

    Выберите "Учетная запись по умолчанию", чтобы данная учетная запись выбиралась по умолчанию в раскрывающемся списке учетных записей при создании папок iFolder.

  3. На вкладке "Учетные записи" в диалоговом окне "Настройки iFolder" выберите учетную запись iFolder, затем щелкните "Подробная информация" для просмотра указанной ниже информации о Вашей учетной записи.

    Параметр Описание

    Сервер

    Имя хоста сервера. Например, если DNS-имя сервера "svr21.example.com", то в отчете будет указан "svr1".

    Описание

    Удобное для пользователя описание корпоративного сервера, простого сервера или компьютера хоста рабочей группы, указанное администратором. Если описание не было введено, используется значение "отсутствует".

    Доступно

    Общее количество пространства, доступного в данный момент для папок iFolder в этой учетной записи iFolder. Это значение равно квоте за вычетом уже использованного пространства.

    Используется

    Общее количество пространства, использованного в данный момент всеми папками iFolder, владельцем которых Вы являетесь в этой учетной записи.

    Квота

    Общее количество пространства, выделенного для Вашей учетной записи на сервере. Данное значение может быть квотой, устанавливаемой по умолчанию для всех пользователей, или квотой для отдельного пользователя. Если квота не установлена, значение является пустым или обозначается как Н/Д (недоступно).
