Indicazioni per l'ubicazione e l'uso di cartelle iFolder

In genere, quando si crea una cartella iFolder o se ne configura una condivisa da altri utenti è possibile collocarla in qualsiasi ubicazione del disco rigido locale. Tuttavia esistono alcune limitazioni pratiche e specifiche. Tenere presenti le seguenti indicazioni relative all'ubicazione e all'uso di cartelle iFolder nel computer:

  • Il nome di una cartella iFolder deve essere univoco nella directory in cui viene collocata la cartella.

  • Il nome di una cartella iFolder non può contenere i seguenti caratteri non validi:

    \/:*?"<>|;

    Per altre convenzioni di denominazione, vedere Convenzioni di denominazione per le cartelle iFolder e i relativi file e cartelle.

  • La cartella iFolder deve risiedere su un disco rigido locale; l’ubicazione non può trovarsi su un’unità di rete o non fisica.

  • Una cartella iFolder non può trovarsi alla radice dell’unità disco rigido.

  • Una cartella iFolder può essere creata solo da una cartella per la quale si dispone dei diritti di accesso in lettura e scrittura dei file. Quando si configura una cartella iFolder condivisa in un computer, è necessario disporre dei diritti di accesso di file system per la cartella in cui si colloca la cartella iFolder.

  • Una cartella iFolder non può contenere o trovarsi in una cartella di applicazioni, ad esempio la cartella Programmi.

  • La cartella iFolder non può contenere o trovarsi nella cartella dei metadati iFolder. L'ubicazione di default della cartella di metadati è la seguente:

    • Linux: nella cartella home nella sottodirectory <Home Directory>\.local\share\simias

    • Windows: nella directory C:\Documents and Settings\nomeutente_accesso_Windows\Impostazioni locali\Dati applicazione\simias

    • Macintosh: in $HOME/.local/share/simias

  • Una cartella iFolder non può contenere altre cartelle iFolder o trovarsi in esse, compresa la cartella iFolder di Novell iFolder 2.x.

    Non sono supportate cartelle iFolder di rete. Ad esempio, non è possibile creare cartelle iFolder associate alle directory di rete /home degli utenti mappate alle relative workstation. Non è inoltre supportato l'accesso WebDAV dal desktop di un utente alla raccolta di cartelle iFolder sul volume di dati utente del server iFolder.

  • Sebbene sia possibile eseguire il login a più conti iFolder contemporaneamente, una cartella iFolder può essere associata a un solo conto. Non è possibile sincronizzare una cartella iFolder con più conti. Quando si crea una cartella iFolder, essa può essere condivisa solo con altri utenti e gruppi LDAP di cui è stato eseguito il provisioning nello stesso server o dominio di iFolder.

  • Se condivisa con un gruppo LDAP, la cartella iFolder è disponibile per tutti i membri del gruppo.

  • Se più utenti condividono un computer, la riservatezza delle copie locali delle cartelle iFolder memorizzate nello spazio su disco allocato a un utente è garantita solo se il file system supporta le funzioni di riservatezza dei dati e tali funzioni sono configurate per l'uso.

    In Windows, il file system NTFS include funzioni per la sicurezza quali le autorizzazioni per file e cartelle e il sistema EFS (Encrypting File System), utilizzato per memorizzare file cifrati in volumi NTFS. Queste funzioni non sono supportate nei file system FAT32. Accertarsi che le identità di login locale degli utenti non dispongano dei diritti di amministratore o di diritti equivalenti, a eccezione ovviamente dell'identità dell'amministratore. L'amministratore può accedere a tutti i file e a tutte le directory del computer. Nel caso di computer condivisi, dove la sicurezza e la riservatezza dei dati rivestono un'importanza particolare per gli utenti, questi ultimi possono cifrare la copia locale dei dati iFolder personali tramite EFS o tecnologie per la cifratura del file system di terze parti.

    I file system Linux e Macintosh sono progettati per l'uso in ambienti multiutente. Le autorizzazioni per i file consentono di controllare l'accesso di utenti, gruppi o altre identità. Le autorizzazioni per i file di proprietà di un utente sono tipicamente impostate per consentire l'accesso completo al proprietario e non consentire alcun accesso ai gruppi o ad altre identità.

    La memorizzazione di cartelle iFolder in un volume FAT32 locale in Linux richiede particolare attenzione poiché i file system FAT32 non sono conformi allo standard POSIX. Per ulteriori informazioni, vedere il Creazione di cartelle iFolder in un punto di montaggio FAT32 (Linux).

  • Se gli utenti dello stesso computer condividono cartelle iFolder, nel dispositivo possono coesistere più copie delle cartelle e ciascun utente dispone di una copia scaricata dal server. Identità di login locale diverse non possono condividere la stessa ubicazione per le cartelle iFolder condivise.

Per ulteriori informazioni, vedere Note legali.
