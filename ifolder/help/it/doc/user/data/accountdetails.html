Visualizzazione e modifica delle impostazioni del conto iFolder

  1. Fare clic con il pulsante destro del mouse sull'icona iFolder nell'area di notifica e scegliere Conti per aprire la scheda Conti della finestra di dialogo Preferenze di iFolder.

  2. Selezionare il conto per visualizzarne o modificarne i parametri seguenti:

    Parametro Descrizione

    Server

    Nome DNS del server aziendale. Ad esempio: svr21.esempio.com.

    Nome utente

    Identità dell'utente associata la conto. Il formato da utilizzare può corrispondere all'ID utente di rete (nome comune LDAP) o all'indirizzo di e-mail, a seconda dell'impostazione definita dall'amministratore iFolder.

    Nel caso, ad esempio, dell'utente Mario Rossi al quale sono associati l'ID utente mrossi e l'indirizzo di e-mail mario.rossi@esempio.com, l'amministratore configura un'impostazione del server in base alla quale viene definito quale dei due formati deve essere accettato nel campo Nome utente ai fini dell'autenticazione.

    È possibile creare un solo conto per ogni server aziendale.

    Parola d'ordine

    Parola d'ordine associata al nome utente.

    Memorizza parola d'ordine

    Selezionare questa opzione per eseguire il login al conto iFolder automaticamente a ogni accesso al computer. Per informazioni, vedere Login a un conto iFolder.

    Se la parola d'ordine di rete dell'utente viene modificata, al successivo tentativo di login al conto iFolder l'autenticazione automatica ha di conseguenza esito negativo. Verrà pertanto richiesto di eseguire il login con la nuova parola d'ordine e sarà possibile digitarla e salvarla.

    Abilita conto

    Selezionare l'opzione Abilita conto per consentire la sincronizzazione in background degli inviti di iFolder, dell'elenco degli utenti e delle cartelle iFolder relativi al conto. L'opzione consente di controllare le funzioni soltanto sul lato client.

    Lo stato del conto, Abilitato o Disabilitato, viene indicato nella colonna Stato dell'elenco Conti. Se si dispone di più conti, è possibile visualizzarne lo stato di abilitazione generale.

    Se si utilizza iFolder con Novell® iFolder® 3.x, il conto dell'utente può essere disabilitato (in genere temporaneamente) dall'amministratore del sistema iFolder sul lato server. In tal caso, possono verificarsi le condizioni seguenti:

    • Se è già stato eseguito il login al momento della disabilitazione del conto sul lato server, l'utente potrà continuare comunque a gestire le cartelle iFolder e la sincronizzazione dei dati con il server non verrà interrotta fino al termine della sessione che può essere determinato dal logout, dalla disabilitazione della sincronizzazione o dalla disconnessione da parte del server.
    • Se il conto viene disabilitato sul lato server, verrà visualizzato un messaggio popup di notifica al successivo tentativo di login. Contattare l'amministratore del sistema iFolder per assistenza.

    Conto di default

    Selezionare questa opzione per impostare il conto come default nell'elenco a discesa dei conti quando si procede alla creazione di cartelle iFolder.

  3. Nella scheda Conti della finestra Preferenze di iFolder, selezionare il conto iFolder e fare clic su Dettagli per visualizzare le informazioni seguenti:

    Parametro Descrizione

    Server

    Nome host del server aziendale. Se ad esempio il nome DNS del server è svr21.esempio.com, verrà indicato svr21.

    Descrizione

    Descrizione del server aziendale, del server semplice o del computer host del gruppo di lavoro digitata dall'amministratore. Se non è stata digitata alcuna descrizione, viene visualizzato il valore "nessuna".

    Disponibile

    La quantità totale di spazio attualmente allocata all'utente per le cartelle iFolder associate al conto. Il valore corrisponde alla quota meno lo spazio utilizzato.

    Usato

    La quantità totale di spazio attualmente occupata dalle cartelle iFolder associate al conto utente.

    Quota

    La quantità totale di spazio allocata al conto iFolder nel server aziendale. Il valore può corrispondere alla quota di default impostata per tutti gli utenti o a una quota specifica per l'utente. Se non è configurata alcuna quota, il valore è assente o non applicabile (N/A).
