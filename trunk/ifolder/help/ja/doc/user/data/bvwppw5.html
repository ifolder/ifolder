iFolderトラフィックに対するローカルファイアウォールの設定

ローカルファイアウォールを使用する場合は、iFolderクライアントがMono XSP Serverとローカルに(同じコンピュータで)通信するために、iFolderがローカルダイナミックポートを選択したり、ローカルスタティックポートを設定したりできるようなファイアウォールを構成する必要があります。iFolderは、Mono XSP Webサービスを使用して、HTTPまたはHTTPS(デフォルトはSSL)経由でiFolderアカウントが存在するNovell iFolderエンタープライズサーバと通信します。iFolderの管理者は、ユーザ環境でHTTPとHTTPSのどちらの通信を使用するかを決定するiFolderサービスを設定します。

Mono XSPはトラフィックに対して標準的なHTTPまたはHTTPSポートを使用するので、お使いのファイアウォールが、Mono XSPとNovell iFolderエンタープライズサーバ間のトラフィックをブロックすることはありません。ただし、クライアントとMono-XSP間のトラフィックをブロックすると、クライアントはサーバと通信する手段を失います。これにより、iFolderアカウントにログインしてiFolderのデータを同期することができなくなります。

最初は、iFolder環境設定でクライアントとMono XSPとの通信で使用する特定のポートを指定しません。iFolderが最初に実行されるとき、Mono XSPからの要求をリッスンするために使用可能なローカルポートが開かれ、Mono XSP経由でサーバにデータを送信します。そのポート番号が環境設定ファイルに書き込まれ、それ以降iFolderを開始するときは、ポートが使用不可でない限り、常にこのポートが使用されます。そのポートが別のアプリケーションで使用されている場合、iFolderは使用可能な別のポートを選択し、環境設定ファイルにそのポートを再設定し、以降はその新しいポートを使用します。たとえば、iFolderが停止し、別のアプリケーションがそのポートを使用する場合、iFolderを再起動したときに、iFolderで別のポートを選択して使用するように設定する必要があります。


ローカルダイナミックポートの使用

お使いのコンピュータでローカルファイアウォールを使用する場合、iFolderがサーバに接続してユーザのリストとiFolderデータを同期するときは、ファイアウォールは常にiFolderとMono XSPサーバ間のローカルトラフィックを検出する必要があります。クライアントが起動する際のポップアップアラートをトラフィックが受信しないようにすることができます。トラフィックを永続的に設定すれば、ファイアウォールは二度とアラートを送信しません。


ローカルスタティックポートの使用

オプションで、Simias.configファイルのService ManagerセクションにWeb Service Port Rangeパラメータを追加して、Mono XSP サーバがこのローカルトラフィック用にスタティックポートまたは一定の範囲のポートを使用するようにできます。これにより、ファイアウォールでこのポートまたは一定の範囲のポートが開かれ、Mono XSPとの通信が可能になります。

Simias.configファイルは、次の場所にあります。

オペレーティングシステム Simias.configファイルの場所

Linux

/home/username/.local/share/simias/Simias.config

Windows

C:\Documents and Settings\username\Local Settings\Application Data\simias\Simias.config

Macintosh

/Users/username/.local/share/simias/Simias.config

Simias.configファイルのService Managerセクションに、Web Service Port Range設定を追加します。次に例を示します。

<環境設定>
  <section name="ServiceManager">
    <setting name="WebServicePortRange" value="monoxsp_port" />   
  </section>
</configuration>

monoxsp_portを、ポート番号またはポート番号の範囲で置換します。

  • 単一のスタティックポートを使用する場合は、単一の4桁のポート番号の値(1234など)を指定します。次に例を示します。
    <setting name="WebServicePortRange" value="1234" />
  • 指定したポートの範囲内で可能なポートを使用する場合は、ポートの範囲("1234-1236"など)を指定します。次に例を示します。
    <setting name="WebServicePortRange" value="1234-1236" />

システムが特定のポートまたはポート範囲を使用するように設定するには

  1. ファイアウォールを、iFolderトラフィックで使用するポートまたはポート範囲でのトラフィックを許可するように設定します。

    ファイアウォールのポートを開く方法については、お使いのファイアウォールのマニュアルを参照してください。

  2. iFolderアイコンを右クリックしてから［終了］を選択します。

  3. Service ManagerセクションにWeb Service Port Rangeパラメータを追加して、Simias.configファイルを変更します。

  4. iFolderクライアントを起動します。

    詳細については、「iFolderクライアントの起動」を参照してください。
