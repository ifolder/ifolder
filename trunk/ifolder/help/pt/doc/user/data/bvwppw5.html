Configurando as definições de firewall local para o tráfego do iFolder

Se usar um firewall local, você deverá definir o seu firewall para permitir que o iFolder escolha uma porta dinâmica local ou configure uma porta estática local para que o cliente do iFolder se comunique localmente (o mesmo computador) com o Servidor Mono XSP. O iFolder utiliza os serviços Web do Mono XSP para se comunicar via HTTP ou HTTPS (SSL, padrão) com os servidores empresariais Novell iFolder onde você tem as suas contas do iFolder. O administrador do iFolder configura os serviços iFolder que determinam se as comunicações utilizam HTTP ou HTTPS em seu ambiente.

O seu firewall não bloqueia o tráfego entre o Mono XSP e o servidor empresarial Novell iFolder porque o Mono XSP utiliza as portas HTTP ou HTTPS padrão para o tráfego. Porém, se você bloquear o tráfego entre o cliente e o Mono-XSP, o cliente não poderá se comunicar com o servidor. Você não conseguiria efetuar login em sua conta do iFolder e sincronizar dados do iFolder.

Inicialmente, a configuração do iFolder não especifica uma porta particular a ser usada para as comunicações cliente-a-Mono XSP. Quando o iFolder é executado pela primeira vez, ele abre uma porta local disponível para escutar as solicitações do Mono XSP e enviar dados para o servidor via Mono XSP. Ele grava o número da porta em um arquivo de configuração e utiliza essa porta posteriormente, a não ser que ela não esteja disponível quando o iFolder for iniciado. Se a porta estiver sendo usada por outro aplicativo, o iFolder seleciona uma porta diferente disponível, reconfigura a porta no arquivo de configuração e utiliza a nova porta posteriormente. Por exemplo, se o iFolder estiver desativado e outro aplicativo utilizar a porta, quando você reiniciar o iFolder, o iFolder deverá selecionar e configurar uma porta diferente a ser usada.


Utilizando uma porta dinâmica local

Se você usar um firewall local no computador, o firewall deverá detectar o tráfego local entre o iFolder e o servidor Mono XSP sempre que o iFolder entrar em contato com o servidor para sincronizar suas listas de usuários e dados do iFolder. Permita que o tráfego evite o recebimento do alerta pop-up sempre que o cliente for iniciado. Se você permitir o tráfego permanentemente, o firewall não deverá lembrá-lo novamente.


Utilizando uma porta estática local

Você pode adicionar opcionalmente o parâmetro Faixa de Porta de Serviço Web na seção Gerenciador de Serviços do arquivo Simias.config para permitir que o servidor Mono-XSP utilize uma porta estática ou uma faixa de portas para este tráfego local. Essa porta ou faixa de portas pode ser aberta no firewall para permitir as comunicações para o Mono-XSP.

Você pode encontrar o arquivo Simias.config nos seguintes locais:

Sistema Operacional Localização do arquivo Simias.config

Linux

/home/username/.local/share/simias/Simias.config

Windows

C:\Documents and Settings\username\Local Settings\Application Data\simias\Simias.config

Macintosh

/Users/username/.local/share/simias/Simias.config

Adicione a Configuração Web Service Port Range (Faixa de Porta de Serviço Web) à seção Service Manager (Gerenciador de Serviços) do arquivo Simias.config. Por exemplo:

<configuration>
  <section name="ServiceManager">
    <setting name="WebServicePortRange" value="monoxsp_port" />   
  </section>
</configuration>

Substitua monoxsp_port por um número de porta ou faixa de números de porta:

  • Especifique um valor de um único número de quatro dígitos (como 1234) para utilizar uma única porta estática. Por exemplo: 
    <setting name="WebServicePortRange" value="1234" />
  • Especifique uma faixa de portas (como "1234-1236") para usar qualquer porta disponível na faixa especificada de portas. Por exemplo: 
    <setting name="WebServicePortRange" value="1234-1236" />

Para configurar o seu sistema para usar uma porta específica ou uma faixa de portas:

  1. Configure seu firewall para permitir tráfego para a porta ou faixa de portas que deseja usar para o tráfego do iFolder.

    Consulte a documentação do seu firewall para obter informações sobre como abrir uma porta no firewall.

  2. Interrompa o cliente iFolder clicando o botão direito do mouse no ícone do iFolder e, em seguida, selecione Sair.

  3. Modifique o arquivo Simias.config adicionando o parâmetro Web Service Port Range à seção Service Manager.

  4. Inicie o cliente iFolder.

    Para obter informações, consulte Iniciando o Cliente iFolder.
