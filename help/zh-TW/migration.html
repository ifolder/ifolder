從 iFolder 2.x 移轉至 iFolder 3.7

本節討論以下各項：

  • 瞭解移轉處理程序

  • 移轉程序

瞭解移轉處理程序

Novell iFolder 提供簡單移轉方式，從 iFolder 2.x 移轉至 Novell iFolder 3.7 的 iFolder 用戶端。iFolder 用戶端中所提供的「移轉助理」可協助您輕鬆快速地移轉現有的 iFolder 2.x 資料。

在啟動期間，「移轉助理」會偵測 2.x 的安裝，並提示您移轉資料。在資料移轉期間，它會再次提示您轉換現有的資料夾或者製作 2.x iFolder 的複本。如需有關此作業的詳細資訊，請參閱 步驟 6。

iFolder 會提示您重新同步化檔案至 3.7 伺服器，以及是否需要保留 2.x 用戶端直到 3.7 完成同步化的任務。在此程序中，只有資料會移轉，因此使用者必須在新用戶端中設定優先設定。

移轉程序

  1. 安裝適用於 iFolder 3.7 的 iFolder 用戶端，然後設定您的 iFolder 3.7 Enterprise Server 帳戶。

    如需有關安裝用戶端和設定帳戶的詳細資訊，請參閱 管理 iFolder 帳戶及優先設定。

  2. 在 iFolder 瀏覽器中，在您的本機電腦上建立一個或以上的 iFolder。

    您可以為 iFolder 使用任何有效名稱。請勿在與您目前 iFolder 2.x 目錄相同的目錄路徑中建立 iFolder。如需更多資訊，請參閱建立和上傳 iFolder。

  3. iFolder 會顯示關於存在舊版資料的訊息，以及移轉的申請。

    按一下「確定」以開啟可用 2 的清單。x iFolder 的清單。

  4. 選取您要移轉的 2.您要移轉的 x iFolder。

    按一下「移轉」以顯示「移轉助理」。

    附註：您也可以透過「iFolder」 > 「Migration」 > 「從 iFolder 2.x 移轉」前往「移轉助理」。

    「移轉助理」會帶領您完成資料移轉的「移轉」程序。

  5. 按一下「轉遞」以顯示移轉選項。

  6. 選取下列其中一個：

    移轉 iFolder 並解除連接 2.x 網域： 選取此選項以便解除連接現有資料夾與 iFolder 2。x 領域。

    建立 iFolder 的副本並移轉： 選取此選項以建立 iFolder 2 的副本。x 檔案的副本，並將檔案轉換成 iFolder 3.7。這並不會從 ifolder 2.x 領域移除檔案，即使您已經將檔案移轉到 iFolder 3.7。

    如果您選取此選項，您還可以選取複製上層資料夾，只要您不只是想要複製 2.x 資料夾的內容，還想複製上層資料夾。如果您保持不選取這個選項，則只會將選取資料夾的內容移轉到 3.7。

    按一下「瀏覽」以尋找位置儲存移轉的資料夾。

    例如，c:/Documents and Settings/<user name>/ifolder2dir 是個 2.x iFolder 且 dir1 和 dir2 是 ifolder2dir 之下的目錄。當您選取選項「複製上層目錄」時，它會在新位置及 dir1 和 dir2 之複製目的地目錄中建立 ifolder2dir 。如果保持不選取這個選項，只有 dir1 和 dir2 目錄會移動到指定的位置。

  7. 按一下「轉遞」選取領域和「保全性」選項，如下所示：

    1. 從下拉式清單中選取領域。

    2. 選取所要的保全性選項：

      • 按一下「加密」以加密所選取的 iFolder。

        如果在登入時沒有設定通關密語，iFolder 會提示您輸入通關密碼。您可以略過此選項，稍候再設定通關密語。

        如果您目前未連接至 iFolder 領域，系統會提示您連接至領域。

      • 選取「共用」與 iFolder 3.7 領域中的其他成員共用 iFolder。

        您可以將加密的 2.x iFolder 移轉到共用資料夾。當您嘗試移轉加密的 2.x iFolder 到共用資料夾，會提示您確認動作。

        按一下「確定」，確認您的動作。

  8. 按一下「轉遞」移轉選取的 iFolder。

  9. 確認您輸入的資訊是否正確，如果必要，請按一下「上一步」以返回前一頁做更正。

    如果提供的資訊正確，則按一下「移轉」開始進行移轉。

    按一下「上一步」以返回並進行任何必要的變更。

  10. 按一下「完成」以完成移轉。

有關更詳細的資訊，請參閱 法律聲明.
