建立和上傳 iFolder

您可以藉由轉換符合下列指示的現有資料夾來建立 iFolder：

  • iiFolder 位置和使用指南

  • iFolder 及其資料夾和檔案的命名慣例

如果該資料夾不存在，您可以瀏覽至某個位置並建立新資料夾作為程序的一部分。

重要：您只能共用一般 iFolder。加密的 iFolders 無法共用。

在 iFolder 中，您可以設定任何次目錄結構，配合您個人或公司的工作習慣。 該次目錄結構在所有成員 iFolder 之間均保持不變。 每個工作站都可以為共用 iFolder 指定不同的父目錄。

當您超過個別使用者可以建立的 iFolder 數量限制時，iFolder 會顯示錯誤訊息。例如，如果您的管理員將「每位使用者的 iFolder 數目」規則設定為 10，而您試圖建立第十一個 iFolder，您會收到違反規則通知。

Linux

您可以使用檔案管理員或可在用戶端應用程式中使用的 iFolder 瀏覽器，建立 iFolder。使用下列方式建立 iFolder：

  • 在 iFolder 瀏覽器中建立 iFolder

  • 在檔案管理員中建立 iFolder

在 iFolder 瀏覽器中建立 iFolder

  1. 在通知區域中，使用下列其中一個方法開啟 iFolder 瀏覽器：

    • 按一下「iFolder 服務」圖示。

    • 以滑鼠右鍵按一下「iFolder 服務」圖示，然後按一下「iFolder」。

  2. 按一下「建立 iFolder」。

  3. 在「更多選項」之下，指定這個 iFolder 您要使用的「iFolder 帳戶」。

    如果您有多個 iFolder 帳戶，則預設為列出的第一個帳戶。 使用下拉式清單以選取其他帳戶。

  4. 在瀏覽器中，導覽至要建立 iFolder 的目錄。

  5. 選取您要轉換為 iFolder 的一般資料夾，然後使用下列方式之一：

    • 如果一般資料夾存在，則選取該資料夾，然後按一下「新增」。

    • 如果不存在一般資料夾，按一下「建立資料夾」以建立資料夾，選取新建立的資料夾，然後按一下「新增」。

  6. 從提供的選項中選取 iFolder 「類型」。您可以決定要建立加密的 iFolder 或一般 iFolder。

    重要：只有 iFolder 管理員將您的加密規則設定為「開啟」時，您才可以建立加密的 iFolder。如果沒有，您只能建立一般 iFolder，而且預設會選取這個選項。在這個情況下，會關閉「類型」的兩個選項。

    • 選取「通關密語加密」啟用選取的 iFolder 之加密功能。

      如果您在登入會期中尚未設定通關密語，而且選取「通關密語加密」，系統會提示您輸入通關密語。在「通關密語」對話方塊中指定通關密語。

    • 選取「一般」啟用選取之 iFolder 的共用功能。

      iFolder 會在通知區域以立現式訊息通知共用 iFolder 的可用性。共用的使用者可以按一下該訊息在這個時候設定 iFolder，或關閉訊息並於稍後下載 iFolder。

  7. 選取「安全同步」使您建立的 iFolder 中的檔案能夠進行安全同步。

    您可以為一般和加密的 iFolder 啟用安全通道。選取這個選項後，iFolder 可讓您使用安全通道進行伺服器和本地電腦上 iFolder 間的同步。

    附註：您也可以用滑鼠右鍵按一下 iFolder 主視窗中的「iFolder」>「內容」選項，為電腦上選取的 iFolder 啟用安全同步。要執行此操作，請以滑鼠右鍵按一下 iFolder，並選取「內容」，然後在「一般」索引標籤中選取「安全同步」核取方塊。

  8. 按一下「確定」。

    iFolder 會出現在「此電腦上的 iFolder」清單中。

  9. 如果出現「iFolder 已建立」訊息，請執行下列其中一項：

    • 按一下「關閉」，關閉該訊息。

    • 選取「不要再顯示這個訊息」以停用未來的通知，然後按一下「關閉」，關閉該訊息。

      如果稍後您希望獲得該確認訊息，請移至「iFolder 優先設定」，選取「建立 iFolder 時顯示確認對話方塊」，然後按一下「確定」。

在檔案管理員中建立 iFolder

  1. 使用檔案管理員導覽至要轉換為 iFolder 的資料夾，然後選取該資料夾。

  2. 請使用下列其中一個方法：

    • 以滑鼠右鍵按一下資料夾圖示，然後按一下「轉換為 iFolder」。

    • 在工具列上，按一下「檔案」 > 「轉換成 iFolder」。

    資料夾圖示變更成重疊 iFolder 徽章 (綠色的 “i”) 的資料夾。

  3. 如果出現「iFolder 已建立」訊息，請執行下列其中一項：

    • 按一下「關閉」，關閉該訊息。

    • 選取「不要再顯示這個訊息」關閉未來的通知，然後按一下「關閉」離開訊息。

      如果稍後您希望獲得該確認訊息，請移至「iFolder 優先設定」，選取「建立 iFolder 時顯示確認對話方塊」，然後按一下「確定」。

跨平台用戶端

您可以使用用戶端應用程式中提供的 iFolder 瀏覽器建立 iFolder。

  1. 從固定點按一下「iFolder 服務」圖示開啟 iFolder 瀏覽器。

  2. 使用下列其中一個方法開啟瀏覽器。

    • 按一下工具列中的「新增」

    • 按一下功能表列中的「檔案」>「新增」

  3. 在瀏覽器中，導覽至要建立 iFolder 的目錄。

  4. 選取要改成 iFolder 的一般資料夾，然後按一下「開啟」。

  5. 從下列提供的選項中選取 iFolder 類型：

    重要：只有 iFolder 管理員將您的加密規則設定成「開啟」之後，您才可以建立加密的 iFolder。如果沒有，您只能建立一般 iFolder，而且預設會選取這個選項。在這個情況下，會關閉「類型」的兩個選項。

    • 選取「通關密語加密」啟用選取的 iFolder 之加密功能。

      如果管理者已經將您的加密政策設定成「開啟」，您就可以建立加密的 iFolder 或共用 iFolder。

      如果您在登入會期中未設定通關密語，而且選取「通關密語 > 加密」，系統會提示您輸入通關密語。

    • 選取「一般」啟用選取之 iFolder 的共用功能。

  6. 選取「安全同步」使您建立的 iFolder 中的檔案能夠進行安全同步。

    您可以為一般和加密的 iFolder 啟用安全通道。選取這個選項後，iFolder 可讓您使用安全通道進行伺服器和本地電腦上 iFolder 間的同步。

  7. 按一下「確定」。

    iFolder 隨即出現在 iFolder 瀏覽器視窗中。

在 FAT32 裝載點上建立 iFolder (Linux)

在 Linux 上，iFolder 支援在 FAT32 檔案系統的裝載點上建立及使用 iFolder。iFolder 使用者必須擁有您要儲存 iFolder 的 FAT32 裝載點，且 /etc/fstab 檔案必須明確識別該裝載點之 iFolder 使用者的本地使用者 ID，您才可以在 Linux 電腦的 FAT32 卷冊上建立新的 iFolder 或設定共用 iFolder。

重要：此為必要條件，因為 FAT32 不支援檔案許可權。任何其他不支援許可權的檔案系統都會有相同的問題。

若為雙開機系統，您可以如同在未將資料夾設為 iFolder 的系統中存取一般資料夾一樣，存取 iFolder 中的資料。 如果您在兩個平台中同時將資料夾設為 iFolder，則資料集會儲存在伺服器上兩個不同的 iFolder 中，這將耗費兩倍的已配置伺服器磁碟資源。

因為 FAT32 檔案系統不支援 POSIX 檔案系統許可權，請確定 iFolder 使用者是裝載點的本地擁有者。 請修改 /etc/fstab 檔案，將使用者本地登入識別的使用者 ID (UID) 參數設為在該裝載點上建立 iFolder 的 iFolder 使用者。

  1. 以 root 使用者登入電腦。

  2. 建立要裝載 FAT32 檔案系統的目錄。 在指令提示下，輸入

    mkdir /home/username/fsmount

    將 username 取代為要將 FAT32 磁碟機用於 iFolder 之使用者的使用者名稱。 將 fsmount 取代為要用做裝載點的目錄名稱。 例如，如果 FAT32 磁碟機或分割區的使用者名稱為 jsmith，目錄名稱為 fat32mntpt，則輸入

    mkdir /home/jsmith/fat32mntpt

  3. 將 FAT32 磁碟機裝載到想要的裝載點。 在指令提示下，輸入

    mount -t vfat /dev/hda1 /home/username/fsmount

    將 /dev/hda1 取代為非 Linux 裝置的裝置或分割區名稱。 將 username 取代為 iFolder 使用者的使用者名稱。 將 fsmount 取代為要用做裝載點的目錄名稱。

    繼續範例，其中 FAT32 檔案系統 (vfat) 位於 hda4 裝置 (/dev/hda4)，使用者名稱為 jsmith，裝載點目錄為 /home/jsmith/fat32mntpt，請輸入

    mount -t vfat /dev/hda4 /home/jsmith/fat32mntpt

  4. 編輯 /etc/fstab 檔案，即在定義 FAT32 裝載點的行上新增 iFolder 使用者的 uid=username 參數。

    /dev/hda1  /home/username/fsmount  vfat  uid=username

    繼續範例，將 uid=jsmith 新增至裝載點以修改 /etc/fstab 檔案：

    /dev/hda4  /home/jsmith/fat32mntpt  vfat  uid=jsmith

有關更詳細的資訊，請參閱 法律聲明.
