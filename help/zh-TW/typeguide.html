檔案類型和大小不同步的指示

使用 iFolder 用戶端時，您的 iFolder 管理員可能會在系統、使用者、LDAPGroup 或 iFolder 層級，指定檔案類型限制以及檔案大小上限限制。您的帳戶也可能對於您自己在 iFolder 中資料可使用的空間容量受到限制。當您參與其他 iFolder 時，會針對 iFolder 的擁有者計算在伺服器上所佔用的空間大小。

有些檔案類型不適合同步化，例如作業系統檔案、檔案管理員建立的隱藏檔案，或作為連結檔案集合來執行的資料庫。您僅能針對您的企業包含重要檔案類型，或者排除可能與企業不相關的檔案，例如 .mp3。

作業系統檔案

您不應轉換系統目錄為 iFolder。大部分系統檔案不會經常變更，因此建議保留基本系統的影像檔和關鍵軟體，不要嘗試將這些檔案同步至伺服器。

隱藏檔案

如果您的檔案系統使用隱藏檔案來追蹤顯示優先設定，您的管理員可能會限制這些檔案類型，將其排除在系統上同步化。通常，隱藏檔案僅與其建立時的所在特殊電腦相關，並且每當檔案或目錄被存取時就會變更。您不需要保留這些檔案，將它們同步會導致重複的檔案衝突錯誤。

例如，iFolder 會自動排除兩種隱藏檔案管理員檔案，即 thumbs.db 和 .DS_Store。

資料庫檔案

iFolder 會同步個別檔案或個別檔案變動的部分；它不會以檔案集來同步化檔案。如果您具有作為連結檔案集合執行的資料庫檔案，請不要嘗試在 iFolder 中將它們同步。

請不要嘗試透過製作 GroupWise 歸檔、快取記憶體或將遠端目錄加入 iFolder，來同步化您的 GroupWise® 資料。如果您這麼做，GroupWise 資料會在同步化檔案幾次之後毀損。GroupWise 需要將歸檔中的檔案作為檔案集合來維護。

有關更詳細的資訊，請參閱 法律聲明.
