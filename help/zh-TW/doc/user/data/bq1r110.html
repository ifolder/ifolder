iFolder 的主要功能

開始使用 iFolder 之前，瞭解下列主要功能很重要：

  • iFolder 用戶端
  • iFolder 帳戶
  • 共用的 iFolder
  • iFolder 存取權限
  • 檔案同步化與資料管理
  • 同步化記錄


iFolder 用戶端

iFolder 用戶端會與您的作業系統整合，以在您的本地桌面環境中提供 iFolder 服務。iFolder 支援下列作業系統：

  • Novell Linux Desktop 9
  • Windows 2000/XP
  • Macintosh OS X v10.3 或更新版本

iFolder 用戶端可讓你管理自己的 iFolder，以及選取要在每個電腦上設定的共用 iFolder。如需更多資訊，請參閱管理 iFolders。


iFolder 帳戶

您必須設定至少一個 iFolder 服務帳戶的組態，才可以建立 iFolder。iFolder 會期會在您登入 iFolder 服務帳戶時開始，並在您登出或離開 iFolder 用戶端時結束。僅當您的會期是在使用中且您是在線上工作時，iFolder 才會將檔案與其他使用中 iFolder 同步。無論您是否登入帳戶，都可以隨時存取本地 iFolder 中的資料。如需更多資訊，請參閱設定 iFolder 帳戶。


共用的 iFolder

iFolder 是一個本地目錄，可透過中央電腦選擇性地與使用者指定的使用者群組共用及同步化檔案。所有 iFolder 成員都可以存取 iFolder 檔案，而且如果具有適當權限的話，還可以對其進行變更。您可以跨越多個工作站存取 iFolder，並與其他人共用。

您可以直接在檔案管理員中或在屬於用戶端的 iFolder 瀏覽器中使用 iFolder。在 iFolder 中，您可以設定任何次目錄結構，配合您個人或公司的工作習慣。該次目錄結構在所有成員 iFolder 之間均保持不變。當您共享 iFolder 時，具有「寫入」權限的成員使用者可以修改該目錄結構，然後那些變更會套用至 iFolder 的所有副本。每個使用者都可以將共用 iFolder 置於其電腦中符合放置 iFolder 的指示的任何位置。


iFolder 存取權限

iFolder 用戶端支援 iFolder 的擁有者，並支援其成員的三個存取層級：

  • 擁有者：僅有一個使用者可以充當 iFolder 的擁有者。這通常是建立 iFolder 的使用者。擁有者使用者可以使用 iFolder 用戶端將擁有權轉讓給 iFolder 的其他成員。

    iFolder 的擁有者具有「完全控制」權限。此使用者具有 iFolder 的讀／寫存取權，可管理成員資格及成員使用者的存取權限，還可以移除任何成員的「完全控制」權限。

  • 完全控制：共享 iFolder 的成員，具有「完全控制」存取權限。此成員具有 iFolder 的讀╱寫存取權，並可管理除擁有者之外所有使用者的成員資格及存取權限。

  • 讀╱寫：共用 iFolder 的成員，具有 iFolder 中目錄及檔案的「讀╱寫」存取權限。

  • 唯讀：共用 iFolder 的成員，具有 iFolder 中目錄及檔案的「唯讀」存取權限。此成員可以修改檔案，但本地變更不會同步化至其他成員的 iFolder 副本。

任何 iFolder 使用者都可以擁有一些 iFolder，並可成為其他 iFolder 的成員。作為擁有者使用者，您將始終具有「完全控制」權限。作為成員使用者，您在每個共享 iFolder 中的存取層級可能會不同，這要視具有「完全控制」權限的任何其他成員授予您的存取權而定。


檔案同步化與資料管理

當您設定 iFolder 帳戶時，您可以啟用「記住密碼」，以便您工作時 iFolder 可以在背景中同步化 iFolder 邀請及檔案。

您可以將 iFolder 用戶端的組態設定為在每次您登入電腦桌面環境時自動執行。在您使用本地 iFolder 的檔案時，該會期會在背景中執行，追蹤並記錄您所做的任何變更。

您可以在指定的時間間隔或視需要同步化檔案。


同步化記錄

同步化記錄顯示 iFolder 背景活動的記錄，如上載及下載不同 iFolder 的檔案。
