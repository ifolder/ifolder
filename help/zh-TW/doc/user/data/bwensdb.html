檔案類型和大小不同步的指示

當 iFolder 用戶端與 Novell iFolder 3.x 搭配使用時，您的 iFolder 管理員可能會指定系統、使用者或 iFolder 層級上的檔案類型限制及最大檔案大小限制。在可用於您所擁有之 iFolder 中資料的空間量上，您的帳戶可能也會受到限制。當您參與其他 iFolder 時，會針對 iFolder 的擁有者計算在伺服器上所佔用的空間大小。

一些檔案類型不適合進行同步，例如作業系統檔案、檔案管理員所建立的隱藏檔案，或作為連結之檔案集合執行的資料庫。您可以只包含用於您公司的關鍵檔案類型，或排除很可能與公司無關的檔案，例如 .mp3 檔案。


作業系統檔案

請勿將系統目錄轉換成 iFolder。大部分系統檔案不會經常變更，因此建議保留基本系統的影像檔和關鍵軟體，不要嘗試將這些檔案同步至伺服器。


隱藏檔案

如果您的檔案系統使用隱藏檔案追蹤顯示優先設定，則您的管理員可能會限制那些檔案類型，以在系統的同步中將它們排除在外。通常，隱藏檔案只與建立它們的特定電腦相關，並在每次存取檔案或目錄時發生變更。您不需要保留這些檔案，將它們同步會導致重複的檔案衝突錯誤。

例如，iFolder 會自動排除名為 thumbs.db 和 .DS_Store 的兩個隱藏檔案管理員檔案。


資料庫檔案

iFolder 會同步個別檔案或個別檔案的已變更部份，它不會將檔案作為一個集合進行同步。如果您具有作為連結檔案集合執行的資料庫檔案，請不要嘗試在 iFolder 中將它們同步。

請勿透過將 GroupWise 歸檔、快取或遠端目錄轉換為 iFolder，同步您的 GroupWise® 資料。如果這麼做，則在同步檔案數次之後，GroupWise 資料檔案便會被損壞。GroupWise 需要將歸檔中的檔案作為檔案集合來維護。
