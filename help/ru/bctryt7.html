Обновление клиента iFolder 3.x

Темы раздела:

  • Автоматическое обновление на версию iFolder 3.7

  • Обновление на версию iFolder 3.7 вручную

ВАЖНО.Перед обновлением клиента 3.4.1 необходимо создать резервную копию локального хранилища Simias. Если резервное копирование не выполнено, вся конфигурационная информация компьютера будте утрачена, так как при сбое обновления харнилище Simias удаляется.

Автоматическое обновление на версию iFolder 3.7

Автоматическое обновление возможно только при наличии прав доступа "root", так как для обновления необходимо выполнить установку пакетов RPM. Во время обновления отображается запрос на ввод пароля "root". При отсутствии прав доступа "root" автоматическая установка пакета не выполняется. Однако пакеты клиента загружаются во временный каталог. Клиент Linux загружается в каталог /tmp/<32chardir>/ifolder3-linux.tar.gz. Это позволяет выполнить обновление позднее, так как в каталоге содержатся два сценария установки, которые можно запустить вручную. Пока клиент не будет удален, Вы не можете установить соединение с доменом iFolder 3.7 с использованием клиента iFolder 3.x.

  1. С помощью клиента iFolder 3.x более ранней версии, чем iFolder 3.7, попытайтесь установить соединение с сервером iFolder. Отображается запрос на выполнение обновления.

  2. Щелкните Да для автоматического обновления клиента iFolder 3.x.

При использовании iFolder 3.4.1 выбор варианта Нет и попытка подключения клиента iFolder 3.4.1 к серверу iFolder 3.7 приводит к отображению сообщения об ошибке. Щелкните ОК, чтобы закрыть сообщение.

ПРИМЕЧАНИЕ.Клиент iFolder 3.x (версии, отличной от iFolder 3.4.1) устанавливает соединение с сервером iFolder 3.7 даже в том случае, если выбран отказ от обновления. Хотя соединение клиента 3.x с сервером 3.7 установлено, выполнение каких-либо операций iFolder с помощью этого клиента невозможно. При попытке преобразования существующей папки в папку iFolder сервер не учитывает эти изменения и отображается сообщение об ошибке с информацией о том, что сервер недоступен. Единственной операцией, которую может выполнить пользователь, является загрузка пакетов RPM с сервера для установки iFolder 3.7.

Обновление на версию iFolder 3.7 вручную

  1. Откройте терминал. В командной строке введите install-ifolder.sh.

  2. Введите пароль "root" при появлении запроса.

    Если клиент iFolder уже запущен, выполнение этой операции приводит к его закрытию, затем выполняется обновление веб-сервера XSP и клиента iFolder, а также подключаемого модуля Nautilus-ifolder3.

  3. Запустите iFolder из меню Приложение.

Дополнительную информацию см. в Юридическая информация.
