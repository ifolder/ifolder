Разрешение конфликтов файлов

При возникновении конфликтов между одновременно сохраненными версиями файла или, когда символы в именах файла отличается только значением регистра, файл получает отметку конфликта имен и сохраняется в области конфликта, а клиент iFolder отправляет предупреждение о таком конфликтном файле.


Конфликты версий

Перед устранением конфликтов версий нужно проконсультироваться с другими пользователями. Затем выберите правильную версию в диалоговом окне устранения конфликтов.

  1. При возникновении конфликта в области оповещения появляется всплывающее сообщение. Щелкните значок "X", чтобы закрыть сообщение.

    Можно отключить всплывающие сообщения, установив соответствующие параметры в окне "Настройки iFolder" > на вкладке "Общие".

  2. В области оповещения щелкните правой кнопкой значок iFolder , затем щелкните "Папки iFolder" для открытия средства просмотра iFolder.

  3. Выберите конфликтный файл, затем щелкните "Разрешить" , чтобы просмотреть диалоговое окно "Разрешение конфликтов".

  4. Сравните данные файла на локальной рабочей станции с данными файла на сервере, определите, какую версию файла нужно сохранить, и нажмите кнопку сохранения, чтобы сохранить нужную версию файла.

  5. После устранения конфликта щелкните кнопку "OK".


Конфликты имен файлов

Конфликты имен файла возникают, когда символы имени локальной и серверной версий файла имеют различные значения регистра. Причем, содержимое версий файла может быть идентичным или отличаться. Данный тип конфликта можно устранить путем переименования файлов, но внимательно изучите, какую именно версию файла нужно переименовать.

  • Если содержимое файлов не отличается, можно переименовать любой из этих файлов в диалоговом окне "Разрешение конфликтов".
  • Если содержимое отличается, убедитесь, что у Вас есть копия локального файла и назначьте ей другое имя, затем вернитесь в диалоговое окно "Разрешение конфликтов" и сохраните серверную версию файла. Это действие позволяет сохранить в папке iFolder обе версии файла с различными именами. При переименовании версии файла в диалоговом окне "Разрешение конфликтов", эта версия файла перезаписывает другую, содержимое которой удаляется.

  1. При возникновении конфликта в области оповещения появляется всплывающее сообщение. Щелкните значок "X", чтобы закрыть сообщение.

  2. В области оповещения щелкните правой кнопкой значок iFolder , затем щелкните "Папки iFolder" для открытия средства просмотра iFolder.

  3. Выберите конфликтный файл, затем щелкните "Разрешить" , чтобы просмотреть диалоговое окно "Разрешение конфликтов".

  4. Убедитесь в конфликте имен файла и определите, отличается ли содержимое этих версий файла.

  5. Выполните одно из следующих действий:

    • Если содержимое не отличается, переименуйте один из файлов, щелкните кнопку сохранения, затем нажмите "OK", чтобы закрыть диалоговое окно "Разрешение конфликтов". Действие выполнено.
    • Если содержимое версий отличается, выполните действие 6.

  6. Закройте диалоговое окно "Разрешение конфликтов".

  7. Откройте менеджер файлов и найдите локальную копию конфликтного файла.

  8. Скопируйте файл и сохраните его с уникальным именем в папке iFolder.

    iFolder синхронизирует новый файл с сервером и компьютерами пользователей.

  9. Перейдите в диалоговое окно "Разрешение конфликтов", выберите серверную версию файла, нажмите кнопку сохранения, затем нажмите "OK".

    Серверная версия файла будет загружена на локальную рабочую станцию. Она перезапишет локальную копию файла с этим именем.
