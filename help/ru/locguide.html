Рекомендации по размещению и использованию папок iFolder

Как правило, папки iFolder могут располагаться в любом месте локального жесткого диска как при создании папки iFolder, так и при установке такой папки для совместного использования. Однако существует несколько особых ограничений. При размещении и использовании папок iFolder на компьютере примите во внимание следующие рекомендации:

  • Папка iFolder должна иметь уникальное имя в каталоге, где она будет расположена.

  • В имени папки iFolder запрещено использовать следующие недопустимые символы:

    \/:*?"<>|;

    Другие правила именования см. в разделе Соглашения об именовании папки iFolder и содержащихся в ней папок и файлов.

  • Папка iFolder должна находиться на локальном жестком диске, она не может располагаться на сетевом диске или на логическом (не физическом) диске.

  • Нельзя указывать в качестве папки iFolder корень жесткого диска.

  • Папку iFolder можно создать только в папке, на файлы которой у Вас есть права чтения и записи. При установке совместно используемой папки на компьютере у Вас должны быть права доступа файловой системы на папку, в которой будет расположена папка iFolder.

  • Папка iFolder не может содержать папку приложений, например, папку "Program Files", или находиться в ней.

  • Папка iFolder не может содержать папку метаданные iFolder или находиться в ней. Местоположением по умолчанию папки метаданных является:

    • Linux: Личная папка в подкаталоге <личный каталог>\.local\share\\simias

    • Windows: Каталог C:\Documents and Settings\\имя_пользователя_Windows\Local Settings\Application Data\simias

    • Macintosh: в подкаталоге $HOME/.local/share/simias

  • Папка iFolder не может содержать другие папки iFolder, включая Novell iFolder 2, или находиться в таких папках.x.

    Приложение iFolder не поддерживает использования сетевых папок в качестве папок iFolder. Например папками iFolder не могут быть пользовательские сетевые папки или личные каталоги, назначенные рабочей станции пользователя. Приложение iFolder также не поддерживает пользовательский доступ WebDAV с рабочего стола к папкам iFolder, находящимся на томе с личными данными пользователя на сервере iFolder.

  • Несмотря на то, что можно одновременно выполнить вход под несколькими учетными записями iFolder, папка iFolder может быть ассоциирована только с одно учетной записью iFolder. Можно синхронизировать одну папку iFolder с несколькими учетными записями. При создании папки iFolder права на ее совместное использование предоставляются только пользователям и группам LDAPGroup с персонифицированным доступом на том же сервере или в том же домене iFolder.

  • Папка iFolder, совместно используемая членами группы LDAPGroup, доступна всем членам этой группы LDAPGroup.

  • Если компьютер используется несколькими пользователями, локальные копии папок iFolder, хранящиеся в дисковом пространстве с личными данными пользователя, становятся конфиденциальными, если функции конфиденциальности поддерживаются файловой системой и их использование в системе сконфигурировано.

    В системе NTFS Windows есть встроенные функции безопасности, такие как разрешения на использование файлов и папок, шифрующая файловая система (EFS - Encrypting File System), которая представляет собой технологию хранения зашифрованных файлов на томах NTFS. Эти функции безопасности не поддерживаются файловой системой FAT32. Убедитесь, что другие пользователи этого компьютера (конечно, кроме его администратора) не обладают правами администратора или эквивалентными правами. Администратор наделен правами доступа ко всем файлам и каталогам на компьютере. На совместно используемых компьютерах, где конфиденциальность и безопасность играют существенную роль для каждого пользователя, предусмотрена возможность шифрования локальных копий данных iFolder в системе EFS или в шифрующей файловой системе других производителей.

    Файловые системы Linux и Macintosh разработаны для многопользовательского использования. Разрешения на использование файлов позволяют контролировать доступ пользователей, групп и т.д. Обычно пользователь обладает полными правами доступа к своим личным файлам; группы и другие пользователи не имеют прав доступа к этим файлам.

    Если папки iFolder хранятся локально на томе FAT32 в системе Linux, работать с ними необходимо по-другому, так как файловые системы FAT32 не совместимы с POSIX*. См. дополнительные сведения: Создание папок iFolder в точке монтирования FAT32 (Linux).

  • Если пользователи совместно используют папки iFolder на одном компьютере, могут существовать несколько копий папки iFolder на устройстве (по одной копии для каждого пользователя, который загружал папку с сервера). Различные вошедшие пользователи не могут использовать одну папку для совместной работы с ресурсами iFolder.

Дополнительную информацию см. в Юридическая информация.
