Upgrade di iFolder 3.Client x

In questa sezione verranno illustrati i seguenti argomenti:

  • Upgrade automatico a iFolder 3.7

  • Upgrade manuale a iFolder 3.7

IMPORTANTE:Prima di eseguire l'upgrade del client 3.4.1, è necessario eseguire il backup dell'archivio Simias locale. Se non si esegue il backup, tutte le informazioni di configurazione verranno rimosse dal computer poiché l'archivio Simias viene cancellato quando l'upgrade ha esito negativo.

Upgrade automatico a iFolder 3.7

L'upgrade automatico è possibile solo se si dispone dei diritti di accesso di utente root, poiché il processo richiede l'installazione dei pacchetti RPM. La password root viene richiesta durante l'upgrade. Se non si dispone dei diritti di accesso di utente root, il pacchetto non viene installato automaticamente. Tuttavia, i pacchetti del client vengono scaricati in un'ubicazione temporanea. Il client Linux viene scaricato in /tmp/<32chardir>/ifolder3-linux.tar.gz. Ciò consente di eseguire l'upgrade in un secondo momento poiché l'ubicazione contiene due script di installazione eseguibili manualmente. Finché non viene eseguito l'upgrade del client, non è possibile connettersi al dominio di iFolder 3.7 con un client iFolder 3.x.

  1. Da un client iFolder 3.x precedente a iFolder 3.7, provare a connettersi al server iFolder. Verrà chiesto se si desidera eseguire l'upgrade.

  2. Fare clic su Sì per eseguire automaticamente l'upgrade del client iFolder 3.x.

Per iFolder 3.4.1, se si fa clic su No e si tenta di connettere il client iFolder 3.4.1 al server iFolder 3.7, verrà visualizzato un messaggio di errore. Fare clic su OK per chiudere il messaggio.

NOTA:Una cartella iFolder 3.x (diverso da iFolder 3.4.1) è in grado di connettersi al server iFolder 3.7 anche se si sceglie di non eseguire l'upgrade. Nonostante il client 3.x rimanga connesso al server 3.7, esso non consente di eseguire alcuna operazione. Quando si tenta di convertire una cartella esistente in cartella iFolder, il server non tiene conto delle modifiche e visualizza un messaggio di errore indicante che non è disponibile. L'unica operazione possibile è lo scaricamento dei nuovi RPM dal server per l'installazione di iFolder 3.7.

Upgrade manuale a iFolder 3.7

  1. Aprire un terminale. Al prompt dei comandi, eseguire install-ifolder.sh.

  2. Quando richiesto, immettere la password root.

    In questo modo il client iFolder eventualmente in esecuzione verrà chiuso e il server Web XSP e il client iFolder verranno aggiornati con il plug-in Nautilus-ifolder3.

  3. Avviare iFolder dal menu dell'applicazione.

Per ulteriori informazioni, vedere Note legali.
