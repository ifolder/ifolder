Risoluzione dei conflitti tra file

Quando vengono rilevati conflitti a causa di versioni di file salvate contemporaneamente oppure di nomi file che differiscono soltanto per l'utilizzo di maiuscole e minuscole, viene indicato che si è verificato un conflitto di denominazione e i file vengono archiviati nell'area dei conflitti. Verrà quindi inviato un avviso in modo che i file interessati vengano contrassegnati tramite il client iFolder.


Conflitti di versione

È consigliabile che i conflitti di versione vengano gestiti e risolti in collaborazione con gli altri membri. Sarà quindi possibile scegliere la versione corretta del file nella finestra di dialogo Risolvi conflitti.

  1. Quando si verifica un conflitto, nell'area di notifica viene visualizzato un messaggio. Fare clic su X per chiudere il messaggio.

    È possibile disabilitare la visualizzazione dei messaggi popup tramite le impostazioni disponibili nella scheda Generale della finestra Preferenze di iFolder.

  2. Nell'area di notifica fare clic con il pulsante destro del mouse sull'icona iFolder e scegliere Cartelle iFolder per aprire il browser.

  3. Selezionare il file in conflitto, quindi fare clic su Risolvi per visualizzare la finestra di dialogo Risolvi conflitti.

  4. Esaminare le informazioni relative al file nel computer locale in uso e confrontarle con quelle del server per stabilire quale versione del file mantenere, quindi fare clic su Salva questa versione.

  5. Dopo aver risolto il conflitto, fare clic su OK.


Conflitti dei nomi di file

Questo tipo di conflitto si verifica quando il nome della versione locale del file e quello della corrispondente versione sul server differiscono per l'utilizzo di maiuscole e minuscole. Il contenuto dei file può quindi essere identico o meno. È possibile risolvere tali conflitti rinominando i file, tuttavia è necessario eseguire tale operazione con la massima cautela.

  • Se il contenuto è identico, è possibile rinominare uno dei file nella finestra di dialogo Risolvi conflitti.
  • In caso contrario, eseguire una copia del file locale e assegnargli un nome diverso. Tornare quindi alla finestra di dialogo Risolvi conflitti e salvare la versione del file disponibile sul server. Ciò consente di mantenere entrambe le versioni dei file nella cartella iFolder con nomi differenti. Se si rinomina una versione di file nella finestra di dialogo Risolvi conflitti, tale versione verrà utilizzata per sovrascrivere l'altra, il che determina l'eliminazione definitiva dei contenuti del file sostituito.

  1. Quando si verifica un conflitto, nell'area di notifica viene visualizzato un messaggio. Fare clic su X per chiudere il messaggio.

  2. Nell'area di notifica fare clic con il pulsante destro del mouse sull'icona iFolder e scegliere Cartelle iFolder per aprire il browser.

  3. Selezionare il file in conflitto, quindi fare clic su Risolvi per visualizzare la finestra di dialogo Risolvi conflitti.

  4. Accertarsi che si tratti di un conflitto relativo ai nomi dei file e verificare se il contenuto dei file è identico o meno.

  5. Eseguire una delle operazioni seguenti:

    • Se il contenuto è identico, rinominare uno dei file, fare clic su Salva questa versione, quindi su OK per chiudere la finestra di dialogo Risolvi conflitti. Ciò consente di concludere il task.
    • Se il contenuto è diverso, passare al punto 6.

  6. Chiudere la finestra di dialogo Risolvi conflitti.

  7. Aprire un manager di file per individuare la copia locale del file in conflitto.

  8. Copiare il file e salvarlo con un nome univoco nella cartella iFolder.

    Verrà eseguita la sincronizzazione del file come nuovo elemento nel server e nei computer membri.

  9. Tornare alla finestra di dialogo Risolvi conflitti, selezionare la versione del file memorizzata sul server, fare clic su Salva questa versione e quindi su OK.

    La versione del file sul server viene scaricata nel computer locale e sostituita alla copia locale del file con lo stesso nome.
