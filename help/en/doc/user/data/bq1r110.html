Key Features of iFolder

Before you begin to use iFolder, it is important to understand the following key features:

  • The iFolder Client
  • Shared iFolders
  • iFolder Access Rights
  • Account Set Up for Enterprise Servers
  • Access Authentication
  • File Synchronization and Data Management
  • Sync Log


The iFolder Client

The iFolder client integrates with your operating system to provide iFolder services in your native desktop environment. iFolder supports the following operating systems:

  • Novell Linux Desktop 1.0
  • Windows 2000/XP
  • Macintosh

An iFolder session begins when you log in to an iFolder services account and ends when you log out, or when you exit the iFolder client. Your iFolders synchronize files with other active iFolders only when your session is active and you are working online. You can access data in your local iFolders at any time, whether you are logged in to the account, or not.

The iFolder client allows users to manage their iFolders. For information, see Managing iFolders.


Shared iFolders

An iFolder is a local directory that selectively shares and synchronizes files with a user-specified workgroup in a peer-to-peer sharing model. The iFolder files are accessible to all workgroup members and can be changed by those with the rights to do so. You can share iFolders across multiple workstations and with others.

You work with iFolders directly in your file manager. Within the iFolder, you can set up any subdirectory structure that suits your personal or corporate work habits. The subdirectory structure is constant across all member iFolders. Each workstation can specify a different parent directory for the shared iFolder.


iFolder Access Rights

The iFolder client provides an Advanced Properties page for an iFolder where you can make the iFolder available to other users and to specify access rights for each one. There are multiple levels of access rights for a shared iFolder:

  • Owner: The owner of the primary shared iFolder, with ownership and the Full Control right. The iFolder owner has read/write access to the iFolder, manages membership and access rights for users, and can remove the Full Control right for any member. The owner's computer is the host for the iFolder workgroup and must be active and online to synchronize files with member iFolders. With an enterprise server, the disk space used by the owner's iFolders count against the owner's iFolder disk quota for that account.

    Only one user serves as the owner. This is typically the user that created the iFolder. The owner can transfer ownership status to another user.

  • Full Control: A member of the iFolder workgroup, with the Full Control access right. The user with the Full Control right has read/write access to the iFolder and manages membership and access rights for all users except the owner.

    Multiple users can serve as Admin owners. An Admin owner can add and remove Full Control access rights only for other Admin owners, not the Bill owner.

  • Read/Write: A member of the iFolder workgroup, with the Read/Write access right to directories and files in the iFolder.

  • Read Only: A member of the iFolder workgroup, with the Read Only access right to directories and files in the iFolder.

With an enterprise server, the server hosts every iFolder. Users create the iFolder and the enterprise server makes it available to the specified list of users you want to share it with.

Any given workstation can be both a host and a client in the workgroup model. You can host multiple iFolders on your workstation and be a member of iFolder workgroups hosted by others. Your level of membership in each shared iFolder can differ.


Account Set Up for Enterprise Servers

The iFolder client allows you to set up multiple accounts across domains. You specify the server address, username, and password to uniquely identify an account.


Access Authentication

Whenever an iFolder connects to member iFolders in the workgroup to synchronize files, it authenticates with basic HTML and SSL connections.

Whenever iFolder connects to an enterprise server to synchronize files, it connects with basic HTML and SSL connections to the server, and the server authenticates the user against the domain's LDAP directory service.


File Synchronization and Data Management

An iFolder client runs automatically each time you log in to your computer's desktop environment. The session runs in the background as you work with files in your local iFolders, tracking and logging any changes you make.

When synchronizing files in workgroup mode, the iFolder client performs the data exchange transparently with the appropriate primary iFolder or with other member iFolders, depending on the status of your workstation as a client or a host, respectively, for any given iFolder.


Sync Log

The log displays a log of your iFolder background activity.
