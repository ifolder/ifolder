Managing iFolders

An iFolder is a local directory that selectively shares and synchronizes files with a user-specified workgroup in a peer-to-peer sharing model. The iFolder files are accessible to all workgroup members and can be changed by those with privileges to do so.

You can make any local directory an iFolder if it meets certain criteria:

  • The root directory cannot be an iFolder.
  • An iFolder cannot contain an iFolder.
  • Because an iFolder cannot contain an iFolder, if you also use Novell® iFolder® 2.1x or earlier on your workstation, you must comply with the following restrictions:
    • Any local directory where you store your Novell iFolder data should not be converted to an iFolder.
    • Any directory that contains a local directory where you store your Novell iFolder data should not be converted to an iFolder.

You work with iFolders directly in your file manager.

Within the iFolder, you can set up any subdirectory structure that suits your personal or corporate work habits. The subdirectory structure is constant across all member iFolders.
