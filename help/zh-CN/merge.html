合并 iFolder

合并功能可帮助您合并 iFolder 数据并将 iFolder 服务器和客户程序之间的数据传输量降至最低。您可将服务器上 iFolder 中的内容与本地计算机上现有文件夹的内容合并在一起。

  • 合并 iFolder 与文件夹

  • 将现有 iFolder 移动到其他位置

  • 将迁移的 2.x iFolder 与 3.7 或更高版本的 iFolder 合并

合并 iFolder 与文件夹

您可将同名的 iFolder 和常规文件夹合并在一起。例如，您可将名为“corporate”的 iFolder 的内容与名为“corporate”的常规文件夹的内容合并在一起。

假定 iFolder“corporate”具有三个子文件夹：hr、finance 和 marketing。现在，您要将“corporate”iFolder 的内容下载到名为“corporate”的文件夹中，该文件夹只包含“hr”和“finance”两个文件夹。在此情况下，您不必将所有三个文件夹都下载到“corporate”文件夹，这可避免增加传输率。您可将“corporate”iFolder 与“corporate”文件夹合并在一起，这样，“合并到文件夹”功能将自动避免下载子文件夹“hr”和“finance”，而只下载“marketing”子文件夹。如果“corporate”iFolder 和“corporate”文件夹中的数据是不同的，则出现版本冲突。您可手工解决这些冲突。同样，如果出现文件名冲突，则也可手工解决。

要将服务器上的 iFolder 与本地计算机上的现有文件夹合并在一起：

  1. 在 iFolder 浏览器中，从选定服务器的 iFolder 列表中右击所需的 iFolder，然后选择“合并到文件夹”。

  2. 浏览并选择文件夹，然后单击“确定”。

现在，对这两个文件夹的内容进行比较，并从 iFolder 下载该常规文件夹中不存在的文件。如果 iFolder 中不存在该常规文件夹中的文件，则上载这些文件。

将现有 iFolder 移动到其他位置

要将名为“corporate”的 iFolder 移动到客户机上的其他位置：

  1. 将“corporate”iFolder 转换为常规文件夹。

    重要说明：您不得删除服务器上的 iFolder。

    有关更多信息，请参见将 iFolder 恢复为常规文件夹。

  2. 将“corporate”文件夹移动到其他位置。

    重要说明：您不得在移动文件夹时更改其名称。

  3. 在 iFolder 浏览器窗口中，选择服务器上的“corporate”iFolder，然后单击“合并到文件夹”。

  4. 选择本地计算机上的“corporate”文件夹，然后单击“确定”以合并内容。

    注：如果您使用“立即同步”（而不是“合并到文件夹”，如移动 iFolder中所述），则系统将检查服务器上的 iFolder 中的所有数据并同步所有数据。数据传输时间因此而延长。换言之，“合并到文件夹”将把数据传输量降至最低，因为它只合并 iFolder 和常规文件夹中的内容。

    有关合并的更多信息，请参见合并 iFolder 与文件夹。

将迁移的 2.x iFolder 与 3.7 或更高版本的 iFolder 合并

您可以将迁移的 2.x iFolder 与在 iFolder 3.7 或更高版本服务器中创建的 iFolder 合并。有关迁移的更多信息，请参见Novell iFolder 迁移和升级。

迁移后，2.x iFolder 将移至 iFolder 3.7 或更高版本的服务器。您可以选择将 3.7 或更高版本服务器上的迁移的 iFolder 下载到客户机。但是，客户机上可能已存在这些 iFolder。如果遇到这种情况，请使用合并功能以避免将不必要的数据从服务器传输到客户机。

要将迁移的 2.x iFolder 与 3.7 或更高版本的 iFolder 合并，请

  1. 将 2.x iFolder 迁移到 iFolder 3.7 或更高版本的域。

  2. 将本地计算机上的 2.x iFolder 转换为常规文件夹。

  3. 选择每个 iFolder 的服务器拷贝并单击“合并到文件夹”。

  4. 要合并内容，选择该常规文件夹并单击“确定”。

有关更多信息，请参见 法律声明.
