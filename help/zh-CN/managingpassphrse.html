管理加密 iFolder 的通行口令

加密的 iFolder 都具有唯一的在创建 iFolder 过程中自动生成的数据加密钥。密钥通过用户提供的通行口令进行加密，也可通过使用恢复代理的公共密钥进行加密。

如果 iFolder 管理员在服务器上启用加密策略，则会在 iFolder 帐户创建过程中选择恢复代理。

如果 iFolder 管理员启用了加密，则在 iFolder 创建过程中会向您显示用于输入通行口令的对话框。iFolder 使用通行口令来生成唯一的加密钥，以加密或解密 iFolder 服务器上的 iFolder 数据。如果您忘记了通行口令，则无法访问 iFolder 数据。必须重设置通行口令才能获得访问 iFolder 数据的权限。

恢复加密的 iFolder

使用“通行口令恢复向导”来重设置通行口令，或是恢复加密的 iFolder。按照下面给出的步骤来起动“通行口令恢复向导”：

  1. 在 iFolder 浏览器中单击“安全”>“忘记通行口令”来起动向导。

  2. 在“iFolder 帐户”列表中选择要重设置通行口令的域，然后单击“前进”。

    在 iFolder 帐户的创建过程中，如果选中了“Server_Default”作为恢复代理，则会显示“设定新通行口令”页面。要重设置新通行口令，请参阅在恢复代理为 Server_Default 时重设置通行口令。

    另一方面，如果未选择“Server_Default”作为恢复代理，则必须使用在恢复代理不是 Server_Default 时重设置通行口令来重设置通行口令。

在恢复代理为 Server_Default 时重设置通行口令

要设置新的通行口令，请执行下面的步骤：

  1. 在“设置新通行口令”页面，在“新通行口令”字段指定新的通行口令。

  2. 在“确认通行口令”字段重新输入新通行口令。

  3. 在“口令”字段中指定口令，然后单击“前进”。

  4. 成功重设置通行口令后，单击“完成”来关闭向导。

在恢复代理不是 Server_Default 时重设置通行口令

要在选择的恢复代理不是“Server_Default”时重设置通行口令,请执行下面给出的步骤：

  1. 向导欢迎页面会显示与重设置通行口令所需信息相关的信息。单击“前进”显示“通行口令恢复机制”页面。

  2. 您可在向导中使用三种方法来重设置通行口令：

    • 使用机密文件及其口令： 机密文件及其口令用于重设置通行口令。如果保留有机密文件及其口令，您就可以使用向导指定机密文件的位置并提供其口令。要使用机密文件重设置通行口令，请参阅使用机密文件重设置通行口令。

      注：您可从 iFolder 管理员处获取机密文件及其口令。

    • 使用新数据文件： 您也可使用新数据文件来重设置通行口令。您可在执行旧数据文件的导出操作后，从管理员处获取新数据文件。要使用新数据文件重设置通行口令，请参阅使用新数据文件重设置通行口令。

    • 导出旧数据文件： 如果您没有机密文件或新数据文件，则可以通过向导导出旧数据文件，然后将其通过电子邮件发送给管理员。管理员发回新数据文件后，即可使用向导来重设置通行口令。要导出旧数据文件，请参阅使用导出文件重设置通行口令。

下表总结了您所做的决定。

说明

iFolder 帐户列表

显示已配置的 iFolder 帐户列表。默认情况下会选择第一个登录的帐户。

机密文件的位置

表示可从 iFolder 管理员处获取的证书的私用密钥文件位置。

机密口令

代表机密密钥文件的口令。

一次性口令

代表管理员用来加密新数据文件的口令。

新数据文件的位置

表示加密密钥文件的位置。执行旧数据文件导出操作后，可从 iFolder 管理员处获取新数据文件。

旧数据文件的保存位置

表示导出的密钥文件的保存位置。导出旧数据文件后，您可将其通过电子邮件发送给您的管理员。然后，管理员会发送给您新数据文件。

新通行口令

代表输入用于加密唯一数据加密钥的新通行口令，该数据加密钥在 iFolder 创建的过程中自动生成。

确认通行口令

代表您必须再次输入的新通行口令。

使用机密文件重设置通行口令

要使用机密文件重设置通行口令，请执行下面的步骤：

  1. 选择“我有机密文件及其口令”选项，然后单击“前进”。

  2. 填写以下字段：

    机密文件的位置： 指定机密文件的位置，或单击“浏览”按钮来查找并选择机密文件。

    机密口令： 指定机密文件的口令。

    新通行口令： 指定新通行口令。

    确认通行口令 再次输入新通行口令。

  3. 单击“前进”重设置通行口令。

  4. 单击“完成”结束向导。

注：您可从 iFolder 管理员处获取机密文件及其口令。

使用新数据文件重设置通行口令

要使用新数据文件重设置通行口令，请执行下面的步骤：

  1. 选择“我有管理员发送的新数据文件”选项，然后单击“前进”。

  2. 在“新数据文件位置”字段内指定新数据文件的位置，或单击“浏览”按钮查找和选择新数据文件。

  3. 如果从管理员处收到的新数据文件已加密，请选中“上面的文件已加密？”复选框。选中此复选框后，将启用“一次性口令”字段。

  4. 在“一次性口令”字段中，指定管理员用于加密新数据文件的口令。

  5. 在“新通行口令”字段中指定新通行口令。

  6. 在“确认通行口令”字段中再次输入新通行口令。

  7. 单击“前进”重设置通行口令。

  8. 单击“完成”结束向导。

使用导出文件重设置通行口令

要在无机密文件或新数据文件的情况下重设置通行口令：

  1. 选择“我没有机密文件或数据文件”选项，然后单击“前进”。

  2. 在“旧数据文件保存位置”字段中指定旧数据文件必须保存的位置，或单击“浏览”按钮来指定旧数据文件的保存位置。

  3. 单击“前进”来获取旧数据文件。将显示旧数据文件的位置，以及 iFolder 管理员的电子邮件地址。利用此信息来获取新数据文件。

  4. 单击“完成”结束向导。

  5. 将旧数据文件通过电子邮件发送给您的 iFolder 管理员，请求其返回新数据文件。

  6. 接收到新数据文件后，参阅使用新数据文件重设置通行口令来重设置通行口令。

更改通行口令

出于安全原因，您必须定期更改通行口令。可以使用恢复代理发送的一次性通行口令来重设置通行口令。

  1. 在 iFolder 浏览器中选择“安全”>“更改通行口令”。

  2. 在“更改通行口令”窗口中，选择要为其重设置通行口令的 iFolder 帐户，并指定以下细节：

    输入通行口令： 指定当前的通行口令。

    输入新通行口令： 指定要为当前 iFolder 帐户设置的新通行口令。

    重新键入通行口令： 再次指定新通行口令以进行确认。

    恢复代理： 选择为当前 iFolder 帐户所设置的恢复代理。

  3. 单击“重设置”。

有关更多信息，请参见 法律声明.
