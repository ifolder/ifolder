创建和上载 iFolder

您可通过转换满足以下准则的现有文件夹来创建 iFolder：

  • iFolder 的位置和用途的准则

  • iFolder 及其文件夹和文件的命名约定

如果不存在这样的文件夹，则可在此过程中浏览至一个位置，然后创建新的文件夹。

重要说明：您只能共享常规 iFolder。无法共享加密的 iFolder。

在 iFolder 中，您可以根据个人或公司的工作习惯设置任何子目录结构。子目录结构将始终跨越所有成员 iFolder。每个工作站可为共享 iFolder 指定一个不同的父目录。

超出单个用户可创建的 iFolder 数目限制后，iFolder 将显示错误讯息。例如，如果管理员已将“每个用户的 iFolder 数”策略设置为 10，当您尝试创建第十一个 iFolder 时，iFolder 3.7 将通知您违反策略。

Linux

您可以使用文件管理器或 iFolder 浏览器（在客户应用程序中可用）创建 iFolder。使用下面的方法创建 iFolder：

  • 在 iFolder 浏览器中创建 iFolder

  • 在文件管理器中创建 iFolder

在 iFolder 浏览器中创建 iFolder

  1. 在通知区域，使用下列方法之一打开 iFolder 浏览器：

    • 单击 iFolder 服务图标 。

    • 右击 iFolder 服务图标 ，然后单击“iFolder”。

  2. 单击“创建 iFolder”。

  3. 在“更多选项”下，指定您要用于此 iFolder 的“iFolder 帐户”。

    如果您有多个 iFolder 帐户，则首先列出默认帐户。使用下拉列表可选择其他帐户。

  4. 在浏览器中，导航到要创建 iFolder 的目录。

  5. 使用下列方法之一，选择要转换为 iFolder 的常规文件夹：

    • 如果存在常规文件夹，则请选择它，然后单击“添加”。

    • 如果常规文件夹不存在，单击“创建文件夹”以创建它，选择新创建的文件夹，然后单击“添加”。

  6. 从给定选项选择 iFolder “类型”。您可以决定是创建加密的 iFolder 还是创建常规 iFolder。

    重要说明：仅当 iFolder 管理员将您的加密策略设置为“启用”时，您才有权创建加密的 iFolder。如果未设置为“启用”，则您只能创建常规 iFolder，此选项在默认情况下是选定的。在此情况下，将同时禁用这两种“类型”选项。

    • 选择“通行口令加密”可对所选 iFolder 启用加密。

      如果在登录会话过程中未设置通行口令，且您选择“通行口令加密”，则将出现提示，要求您输入通行口令。在“通行口令”对话框中指定通行口令。

    • 选择“常规”可对所选 iFolder 启用共享。

      iFolder 将在通知区域中显示一条弹出讯息以指出共享 iFolder 可用。共享用户可单击该讯息立即开始设置 iFolder，也可关闭该讯息以在稍后下载 iFolder。

  7. 选择“安全同步”可对所创建的 iFolder 中的文件启用安全同步。

    您可以同时对常规 iFolder 和加密 iFolder 启用安全通道。在选中此选项时，iFolder 允许使用安全通道在服务器上的 iFolder 与本地计算机上的 iFolder 间进行同步。

    注：您还可以通过右击 iFolder 主窗口中的“iFolder”>“属性”选项来为所选 iFolder 启用安全同步。为此，请右击“iFolder”并选择“属性”，然后在“一般”选项卡中选择“安全同步”复选框。

  8. 单击确定。

    该 iFolder 显示在“此计算机上的 iFolder”列表中。

  9. 如果显示“已创建 iFolder”讯息，请执行下列操作之一：

    • 单击“关闭”关闭此讯息。

    • 选择“不要再次显示此讯息”以禁用以后的通知，然后单击“关闭”退出该讯息。

      如果稍后想要显示确认讯息，请转至“iFolder 首选设置”，选择“创建 iFolder 时显示‘确认’对话框”，然后单击“确定”。

在文件管理器中创建 iFolder

  1. 使用文件管理器导航到要转换为 iFolder 的文件夹，然后选择该文件夹。

  2. 使用下列方法之一：

    • 右击该文件夹图标，然后单击“转换为 iFolder”。

    • 在工具栏上，单击“文件” > “转换为 iFolder”。

    文件夹图标 将变为带有 iFolder 徽标 （绿色“i”）的文件夹。

  3. 如果显示“已创建 iFolder”讯息，请执行下列操作之一：

    • 单击“关闭”关闭此讯息。

    • 选择“不要再次显示此讯息”以禁用以后的通知，然后单击“关闭”退出该讯息。

      如果稍后想要显示确认讯息，请转至“iFolder 首选设置”，选择“创建 iFolder 时显示‘确认’对话框”，然后单击“确定”。

Windows

您可以使用文件管理器或 iFolder 浏览器（在客户应用程序中可用）创建 iFolder。使用下面的方法创建 iFolder：

  • 在 iFolder 浏览器中创建 iFolder

  • 在文件管理器中创建 iFolder

在 iFolder 浏览器中创建 iFolder

  1. 在通知区域，使用下列方法之一打开 iFolder 浏览器：

    • 单击“iFolder 服务” 图标。

    • 右击“iFolder 服务” 图标，然后单击“iFolder”。

  2. 单击“创建 iFolder”。

  3. 从下拉列表中，选择您要用于此 iFolder 的 iFolder 帐户。

    如果您有多个 iFolder 帐户，则首先列出默认帐户。您可使用下拉列表选择一个不同的帐户。

  4. 在浏览器中，导航到要创建 iFolder 的目录。

  5. 使用下列方法之一，选择要转换为 iFolder 的常规文件夹：

    • 如果存在常规文件夹，则选择该文件夹，然后单击“确定”。

    • 如果不存在常规文件夹，则单击“新建文件夹”以进行创建，然后选择新创建的文件夹。

  6. 从以下给定选项中选择 iFolder 类型：

    重要说明：仅当 iFolder 管理员将您的加密策略设置为“启用”时，您才有权创建加密的 iFolder。如果未设置为“启用”，则您只能创建常规 iFolder，此选项在默认情况下是选定的。在此情况下，将同时禁用这两种“类型”选项。

    • 选择“通行口令加密”可对所选 iFolder 启用加密。

      如果管理员将您的加密策略设置为“打开”，则您有权创建加密的或共享的 iFolder。

      如果在登录会话过程中未设置通行口令，且您选择“通行口令加密”，则将出现提示，要求您输入通行口令。

    • 选择“常规”可对所选 iFolder 启用共享。

      注：与其他 iFolder 成员共享您的 iFolder 时，iFolder 将在通知区域中显示一条弹出讯息以指出共享 iFolder 可用。共享用户可单击该讯息立即开始设置 iFolder，也可关闭该讯息以在稍后下载 iFolder。

  7. 选择“安全同步”可对所创建的 iFolder 中的文件启用安全同步。

    您可以同时对常规 iFolder 和加密 iFolder 启用安全通道。在选中此选项时，iFolder 允许使用安全通道在服务器上的 iFolder 与本地计算机上的 iFolder 间进行同步。

    注：您还可以通过右击 iFolder 主窗口中的“iFolder”>“属性”选项来为所选 iFolder 启用安全同步。为此，请右击“iFolder”并选择“属性”，然后在“一般”选项卡中选择“安全同步”复选框。

  8. 单击确定。

    该 iFolder 显示在“此计算机上的 iFolder”列表中。

  9. 如果显示“已创建 iFolder”讯息，请执行下列操作之一：

    • 单击“关闭”关闭此讯息。

    • 选择“不要再次显示此讯息”以禁用以后的通知，然后单击“关闭”退出该讯息。

      如果稍后想要显示确认讯息，请转至“iFolder 首选设置”，选择“创建 iFolder 时显示‘确认’对话框”，然后单击“确定”。

Macintosh

您可从客户端应用程序中使用 iFolder 浏览器创建 iFolder。

  1. 从对接台单击 iFolder 服务图标以打开 iFolder 浏览器。

  2. 使用下列方法之一打开该浏览器。

    • 在工具栏上单击“新建”

    • 在菜单栏上单击“文件”>“新建”

  3. 在浏览器中，导航到要创建 iFolder 的目录。

  4. 选择要更改为 iFolder 的常规文件夹，然后单击“打开”。

  5. 从以下给定选项中选择 iFolder 类型：

    重要说明：仅当 iFolder 管理员将您的加密策略设置为“启用”时，您才有权创建加密的 iFolder。如果未设置为“启用”，则您只能创建常规 iFolder，此选项在默认情况下是选定的。在此情况下，将同时禁用这两种“类型”选项。

    • 选择“通行口令加密”可对所选 iFolder 启用加密。

      如果管理员将您的加密策略设置为“打开”，则您有权创建加密的或共享的 iFolder。

      如果在登录会话过程中未设置通行口令，且您选择“通行口令” > “加密”，则将出现提示，要求您输入通行口令。

    • 选择“常规”可对所选 iFolder 启用共享。

  6. 选择“安全同步”可对所创建的 iFolder 中的文件启用安全同步。

    您可以同时对常规 iFolder 和加密 iFolder 启用安全通道。在选中此选项时，iFolder 允许使用安全通道在服务器上的 iFolder 与本地计算机上的 iFolder 间进行同步。

  7. 单击确定。

    该 iFolder 将出现在 iFolder 浏览器窗口中。

在 FAT32 装入点上创建 iFolder (Linux)

在 Linux 上，iFolder 支持在 FAT32 文件系统的装入点上创建和使用 iFolder。在 Linux 计算机的 FAT32 卷上创建新的 iFolder 或设置共享 iFolder 之前，iFolder 用户必须拥有要储存 iFolder 的 FAT32 装入点，且 /etc/fstab 文件必须为该装入点显式标识 iFolder 用户的本地用户 ID。

重要说明：这是必须的，因为 FAT32 不支持文件权限。任何其他不支持权限的文件系统也都存在同样的问题。

对于双重引导系统，可以像访问系统中的常规文件夹（未被设置为 iFolder）一样访问 iFolder 中的数据。如果在两个平台上将文件夹都设置为 iFolder，则数据集储存在服务器上的两个不同 iFolder 中，它将占用两倍的已分配服务器磁盘资源。

由于 FAT32 文件系统不支持 POSIX 文件系统权限，因此请确保 iFolder 用户是装入点的本地拥有者。修改 /etc/fstab 文件，将用户的本地登录身份的用户 ID (UID) 参数设置为在该装入点上创建 iFolder 的 iFolder 用户。

  1. 以根用户身份登录到计算机。

  2. 创建要装入 FAT32 文件系统的目录。在命令提示符处，输入

    mkdir /home/username/fsmount

    将 username 替换为想要对 iFolder 使用 FAT32 驱动器的用户的用户名。将 fsmount 替换为要用作装入点的目录名称。例如，如果对于 FAT32 驱动器或分区，用户名为 jsmith，且目录名称为 fat32mntpt，则输入

    mkdir /home/jsmith/fat32mntpt

  3. 在所需的装入点装入 FAT32 驱动器。在命令提示符处，输入

    mount -t vfat /dev/hda1 /home/username/fsmount

    将 /dev/hda1 替换为非 Linux 设备的设备或分区名称。将 username 替换为 iFolder 用户的用户名。将 fsmount 替换为要用作装入点的目录名称。

    同样还是此示例，如果 FAT32 文件系统 (vfat) 位于 hda4 设备 (/dev/hda4) 上，用户名为 jsmith，且装入点目录为 /home/jsmith/fat32mntpt，则输入

    mount -t vfat /dev/hda4 /home/jsmith/fat32mntpt

  4. 编辑 /etc/fstab 文件，将 iFolder 用户的 uid=username 参数添加到定义 FAT32 装入点的行中。

    /dev/hda1  /home/username/fsmount  vfat  uid=username

    同样还是此示例，通过将 uid=jsmith 添加到装入点来修改 /etc/fstab 文件：

    /dev/hda4  /home/jsmith/fat32mntpt  vfat  uid=jsmith

有关更多信息，请参见 法律声明.
