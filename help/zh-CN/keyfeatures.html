iFolder 的主要功能

在开始使用 iFolder 之前，必须先了解以下主要功能：

  • iFolder 共享

  • iFolder 访问权限

  • 文件同步

  • 迁移

  • 加密

  • 升级

  • 合并

  • 默认 iFolder

  • 增强型万维网访问

  • 同步日志

iFolder 共享

通常，当您在多个地点工作或与其他人合作时，您必须认真负责地管理文件版本。 使用 iFolder，您就可以将最新版本的文件放置到您安装了 iFolder 客户程序并且为这些文件创建了一个 iFolder 的任何计算机上。iFolder 还允许您与其他授权用户或 LDAP 组共享多个 iFolder 及其各自的内容。有关共享 iFolder 的更多信息，请参见共享的 iFolder。

iFolder 客户程序通过中央服务器同步多台计算机上的文件，从而实现对共享的支持。它允许您执行以下操作：

  • 在多台计算机上共享文件

  • 与其他用户共享文件

  • 拥有多个 iFolder

  • 参与其他用户共享给您的多个 iFolder

  • 以 iFolder 的拥有者或成员的身份通过多个帐户参与

可以直接在文件管理器中或 iFolder 浏览器（iFolder 客户程序的一部分）中处理 iFolder。在 iFolder 中，您可以根据个人或公司的工作习惯设置任何子目录结构。子目录结构在所有成员 iFolder 中是稳定的。

共享的 iFolder

一个 iFolder 是一个本地目录，可选择性地通过中央服务器与用户指定的用户共享和同步文件。所有 iFolder 成员都可以访问 iFolder 文件，具有适当权限的成员可以更改 iFolder 文件。您可跨多个工作站来访问 iFolder 并可与单个用户和 LDAP 组共享它们。

您可以决定每个 iFolder 有哪些用户参与以及每个成员的访问级别。同样，您也可以参与协作环境中其他用户拥有的共享 iFolder。当您共享 iFolder 时，具有“写入”权限的成员用户可以修改目录结构，并且这些更改将应用于 iFolder 的所有副本。如果共享 iFolder 符合iFolder 的位置和用途的准则的要求，则每个用户都可以将其放在自己计算机的任何位置上。

iFolder 访问权限

iFolder 客户程序支持拥有者和三种 iFolder 成员访问级别：

重要说明：LDAP 组的成员将继承该组的访问权限集。

  • 拥有者： 只有一个用户担当 iFolder 的拥有者。这通常是创建该 iFolder 的用户。拥有者用户可以使用 iFolder 客户程序将所有权转让给 iFolder 的另一个成员。

    iFolder 的拥有者具有“完全控制”权限。此用户对该 iFolder 具有读/写权限，管理成员用户的成员资格和访问权限并可去除任何成员的完全控制权限。

    如果某一 LDAP 组拥有一个 iFolder，则该组的所有成员用户将被授予完全控制权限。

  • 完全控制： 共享 iFolder 的成员，具有“完全控制”访问权限。此成员对该 iFolder 具有读/写权限，管理拥有者之外的所有用户的成员资格和访问权限。

  • 读/写： 共享 iFolder 的成员，对 iFolder 中的目录和文件具有“读/写”访问权限。此成员可以修改文件，但所做的本地更改也会被同步到其他成员的 iFolder 副本。

  • 只读： 共享 iFolder 的成员，对 iFolder 中的目录和文件具有“只读”访问权限。此成员可以修改文件，但所做的本地更改不会被同步到其他成员的 iFolder 副本。

任何 iFolder 用户或 LDAP 组都可拥有某些 iFolder 并可以是其他 iFolder 的成员。作为拥有者用户，您始终具有“完全控制”权限。作为成员用户，您在每个共享 iFolder 中的访问级别可能各不相同，具体取决于由具有“完全控制”权限的成员授予您的访问权限。

文件同步

当您连接到服务器时，iFolder 可以在您工作时在后台同步 iFolder 成员资格和文件。设置 iFolder 帐户后，即可启用“保存我的口令”以便 iFolder 在该计算机上保存口令，当您下次起动 iFolder 时，它将自动连接到服务器并同步数据而不提示输入口令。

您可以将 iFolder 客户程序配置为在每次登录计算机的桌面环境时自动运行。当您对本地 iFolder 中的文件进行处理时，该会话在后台运行，跟踪并记录您所做的任何更改。

可以手动或者以指定的间隔同步文件。

iFolder 也具有增量同步功能。对 iFolder 文件进行更改后，只将增量或更改同步到服务器。换句话说，增量同步只检索自 iFolder 客户程序和服务器上次同步后的更新。这将会提高效率，因为减少了同步时间。

迁移

Novell iFolder 提供了从 iFolder 2.x 到 Novell iFolder 3.7 的 iFolder 客户程序的简易迁移。在 iFolder 客户程序中提供的迁移向导可以帮助您将现有 iFolder 2.x 数据简便快速地迁移到 iFolder 3.6 和更高版本。有关更多信息，请参见Novell iFolder 迁移和升级。

加密

iFolder 为您的机密 iFolder 文件提供了更高的安全性，可防止未授权用户有意或无意地访问这些文件。过去在服务器一端，数据安全无法保障，所以数据可以被管理员或对服务器拥有未经授权的访问权限的任何人访问。此外，当您通过 ISP 提供商访问 iFolder 时，未经授权的个人可能会有机会发现您的机密文件。现在，您可以对文件加密并将其保存到服务器上，然后通过只有您知道的通行口令来检索它们。如果您忘记通行口令，则在创建加密 iFolder 过程中选择的恢复代理可帮助您恢复数据。

重要说明：无法共享加密的 iFolder。

有关加密的更多信息，请参见管理加密 iFolder 的通行口令。

升级

Novell iFolder 3.7 提供了一种简单的升级方法，可将 iFolder 3.x 升级到 Novell iFolder 3.6 和更高版本的客户程序。有关更多信息，请参见升级 iFolder 3。x 客户程序。

合并

使用合并功能，您可以合并 iFolder 数据并将 iFolder 客户程序和服务器之间的数据传输量降至最低。另外，您还可以使用此功能将现有文件夹移动到客户机上的其他位置。有关更多信息，请参见合并 iFolder。

默认 iFolder

iFolder 允许您在配置新帐户时设置默认 iFolder。当您配置新的 iFolder 帐户时，iFolder 将自动检查是否存在默认 iFolder。如果您尚未创建默认 iFolder，则 iFolder 将提示您创建一个。如果已存在默认值，则 iFolder 客户程序将提示您进行下载。您也可以忽略这两个选项。有关更多信息，请参见配置 iFolder 帐户中的步骤 7。

增强型万维网访问

iFolder 提供一个增强型万维网界面，它允许您访问远程服务器上您的 iFolder 文件。使用万维网访问控制台，您可使用万维网浏览器从未安装 iFolder 客户程序的任何工作站来访问您在 iFolder 服务器上的文件。使用万维网访问界面，您可执行 iFolder 客户程序的拥有操作。

与以前的 iFolder 版本相比较，iFolder 3.6 和更高版本允许您通过增强的万维网访问控制台来创建、浏览、共享、上载和删除 iFolder。移动用户也可以通过万维网使用此功能。

同步日志

同步日志显示您的 iFolder 后台活动的日志，如为不同 iFolder 上载和下载文件、违反策略、已排除的文件以及未同步的文件。

有关更多信息，请参见 法律声明.
