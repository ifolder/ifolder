Diretrizes para a localização e o uso de pastas do iFolder

Em geral, você pode colocar pastas do iFolder em qualquer lugar em seu disco rígido ao criar uma pasta do iFolder ou configurar uma pasta desse tipo que seja compartilhada com você. Porém, existem algumas limitações práticas e específicas. Considere as seguintes diretrizes para incluir e usar pastas do iFolder em seu computador:

  • O nome de uma pasta do iFolder deve ser um nome exclusivo no diretório em que você coloca essa pasta.

  • O nome de uma pasta do iFolder não pode conter os seguintes caracteres válidos.

    \/:*?"<>|;

    Para outras convenções de nomeação, consulte Convenções de nomeação para um iFolder e suas pastas e arquivos.

  • O iFolder deve residir em uma unidade de disco rígido local; ele não pode estar em uma unidade de rede ou unidade não física.

  • Uma pasta do iFolder não pode existir na raiz do disco rígido.

  • Um iFolder pode ser criado somente de uma pasta onde você tenha os direitos de acesso leitura e gravação de arquivos do sistema de arquivos. Ao configurar uma pasta do iFolder compartilhada em um computador, você deve ter direitos de acesso do sistema de arquivos à pasta em que está colocando o iFolder.

  • Um iFolder não pode conter ou residir em uma pasta de aplicativos como a pasta Arquivos de Programa.

  • O iFolder não pode conter ou residir na pasta de metadados do iFolder. A localização padrão da pasta de metadados é a seguinte:

    • Linux: na pasta pessoal no subdiretório <Diretório Pessoal>\.local\share\simias

    • Windows: no diretório C:\Documents and Settings\WindowsLogonUsername\Local Settings\Application Data\simias

    • Macintosh: em $HOME/.local/share/simias

  • Uma pasta do iFolder não pode conter nem residir em outras pastas do iFolder, por exemplo, em uma pasta do Novell iFolder 2. x.

    Pastas de rede não são suportadas como pastas do iFolder. Por exemplo, as pastas do iFolder não podem ser os diretórios pessoais/baseados em rede de usuários mapeados para estações de trabalho desses usuários. O iFolder também não suporta o acesso WebDAV de usuários de sua área de trabalho para sua coleção de pastas do iFolder no volume de dados do usuário no servidor iFolder.

  • Embora seja possível efetuar login em mais de uma conta do iFolder de cada vez, uma pasta do iFolder pode ser associada a apenas uma conta do iFolder. Não é possível sincronizar uma pasta do iFolder com várias contas. Quando você cria uma pasta do iFolder, ela pode ser compartilhada apenas com outros usuários e Grupos LDAP que são provisionados no mesmo domínio ou servidor do iFolder.

  • Uma pasta do iFolder está disponível para todos os membros de um Grupo LDAP quando está compartilhada com esse grupo.

  • Quando vários usuários compartilham um computador, as cópias locais das pastas do iFolder serão particulares em relação ao seu espaço de armazenamento no disco se o sistema de arquivos suportar recursos de privacidade e se você configurar seu sistema para usá-los.

    No Windows, o NTFS contém recursos de segurança embutidos, como permissões de arquivo e pasta e o EFS (Encrypting File System - Sistema de arquivos de criptografia), que é a tecnologia usada para armazenar arquivos criptografados em volumes NTFS. Esses recursos de segurança não são suportados nos sistema de arquivos FAT32. Verifique se o login local identifica os usuários (logicamente que não seja a identidade Administrador) não tem direitos de Administrador ou equivalentes. O usuário Administrador tem acesso a todos os arquivos e diretórios no computador. No caso de computadores compartilhados em que a privacidade e a segurança são essenciais para cada usuário, os usuários podem criptografar sua cópia local de dados do iFolder com o EFS ou a criptografia de sistema de arquivos de terceiros.

    Os sistema de arquivos Linux e Macintosh foram projetados para ambientes de vários usuários. As permissões de arquivo permitem controlar o acesso a usuários, grupos e outros. Os arquivos pessoais de um usuário normalmente são definidos com permissões totais para o usuário e nenhum acesso para os grupos e outras configurações.

    Se você armazenar pastas do iFolder localmente em um volume FAT32 no Linux, ele precisará de tratamento especial, pois os sistemas de arquivos FAT32 não são compatíveis com POSIX*. Para obter informações, consulte a Criando pastas iFolder em um Ponto de Montagem FAT32 (Linux).

  • Se os usuários do mesmo computador compartilharem pastas do iFolder, várias cópias do iFolder poderão existir no dispositivo, com uma cópia para cada usuário que fizer seu download do servidor. Identidades de login local distintas não podem compartilhar uma única localização para as pastas compartilhadas do iFolder.

Para obter mais informações, consulte Informações legais.
