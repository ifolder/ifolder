Création d'un dossier iFolder

Vous pouvez créer un dossier iFolder en convertissant un dossier existant qui répond aux critères suivants :

  • Directives pour la localisation des dossiers iFolder
  • Conventions de dénomination des fichiers et des dossiers d'un compte iFolder

Si le dossier n'existe pas, vous pouvez accéder à un emplacement et y créer un dossier dans le cadre du processus.

Au sein de iFolder, vous pouvez configurer une structure de sous-répertoires adaptée à vos habitudes professionnelles ou personnelles. La structure de sous-répertoires est constante pour tous les dossiers iFolders membres. Chaque poste de travail peut spécifier un répertoire parent différent dans le cas d'un dossier iFolder partagé.

Vous pouvez créer des dossiers iFolder directement dans votre gestionnaire de fichiers et dans le navigateur de dossiers iFolder, disponible dans l'application client. Pour créer un dossier iFolder, vous pouvez procéder de l'une des façons suivantes :

  • Création de dossiers iFolder dans un gestionnaire de fichiers
  • Création de dossiers iFolder dans le navigateur de dossiers iFolder
  • Création de dossiers iFolder sur un point de montage FAT32 (Linux)


Création de dossiers iFolder dans un gestionnaire de fichiers

  1. Accédez au dossier à convertir en dossier iFolder à l'aide du gestionnaire de fichiers, puis sélectionnez ce dossier.

  2. Procédez de l'une des façons suivantes :

    • Cliquez avec le bouton droit de la souris sur l'icône du dossier, puis cliquez sur Convertir en un dossier iFolder.
    • Dans la barre d'outils, cliquez sur Fichier gt; Convertir en un dossier iFolder.

    L'icône de dossier se transforme en une icône sur laquelle l'icône iFolder vient s'ajouter.

  3. Si le message iFolder créé s'affiche, effectuez l'une des opérations suivantes :

    • Cliquez sur Fermer pour fermer la fenêtre.
    • Sélectionnez Ne plus afficher ce message pour désactiver les notifications futures, puis cliquez sur Fermer pour fermer la fenêtre.

      Si, par la suite, vous souhaitez obtenir des messages de confirmation, accédez aux Préférences iFolder, sélectionnez Afficher la boîte de dialogue de confirmation lors de la création de dossiers iFolder, puis cliquez sur OK.


Création de dossiers iFolder dans le navigateur de dossiers iFolder

  1. Dans la zone de notification, cliquez avec le bouton droit de la souris sur l'icône iFolder , puis cliquez sur Dossiers iFolder pour ouvrir le navigateur de dossiers iFolder.

  2. Cliquez sur Nouveau.

  3. Indiquez le compte iFolder à utiliser pour ce dossier iFolder.

    Si vous disposez de plusieurs comptes iFolder, le compte par défaut est affiché en premier. Pour sélectionner un autre compte, utilisez la liste déroulante.

  4. Dans le navigateur, accédez au répertoire dans lequel créer un dossier iFolder.

  5. Sélectionnez le dossier normal à convertir en dossier iFolder, puis effectuez l'une des opérations suivantes :

    • Si un dossier normal existe, sélectionnez-le, puis cliquez sur OK
    • Dans le cas contraire, cliquez sur Nouveau dossier pour le créer, sélectionnez le nouveau dossier, puis cliquez sur OK.

  6. Cliquez sur OK.

    Le dossier iFolder s'affiche dans la liste des dossiers iFolder.

  7. Si le message iFolder créé s'affiche, effectuez l'une des opérations suivantes :

    • Cliquez sur Fermer pour fermer la fenêtre.
    • Sélectionnez Ne plus afficher ce message pour désactiver les notifications futures, puis cliquez sur Fermer pour fermer la fenêtre.

      Si, par la suite, vous souhaitez obtenir des messages de confirmation, accédez aux Préférences iFolder, sélectionnez Afficher la boîte de dialogue de confirmation lors de la création de dossiers iFolder, puis cliquez sur OK.


Création de dossiers iFolder sur un point de montage FAT32 (Linux)

Sous Linux, iFolder prend en charge la création et l'utilisation de dossiers iFolder sur un point de montage des systèmes de fichiers FAT32. Avant de pouvoir créer un dossier iFolder ou configurer des dossiers iFolder partagés sur un volume FAT32 de votre ordinateur Linux, l'utilisateur iFolder doit être propriétaire du point de montage FAT32 dans lequel les dossiers iFolder doivent être stockés. En outre, le fichier /etc/fstab doit identifier de manière explicite l'ID utilisateur local pour le point de montage de l'utilisateur iFolder.

IMPORTANT :  Ceci est obligatoire, car les systèmes de fichiers FAT32 ne prennent pas en charge les autorisations de fichiers. Tous les autres systèmes de fichiers ne prenant pas en charge les autorisations présentent le même problème.

Si vous disposez d'un système à démarrage double, vous pouvez accéder aux données contenues dans les dossiers iFolder comme s'il s'agissait de dossiers normaux à partir du système dans lequel le dossier n'est pas configuré en tant que dossier iFolder. Si vous avez configuré le dossier en tant que dossier iFolder sur les deux plates-formes, les données sont stockées dans deux dossiers iFolder distincts sur le serveur. Ceci utilise le double des ressources qui vous sont allouées sur le serveur.

Comme les systèmes de fichiers FAT32 ne prennent pas en charge les autorisations de systèmes de fichiers POSIX, vérifiez que l'utilisateur iFolder est le propriétaire local du point de montage. Modifiez le fichier /etc/fstab pour définir le paramètre d'ID utilisateur (UID) de l'identité de login local sur l'utilisateur iFolder qui crée des dossiers iFolder sur ce point de montage.

  1. Loguez-vous à votre ordinateur en tant qu'utilisateur racine.

  2. Créez le répertoire où monter le système de fichiers FAT32. A l'invite de commande, tapez

    mkdir /home/nom_utilisateur/montagefs

    Remplacez nom_utilisateur par le nom de l'utilisateur qui emploiera l'unité FAT32 pour ses dossiers iFolder. Remplacez montagefs par le nom du répertoire à utiliser en tant que point de montage. Par exemple, si le nom d'utilisateur est jdurand et le nom du répertoire est fat32mntpt pour une unité ou une partition FAT32, tapez

    mkdir /home/jdurand/fat32mntpt

  3. Montez l'unité FAT32 au point de montage voulu. A l'invite de commande, tapez

    mount -t vfat /dev/hda1 /home/nom_utilisateur/montagefs

    Remplacez /dev/hda1 par le nom du périphérique ou de la partition du périphérique autre que Linux. Remplacez nom_utilisateur par le nom de l'utilisateur de l'application iFolder. Remplacez montagefs par le nom du répertoire à utiliser en tant que point de montage.

    Si l'on poursuit l'exemple, où le système de fichiers FAT32 (vfat) réside sur le périphérique hda4 (/dev/hda4), le nom d'utilisateur et le répertoire du point de montage sont respectivement jdurand et /home/jdurand/fat32mntpt, tapez

    mount -t vfat /dev/hda4 /home/jdurand/fat32mntpt

  4. Modifiez le fichier /etc/fstab en ajoutant le paramètre uid=nom_utilisateur de l'utilisateur iFolder sur la ligne qui définit le point de montage FAT32.

    /dev/hda1 /home/nom_utilisateur/montagefs vfat uid=nom_utilisateur

    Toujours dans le cadre de l'exemple, modifiez le fichier /etc/fstab en ajoutant uid=jdurand au point de montage :

    /dev/hda4  /home/jdurand/fat32mntpt  vfat  uid=jdurand
