Affichage et modification des paramètres de compte iFolder

  1. Cliquez avec le bouton droit de la souris sur l'icône iFolder dans la zone Notification, puis sélectionnez Comptes pour ouvrir la boîte de dialogue Préférences iFolder dans l'onglet Comptes.

  2. Sélectionnez ce compte pour afficher ou modifier les paramètres suivants :

    Paramètre Description

    Serveur

    Nom DNS du serveur d'entreprise. Par exemple : svr21.exemple.com.

    Nom d'utilisateur

    Identité utilisateur de ce compte. Votre administrateur iFolder décide si le format que vous devez utiliser correspond à votre adresse de messagerie ou à votre ID d'utilisateur de réseau (nom commun LDAP).

    Par exemple, si votre nom est Jean Deschamp, votre ID d'utilisateur jdeschamp et votre adresse de messagerie jean.deschamp@exemple.com, votre administrateur configure un paramètre pour le serveur qui détermine lequel des deux formats accepter dans le champ Nom d'utilisateur à des fins d'authentification.

    Vous ne pouvez créer qu'un seul compte par serveur d'entreprise.

    Mot de passe

    Mot de passe correspondant à votre nom d'utilisateur.

    Mémoriser le mot de passe

    Sélectionnez cette option pour vous loguer automatiquement à ce compte iFolder lorsque vous vous loguez à votre ordinateur. Pour plus d'informations, reportez-vous à la section Connexion à un compte iFolder.

    Si votre mot de passe de réseau change, l'authentification automatique échoue la prochaine fois que iFolder essaie de vous loguer au compte iFolder. iFolder vous invite à vous loguer avec votre nouveau mot de passe et vous permet de le spécifier et de l'enregistrer.

    Activer le compte

    Sélectionnez cette option pour autoriser la synchronisation en arrière-plan des invitations iFolder, de la liste des utilisateurs et des dossiers iFolder de ce compte. Ceci ne concerne que les fonctions côté client.

    L'état Activé ou Désactivé d'un compte est indiqué dans la colonne État de la liste des comptes. Vous pouvez voir immédiatement l'état du paramètre Compte activé si vous avez configuré plusieurs comptes.

    Lorsque vous utilisez iFolder avec Novell® iFolder® 3.x, l'administrateur du système iFolder peut désactiver votre compte iFolder (en principe, provisoirement) sur le serveur. Le système devrait se comporter ainsi :

    • Si vous êtes logué lorsque le compte est désactivé sur le serveur, vous pouvez gérer vos dossiers iFolder et vos données iFolder continuent d'être synchronisées avec le serveur jusqu'à ce que vous mettiez fin à la session en vous déloguant ou en désactivant la synchronisation ou jusqu'à ce que le serveur déconnecte votre session, quel que soit l'événement qui se produit en premier.
    • Si le compte est désactivé sur le serveur, vous recevez un message contextuel vous en informant chaque fois que vous essayez de vous loguer. Contactez votre administrateur système iFolder pour obtenir de l'aide.

    Compte par défaut

    Sélectionnez cette option pour que ce compte soit sélectionné par défaut dans la liste déroulante des comptes lorsque vous créez des dossiers iFolder.

  3. Sur l'onglet Comptes des Préférences iFolder, sélectionnez le compte iFolder voulu et cliquez sur Détails pour afficher les informations suivantes le concernant.

    Paramètre Description

    Serveur

    Nom d'hôte du serveur d'entreprise. Par exemple, si le nom DNS du serveur est svr21.exemple.com, svr1 correspond à l'information rapportée.

    Description

    Description destinée à l'utilisateur du serveur d'entreprise, du serveur unique ou de l'ordinateur hôte du groupe de travail telle qu'elle est entrée par l'administrateur de cet utilisateur. Si aucune description n'est entrée, la valeur est « Aucun ».

    Disponible

    Volume total d'espace disponible pour les dossiers iFolder dans ce compte iFolder. La valeur est égale au quota moins l'espace utilisé.

    Utilisé

    Volume total d'espace actuellement utilisé par tous les dossiers iFolder que vous possédez dans ce compte.

    Quota

    Volume total d'espace alloué à votre compte iFolder sur le serveur d'entreprise. Cette valeur peut correspondre au quota par défaut défini pour tous les utilisateurs ou à un quota propre à l'utilisateur. Si aucun quota n'est défini, la valeur est vide ou égale à S/O (sans objet).
