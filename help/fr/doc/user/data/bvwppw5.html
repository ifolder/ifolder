Configuration des paramètres du pare-feu local pour le trafic iFolder

Si vous utilisez un pare-feu local, vous devez le configurer de sorte qu'il permette à iFolder de choisir un port dynamique local ou de configurer un port statique local pour que le client iFolder communique localement (sur le même ordinateur) avec le serveur Mono XSP. iFolder utilise les services Web Mono XSP pour communiquer, via HTTP ou HTTPS (SSL, par défaut), avec les serveurs d'entreprise Novell iFolder comportant vos comptes iFolder. L'administrateur iFolder configure des services iFolder qui déterminent si les communications utilisent HTTP ou HTTPS dans votre environnement.

Le pare-feu ne bloque pas le trafic entre Mono XSP et le serveur d'entreprise Novell iFolder parce que Mono XSP utilise les ports HTTP ou HTTPS standard pour le trafic. Toutefois, si vous bloquez le trafic entre Mono-XSP et le client, celui-ci ne peut pas communiquer avec le serveur. Vous ne pouvez donc pas vous loguer à votre compte iFolder et synchroniser les données iFolder.

A l'origine, la configuration iFolder ne spécifie pas de port particulier à utiliser pour les communications client-Mono XSP. Lorsque vous exécutez iFolder pour la première fois, il ouvre un port local disponible pour écouter les requêtes du serveur Mono XSP et envoyer des données au serveur via Mono XSP. Il écrit le numéro de port dans un fichier de configuration et utilise ce port par la suite sauf s'il n'est pas disponible lorsque iFolder est lancé. Si le port est utilisé par une autre application, iFolder sélectionne un autre port disponible, reconfigure le port dans le fichier de configuration et utilise le nouveau port par la suite. Par exemple, si iFolder est arrêté et qu'une autre application prend le port, lorsque vous redémarrez iFolder, celui-ci doit sélectionner et configurer un autre port.


Utilisation d'un port dynamique local

Si vous utilisez un pare-feu local sur votre ordinateur, il doit détecter le trafic local entre iFolder et le serveur Mono XSP lorsque iFolder contacte le serveur pour synchroniser sa liste d'utilisateurs et ses données iFolder. Permettez au trafic de ne pas recevoir d'alerte contextuelle lorsque le client est lancé. Si vous autorisez le trafic en permanence, le pare-feu ne doit pas vous rappeler de le faire.


Utilisation d'un port statique local

Vous pouvez, si vous le voulez, ajouter le paramètre Web Service Port Range dans la section Service Manager du fichier Simias.config pour autoriser le serveur Mono-XSP à utiliser un port statique ou une plage de ports pour ce trafic local. Ce port ou cette plage de ports peuvent ensuite être ouverts dans le pare-feu pour permettre les communications pour le serveur Mono-XSP.

Vous trouverez le fichier Simias.config aux emplacements suivants :

Système d'exploitation Emplacement du fichier Simias.config

Linux

/home/nom_utilisateur/.local/share/simias/Simias.config

Windows

C:\Documents and Settings\nom_utilisateur\Local Settings\Application Data\simias\Simias.config

Macintosh

/Users/nom_utilisateur/.local/share/simias/Simias.config

Ajoutez le paramètre Web Service Port Range à la section Service Manager du fichier Simias.config. Par exemple :

<configuration>
  <section name="ServiceManager">
    <setting name="WebServicePortRange" value="port_monoxsp" />   
  </section>
</configuration>

Remplacez port_monoxsp par le numéro de port ou la plage de numéros de port correspondants :

  • Spécifiez un numéro de port statique sous la forme d'une valeur de quatre chiffres (par exemple, 1234). Par exemple : 
    <setting name="WebServicePortRange" value="1234" />
  • Spécifiez une plage de ports (par exemple, 1234-1236). Un port de la plage sera utilisé. Par exemple : 
    <setting name="WebServicePortRange" value="1234-1236" />

Pour configurer le système de sorte qu'il utilise un port ou une plage de ports spécifiques :

  1. Configurez le pare-feu de sorte à autoriser le trafic sur le port ou la plage de ports que vous voulez utiliser pour le trafic iFolder.

    Reportez-vous à la documentation de votre pare-feu pour plus d'informations sur l'ouverture d'un port dans le pare-feu.

  2. Arrêtez le client iFolder en cliquant avec le bouton droit sur l'icône iFolder , puis sélectionnez Quitter.

  3. Modifiez le fichier Simias.config en ajoutant le paramètre Web Service Port Range à la section Service Manager.

  4. Démarrez le client iFolder.

    Pour plus d'informations, reportez-vous à Démarrage du client iFolder.
