  • Sincronización de archivos

  • Cifrado

  • Actualizar

Compartición de carpetas iFolder

Por lo general, al trabajar en varias ubicaciones o en colaboración con otros usuarios, debe gestionar con detenimiento las versiones de archivos. Con iFolder, podrá contar con la versión más reciente de sus archivos en cualquier equipo en el que tenga instalado el Cliente iFolder y haya creado una carpeta iFolder para ellos. iFolder también permite compartir varias carpetas iFolder y su contenido independiente con otros usuarios autorizados o grupos LDAP. Para obtener más información sobre carpetas iFolder compartidas, consulte Carpetas iFolder compartidas.

Es posible compartir archivos con el Cliente iFolder mediante la sincronización de archivos en varios equipos a través de un servidor central. iFolder permite realizar las acciones siguientes:

  • Compartir archivos en varios equipos

  • Compartir archivos con otros usuarios

  • Poseer varias carpetas iFolder

  • Participar en varias carpetas iFolder compartidas con otros usuarios

  • Participar como propietario o como miembro de carpetas iFolder en varias cuentas

Puede trabajar con las carpetas iFolder directamente en el gestor de archivos o en el navegador de iFolder que forma parte del cliente. En la carpeta iFolder, es posible configurar cualquier estructura de subdirectorios que se adecue a los hábitos de trabajo personales o de la empresa. La estructura de subdirectorios es constante en todas las carpetas iFolder miembro.

Carpetas iFolder compartidas

Una carpeta iFolder es un directorio local que comparte y sincroniza archivos de forma selectiva través de un servidor central con un grupo de usuarios especificado por el usuario. Todos los miembros en la carpeta iFolder pueden acceder a los archivos de iFolder, así como modificarlos si disponen de los derechos necesarios. Puede acceder a las carpetas iFolder en distintas estaciones de trabajo y compartirlas tanto con usuarios individuales como con grupos LDAP.

El propietario decide quién participa en cada carpeta iFolder y el nivel de acceso de cada miembro. De forma similar, es posible participar en carpetas iFolder compartidas propiedad de otros usuarios de su entorno de colaboración. Al compartir una carpeta iFolder, los usuarios miembro con permiso de escritura pueden modificar la estructura de directorios. Los cambios realizados se aplicarán a todas las copias de dicha carpeta iFolder. Cada usuario puede buscar la carpeta iFolder compartida en cualquier ubicación de su equipo si cumple con los requisitos de Directrices para la ubicación y utilización de carpetas iFolder.

Derechos de acceso a iFolder

El Cliente iFolder es compatible con el propietario y tres niveles de acceso para miembros en una carpeta iFolder:

IMPORTANTE:Los miembros de un grupo LDAP heredan el conjunto de derechos de acceso de dicho grupo.

  • Propietario: Sólo un usuario puede ser el propietario de una carpeta iFolder. Por lo general, se trata del usuario que crea la carpeta iFolder. El usuario propietario puede utilizar el Cliente iFolder para transferir la propiedad a otro miembro de la carpeta iFolder.

    El propietario de una carpeta iFolder tiene el derecho de control total. Este usuario tiene acceso de lectura/escritura a la carpeta iFolder, gestiona la pertenencia y los derechos de acceso de los usuarios miembro, y puede quitar el derecho de control total a cualquier miembro.

    Si un grupo LDAP es propietario de una carpeta iFolder, todos los usuarios miembro de dicho grupo disponen del derecho de control total.

  • Control total: Miembro de una carpeta iFolder compartida con derecho de acceso de control total. Este miembro tiene acceso de lectura/escritura a la carpeta iFolder y gestiona la pertenencia y los derechos de acceso de todos los usuarios, excepto del propietario.

  • Lectura/Escritura: Miembro de una carpeta iFolder compartida con derecho de acceso de lectura/escritura a los directorios y archivos de dicha carpeta iFolder. Este miembro puede modificar un archivo, pero los cambios locales se sincronizarán con las copias de otros miembros de la carpeta iFolder.

  • Sólo lectura: Miembro de una carpeta iFolder compartida con derecho de acceso de sólo lectura a los directorios y archivos de dicha carpeta iFolder. Este miembro puede modificar un archivo, pero los cambios locales no se sincronizarán con las copias de otros miembros de la carpeta iFolder.

Cualquier usuario de iFolder puede ser el propietario de varias carpetas iFolder y ser miembro de otras. El usuario propietario siempre posee el derecho de control total. En calidad de usuario miembro, el nivel de acceso a cada carpeta iFolder compartida puede variar en función del acceso que le haya otorgado cualquier miembro con derecho de control total.

Sincronización de archivos

Cuando está conectado al servidor, iFolder puede sincronizar la pertenencia a una carpeta iFolder y los archivos de ésta en segundo plano mientras el usuario realiza su trabajo. Al configurar una cuenta de iFolder, puede habilitar la opción Recordar mi contraseña para que iFolder recuerde la contraseña en ese equipo. De este modo, la próxima vez que inicie iFolder, la conexión con el servidor se establecerá automáticamente y se sincronizarán los datos sin que se le solicite la contraseña.

Es posible configurar el Cliente iFolder para que se ejecute automáticamente cada vez que entre al entorno de escritorio del equipo. La sesión se ejecuta en segundo plano mientras se trabaja con los archivos de las carpetas iFolder locales, y se lleva a cabo el registro y seguimiento de los cambios efectuados.

Es posible sincronizar los archivos manualmente o a intervalos especificados.

iFolder también incluye funciones de sincronización delta. Cuando realiza algún cambio en un archivo de una carpeta iFolder, sólo se sincronizan los delta o los cambios con el servidor. O, lo que es lo mismo, la sincronización delta sólo recupera las actualizaciones desde la última sincronización efectuada entre el Cliente iFolder y el servidor. De este modo se incrementa la eficacia, ya que se reduce el tiempo de sincronización.

Migración

Novell iFolder proporciona una migración sencilla y sin problemas de iFolder 2.x al Cliente iFolder de Novell iFolder 3.7. El Asistente de migración incluido en el Cliente iFolder permite migrar los datos existentes de iFolder 2.x a iFolder 3.6 de una manera sencilla y rápida. Para obtener más información, consulte la Migración y actualización de Novell iFolder.

Cifrado

iFolder proporciona una mayor seguridad para los archivos de carpetas iFolder confidenciales con el fin de protegerlos de posibles accesos por parte de usuarios no autorizados, ya sea de manera intencionada o no intencionada. Anteriormente, la protección de los datos no estaba del todo asegurada en el servidor, por lo que el administrador o cualquier usuario que obtuviera acceso no autorizado al servidor podía acceder a los datos. Asimismo, al acceder a los archivos de carpetas iFolder a través de un proveedor ISP, existe el riesgo de que un usuario no autorizado pueda acceder sin darse cuenta a sus archivos confidenciales. Ahora, sin embargo, puede cifrar y guardar los archivos en el servidor y recuperarlos por medio de una contraseña codificada que sólo usted conoce. Si la olvida, el agente de recuperación que haya seleccionado durante la creación de la carpeta iFolder cifrada le ayudará a recuperar los datos.

IMPORTANTE:No es posible compartir las carpetas iFolder cifradas

Para obtener más información sobre el cifrado, consulte Gestión de la contraseña codificada para carpetas iFolder cifradas.

Actualizar

iFolder 3.7 de Novell permite realizar una sencilla actualización de iFolder 3.x al Cliente iFolder de iFolder 3.6 de Novell y versiones posteriores. Para obtener más información, consulte la Actualización del Cliente iFolder 3.x.

Fusionar

Mediante la opción de fusión, puede consolidar los datos de iFolder y minimizar la transferencia de datos entre el Cliente y el servidor iFolder. Esta opción también le permite cambiar la ubicación de una carpeta en el equipo cliente. Para obtener más información, consulte la Fusión de carpetas iFolder.

Carpeta iFolder por defecto

iFolder permite definir una carpeta iFolder por defecto durante la configuración de una cuenta nueva. Al configurar una cuenta de iFolder nueva, iFolder comprueba automáticamente la presencia de una carpeta iFolder por defecto. Si no ha creado ninguna, iFolder le pedirá que lo haga. Si ya existe, el Cliente iFolder le solicitará que la descargue. Si lo desea, también puede omitir ambas opciones. Para obtener más información, consulte el Paso 7 en Configuración de una cuenta de iFolder.

Acceso Web mejorado

iFolder incluye una interfaz Web mejorada que permite acceder a los archivos de carpetas iFolder ubicados en servidores remotos. Con la consola de Web Access, puede utilizar un navegador Web para acceder a los archivos del servidor iFolder desde cualquier estación de trabajo sin disponer del Cliente iFolder. La interfaz de Web Access permite realizar todas las operaciones del Cliente iFolder.

En comparación con las versiones anteriores de iFolder, iFolder 3.6 y las versiones posteriores permiten crear, examinar, compartir, cargar y suprimir carpetas iFolder a través de la consola mejorada de Web Access. Los usuarios móviles también podrán beneficiarse de esta funcionalidad a través de Internet.

Registro de sincronización

El registro de sincronización muestra un registro de la actividad en segundo plano de la carpeta iFolder como, por ejemplo, la carga y descarga de archivos de distintas carpetas iFolder, las infracciones de directivas, los archivos excluidos o los archivos no sincronizados.

Para obtener más información, consulte Información legal.
