Actualización del Cliente iFolder 3.x

En esta sección se analizan los elementos siguientes:

  • Actualización automática a iFolder 3.7

  • Actualización manual a iFolder 3.7

IMPORTANTE:Antes de actualizar el cliente 3.4.1, debe realizar una copia de seguridad del sistema de almacenamiento Simias local. Si no se lleva a cabo la copia de seguridad, se perderá toda la información de configuración del equipo, ya que éste suprime el sistema de almacenamiento Simias cuando se produce un error en la actualización.

Actualización automática a iFolder 3.7

La actualización automática sólo es posible si cuenta con derechos de acceso raíz, ya que el proceso debe instalar los paquetes RPM. La contraseña raíz se solicita durante la actualización. Si no dispone de derechos de acceso raíz, el paquete no se instalará automáticamente. Sin embargo, los paquetes de cliente se descargarán en una ubicación temporal. El Cliente Linux se descargará en /tmp/<32chardir>/ifolder3-linux.tar.gz. De este modo podrá actualizarlo más adelante ya que la ubicación contiene dos guiones de instalación que puede ejecutar manualmente. Hasta que el cliente no se actualice, no podrá conectarse al dominio iFolder 3.7 con un Cliente iFolder 3.x de Novell.

  1. Desde una carpeta iFolder 3.En un cliente iFolder x anterior a iFolder 3.7, intente conectarse al servidor iFolder. El sistema le solicitará si desea llevar a cabo la actualización.

  2. Haga clic en Sí para actualizar automáticamente el cliente iFolder 3.x de Novell.

Para iFolder 3.4.1, si hace clic en No e intenta conectar el Cliente iFolder 3.4.1 al servidor iFolder 3.7, aparecerá el siguiente mensaje de error. Haga clic en Aceptar para cerrar el mensaje.

NOTA:Una carpeta iFolder 3.x (que no sea un cliente iFolder 3.4.1) se conectará al servidor 3.7 aunque seleccione que no desea llevar a cabo la actualización. Aunque el cliente 3.x permanezca conectado al servidor 3.7, no podrá realizar ninguna de las operaciones de iFolder con él. Cuando intente convertir una carpeta existente en una carpeta iFolder, el servidor no tendrá en cuenta estos cambios y mostrará un mensaje de error para indicar que no está disponible. La única operación que puede llevar a cabo es la descarga de los módulos RPM del servidor para instalar iFolder 3.7.

Actualización manual a iFolder 3.7

  1. Abra un terminal. En el indicador de comandos, ejecute install-ifolder.sh.

  2. Introduzca la contraseña raíz cuando se le solicite.

    Si está en ejecución, el Cliente iFolder se cerrará. A continuación, el servidor Web XSP y el Cliente iFolder se actualizarán junto con el módulo auxiliar (plug-in) Nautilus-ifolder3.

  3. Inicie iFolder desde el menú Aplicación.

Para obtener más información, consulte Información legal.
