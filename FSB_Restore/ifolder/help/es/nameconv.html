Convenciones de denominación de una carpeta iFolder y de las carpetas y archivos que contiene

El Cliente iFolder impone convenciones de denominación que abarcan las restricciones colectivas de los sistemas de archivos de Linux y Windows. Para poder sincronizar una carpeta iFolder, una carpeta o un archivo, éstos deben tener un nombre válido que cumpla con las convenciones de denominación.

Utilice las convenciones de denominación siguientes para las carpetas iFolder, así como para las carpetas y archivos que contienen:

  • iFolder es compatible con el conjunto de caracteres Unicode* con la codificación UTF-8.

  • No utilice los siguientes caracteres no válidos en los nombres de carpetas iFolder ni en las carpetas y archivos que contienen:

    \/:*?"<>|;

    iFolder crea un conflicto de nombres si se utilizan caracteres no válidos en un nombre de archivo o carpeta. Para poder sincronizar el archivo o la carpeta, el conflicto debe resolverse.

  • La longitud máxima de nombre de un componente de una sola vía depende del sistema de archivos del cliente. En cuanto a los nombres de archivo, la longitud máxima incluye el punto (.) y la extensión de archivo.

    Si intenta descargar o sincronizar una carpeta iFolder cuya longitud de nombre supera el límite máximo admitido por el sistema de archivos, iFolder le muestra un mensaje de error.

  • A pesar de que los nombres de carpetas iFolder, carpetas y archivos no distinguen entre mayúsculas y minúsculas, éstas se conservarán. Si los nombres de archivos sólo varían en las mayúsculas y minúsculas, iFolder generará un conflicto de nombres. Para poder sincronizar el archivo o la carpeta, el conflicto debe resolverse.

  • Si crea carpetas iFolder en un sistema de archivos FAT32 en Linux, evite la denominación de archivos con todos los caracteres en mayúscula. La gestión de archivos VFAT o FAT32 en Linux cambia automáticamente los nombres de archivo con todos los caracteres en mayúscula y cumple el formato de archivo de MS-DOS 8.3 según el cual los nombres con todos los caracteres en mayúscula cambian a minúscula. Esto crea problemas de sincronización en los archivos si iFolder se establece con derecho de acceso de sólo lectura.

Para obtener más información, consulte Información legal.
